下面是两个agent给出的方案，需要整合融合下


4.1 输出一：标注后的原图
    - 叠加对象框、深度信息、透视线

4.2 输出二：2D 俯视布局图（Top-Down View）
    - 标注相机位置和视锥
    - 标注各对象的平面位置（X, Y）
    - 标注估算的尺寸

4.3 输出三：关键 3D 参数汇总表
    - 每个对象：类型、3D 位置(X,Y,Z)、尺寸估算、旋转
    - 相机：位置、朝向（Pitch/Yaw）、FOV
    - 地形：大致范围、高度变化

4.4 输出四：侧视图（可选）
    - 展示高度关系（地形起伏、树木高度、山脉高度）


输入： Step 3 输出
输出： 俯视布局图 + 标注
细化：
4.1 俯视 2D 布局图
● 将 Step 3 的 3D 坐标投影到 XY 平面
● 绘制每个物体的占地轮廓（矩形/圆形近似）
● 标注 ID、尺寸、朝向
● 网格背景（每格代表 1m 或 5m）
● 这就是 UE 场景的"蓝图"
4.2 侧视标高图（可选）
● 从侧面看的高度标注
● 适用于有高差的场景（山坡、多层建筑）
4.3 对照验证图
● 用 Step 3 的相机参数，将 3D 布局反投影回 2D
● 与原画叠加对比，验证位置是否吻合
● 这一步是核心质量关卡
人工校验点： 对照验证图是关键——如果反投影和原画对不上，说明 Step 3 的估算有问题，需要调整。
