# ArtClaw Tool Manager - UI 设计文档

> 版本: 4.0
> 更新日期: 2026-04-11
> 说明: 基于 UE/DCC 现有设计重新设计 Web 端界面

---

## 设计原则

### 与 UE/DCC 设计对齐

Web 端对话面板参考 UE Slate 和 DCC Qt 面板的成熟设计：
- **布局结构**: 采用类似的垂直分层布局
- **组件命名**: 保持与 UE/DCC 一致的组件命名
- **交互逻辑**: 复用经过验证的交互模式
- **视觉风格**: 暗色主题，与 DCC 工具风格统一

### Workflow/Tool 执行逻辑（核心）

**所有 Workflow 和 Tool 的执行都通过对话面板，由 AI 协助完成**:

```
用户点击 [运行] → 跳转对话面板 → 输入框预填命令 + 引导卡片 → 用户发送 → AI 协助填参 → 执行
```

**执行流程**:
1. 用户在 Workflow/Tool 页面点击 [运行]
2. 系统自动跳转回【对话面板】
3. 输入框**预填**消息（不自动发送）："我想运行 Workflow 'xxx'，请帮我填写参数"
4. 消息区显示一条**系统引导卡片**："即将运行 'xxx'，点击发送开始"
5. 用户点击发送（可在发送前修改消息）
6. AI 在【右侧面板】显示参数表单
7. 用户可手动填写，也可让 AI 根据对话上下文自动填写
8. 用户确认后执行，结果在消息流中显示

**设计目标**: 让更多工作由 AI 来执行，而非用户手动操作
**关键约束**: 跳转后用户必须看到预填内容和引导，不能是空白页面

### 响应式适配

Web 端需要适配不同屏幕尺寸：
- **桌面端**: 完整三栏布局（侧边栏 + 对话区 + 工具区）
- **平板端**: 双栏布局（可折叠侧边栏）
- **移动端**: 单栏布局，底部固定工具栏

**状态栏显示信息**:
| 位置 | 显示内容 | 说明 |
|------|----------|------|

| 元素 | 说明 | 交互 |
|------|------|------|
| 状态按钮 | 显示连接状态（●已连接/●已断开） | 点击展开/折叠 Row 3 |

- 点击设置按钮 → 打开设置面板

**状态颜色**:
- 已连接: ● #4CAF50 (绿色)
- 已断开: ● #888888 (灰色)
- 连接中: ● #FF9800 (橙色)

**上下文用量颜色**:
- <60%: #4CAF50 (绿色)
- 60-80%: #FF9800 (橙色)
- >80%: #F44336 (红色)

### 2.3 平台切换说明

**DCC 软件切换 (DCCSelector)**:
- 选择当前默认对接的 DCC 软件
- 选项: UE5 / Maya 2024 / ComfyUI / SD / SP / None
- 影响: 工具调用时默认发送到哪个 DCC

**Agent 平台切换 (AgentPlatformSelector)**:
- 选择 Agent 平台（与模型无关）
- 选项: OpenClaw / LobsterAI / Claude (直接) / 等
- 影响: 消息发送到哪个平台处理

**Agent 切换 (AgentSelector)**:
- 选择当前平台下的具体 Agent
- 选项依赖 Agent 平台（如 OpenClaw 下有 Claude/GPT 等）
- 影响: 使用哪个 AI 模型处理消息

**切换关系**:
```
用户切换 Agent 平台 → 重新获取该平台下的 Agent 列表 → 默认选中第一个 Agent
用户切换 Agent → 保持当前平台不变
用户切换 DCC → 仅改变默认 DCC 目标，不影响 Agent
```
