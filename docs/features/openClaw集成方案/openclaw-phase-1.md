# OpenClaw Phase 1 Features Backlog

> **对应主文档**：`docs/specs/openClaw集成方案.md`  
> **对应路线图**：`docs/UEClawBridge/specs/开发路线图.md` 阶段 1：开放网关与安全沙盒

---

## 阶段目标

让 OpenClaw 能安全驱动 UE 执行 Python，具备错误回传、回滚与稳定执行能力。

---

## Feature 列表

### 1. `feature/openclaw-run-ue-python`

**目标**：提供 OpenClaw 到 UE 的核心执行入口。

**范围**：
- `run_ue_python(code: str)` Tool
- 标准化返回结构
- Traceback 回传

**交付物**：
- 执行入口定义
- 返回结果结构
- 错误回传约定

**验收标准**：
- [ ] 能执行基础和中等复杂度 UE Python 代码
- [ ] 语法错误和运行时错误可回传
- [ ] 执行成功与失败格式统一

**路线图映射**：
- 1.2 万能执行器

---

### 2. `feature/openclaw-static-guard`

**目标**：在执行前拦截明显危险指令。

**范围**：
- AST 扫描
- 黑名单规则
- 风险分类

**交付物**：
- 预审规则列表
- 拦截结果结构
- 错误提示信息

**验收标准**：
- [ ] 危险调用会在执行前被拒绝
- [ ] 返回结果能说明被拦截原因
- [ ] 黑名单规则可扩展

**路线图映射**：
- 1.3 静态指令预审

---

### 3. `feature/openclaw-undo-transaction-guard`

**目标**：让 AI 写操作具备撤销能力。

**范围**：
- `ScopedEditorTransaction` 封装
- 写操作事务保护
- Undo 兼容验证

**交付物**：
- 事务包装策略
- 场景修改回滚说明
- 异常事务处理约定

**验收标准**：
- [ ] 关键写操作被纳入事务
- [ ] Ctrl+Z 可恢复修改前状态
- [ ] 事务失败不会遗留半执行状态

**路线图映射**：
- 1.4 事务保护

---

### 4. `feature/openclaw-main-thread-dispatch`

**目标**：保证所有 UE API 调用在主线程执行。

**范围**：
- 命令队列
- 主线程切换
- 结果异步回传

**交付物**：
- 主线程调度策略
- 执行链路说明
- 异步返回约定

**验收标准**：
- [ ] 异步线程不直接调用 `unreal` API
- [ ] 执行过程无线程相关崩溃
- [ ] 结果可稳定回传给 OpenClaw

**路线图映射**：
- 1.5 主线程调度器

---

### 5. `feature/openclaw-context-shortcuts`

**目标**：为 OpenClaw 生成的脚本提供简化上下文。

**范围**：
- `S` / `W` / `L` 快捷上下文
- 常用对象注入
- 运行环境说明

**交付物**：
- 上下文字段定义
- 注入规则说明
- 示例脚本

**验收标准**：
- [ ] 脚本可直接使用快捷上下文
- [ ] 文档明确这些上下文的含义
- [ ] 可减少脚本样板代码

**路线图映射**：
- 1.6 快捷上下文注入

---

## 阶段验收清单

- [ ] OpenClaw 可执行中等复杂度 UE Python 指令
- [ ] 执行失败能返回错误给 OpenClaw
- [ ] 可撤销场景修改
- [ ] 执行过程不发生线程崩溃
