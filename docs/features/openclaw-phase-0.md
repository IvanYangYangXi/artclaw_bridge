# OpenClaw Phase 0 Features Backlog

> **对应主文档**：`docs/specs/openClaw集成方案.md`  
> **对应路线图**：`docs/UE_Editor_Agent/specs/开发路线图.md` 阶段 0：环境预设与工程初始化

---

## 阶段目标

让 OpenClaw 能识别 UE Agent、完成最小连接、具备连接可见性和基础日志能力，为后续执行能力打底。

---

## Feature 列表

### 1. `feature/openclaw-config-bootstrap`

**目标**：提供 OpenClaw 接入 UE Editor Agent 的最小可用配置。

**范围**：
- `stdio` 接入示例
- `WebSocket` 接入预留示例
- 环境检查清单

**交付物**：
- OpenClaw 配置样例
- 接入说明文档
- 常见错误排查说明

**验收标准**：
- [ ] OpenClaw 可加载 MCP 配置
- [ ] `tools/list` 能稳定返回结果
- [ ] 配置方式与项目目录结构一致

**路线图映射**：
- 0.1 插件目录与模块骨架

---

### 2. `feature/openclaw-connection-status`

**目标**：将 OpenClaw 连接状态纳入 UE 插件状态中心统一管理。

**范围**：
- 接入 `UUEAgentSubsystem`
- UI 状态联动
- 连接状态广播

**交付物**：
- 连接状态字段
- 状态变更事件
- 工具栏状态反馈

**验收标准**：
- [ ] OpenClaw 连接成功后 UI 状态变化正确
- [ ] 断开连接后状态可重置
- [ ] 状态变更可被日志记录

**路线图映射**：
- 0.2 C++ 编辑器子系统
- 0.3 开发者菜单与入口

---

### 3. `feature/openclaw-dependency-bootstrap`

**目标**：保证 OpenClaw 接入所需桥接依赖和插件侧依赖可被正确识别和准备。

**范围**：
- 桥接层依赖说明
- 插件私有依赖准备策略
- 环境诊断前置检查

**交付物**：
- 依赖清单
- 安装指引
- 初始化检查说明

**验收标准**：
- [ ] 必要依赖缺失时有明确提示
- [ ] 依赖安装路径不污染引擎环境
- [ ] 文档可指导首次部署完成依赖准备

**路线图映射**：
- 0.4 依赖隔离管理机制

---

### 4. `feature/openclaw-log-observability`

**目标**：建立 OpenClaw 请求进入 UE 后的基础可观测链路。

**范围**：
- 请求日志
- 连接日志
- 初始化日志

**交付物**：
- Output Log 规范输出
- 请求 ID / 时间 / 结果记录约定
- 调试信息前缀规范

**验收标准**：
- [ ] 请求进入 UE 后可在日志中定位
- [ ] 连接成功 / 失败有清晰日志
- [ ] 调试信息可用于排查接入问题

**路线图映射**：
- 0.5 日志与调试重定向

---

## 阶段验收清单

- [ ] OpenClaw 能发现 UE Agent
- [ ] `tools/list` 返回基础能力
- [ ] UE 插件 UI 能反映连接状态
- [ ] 依赖准备流程清晰可执行
- [ ] 日志可追踪一次完整连接过程
