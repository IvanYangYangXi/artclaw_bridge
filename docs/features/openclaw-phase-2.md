# OpenClaw Phase 2 Features Backlog

> **对应主文档**：`docs/specs/openClaw集成方案.md`  
> **对应路线图**：`docs/UE_Editor_Agent/specs/开发路线图.md` 阶段 2：感知增强与原生交互

---

## 阶段目标

让 OpenClaw 从“能执行”升级为“有上下文、可确认、有结果反馈”的闭环协作状态。

---

## Feature 列表

### 1. `feature/openclaw-in-editor-chat-entry`

**目标**：为 OpenClaw 提供 UE 内对话入口或唤起承接位。

**范围**：
- 编辑器内入口预留
- 嵌入式承接位
- 外部唤起方案说明

**交付物**：
- 入口定义
- 唤起方式说明
- UI 承接点说明

**验收标准**：
- [ ] 预留清晰入口
- [ ] 明确嵌入或外部唤起方案

**路线图映射**：
- 2.1 编辑器内对话面板

---

### 2. `feature/openclaw-resource-context-sync`

**目标**：把 UE 编辑器上下文映射为 OpenClaw 可读取资源。

**范围**：
- 选中对象资源映射
- 当前资产资源映射
- 相机位置资源映射
- 运行时数据精简

**交付物**：
- MCP Resource 列表
- JSON 输出结构
- 数据精简规则

**验收标准**：
- [ ] 选中对象 / 当前资产 / 相机位置可读取
- [ ] 返回 JSON 精简结构

**路线图映射**：
- 2.2 动态上下文感知与资源映射
- 2.6 运行时数据精简

---

### 3. `feature/openclaw-risk-confirmation`

**目标**：高危操作强制人工确认。

**范围**：
- 风险级别判断
- 原生确认弹窗
- 确认结果回传

**交付物**：
- 风险规则
- 确认 UI 说明
- 回传结果结构

**验收标准**：
- [ ] 删除、大范围修改、保存资产等行为可被拦截确认
- [ ] 确认结果可反馈给 OpenClaw

**路线图映射**：
- 2.3 风险分级确认 UI

---

### 4. `feature/openclaw-viewport-feedback`

**目标**：让执行结果在视口中可见。

**范围**：
- Actor 高亮
- 自动聚焦
- 结果可视反馈

**交付物**：
- `focus_on_actor`
- `highlight_actors`
- 反馈说明

**验收标准**：
- [ ] 可高亮受影响 Actor
- [ ] 可自动聚焦目标区域

**路线图映射**：
- 2.4 视口同步与视觉反馈

---

### 5. `feature/openclaw-editor-mode-filter`

**目标**：根据当前编辑器模式收缩上下文和能力提示。

**范围**：
- 材质编辑器模式识别
- 蓝图编辑器模式识别
- 关卡编辑器模式识别

**交付物**：
- 模式识别策略
- Prompt 过滤规则
- 模式切换说明

**验收标准**：
- [ ] 材质 / 蓝图 / 关卡模式下提示策略可区分

**路线图映射**：
- 2.5 编辑器模式智能过滤

---

### 6. `feature/openclaw-error-self-healing-loop`

**目标**：把执行错误重新带回 OpenClaw，形成自修复循环。

**范围**：
- Traceback 自动回传
- 错误上下文注入
- 重试修复链路

**交付物**：
- 错误回传机制
- 自修复提示模板
- 重试约定

**验收标准**：
- [ ] Traceback 能自动回传
- [ ] 支持失败后修复重试

**路线图映射**：
- 2.7 错误感知与自修复循环

---

## 阶段验收清单

- [ ] OpenClaw 能主动读取 UE 上下文
- [ ] 高危操作有确认链路
- [ ] 执行结果可在视口中直接感知
