# OpenClaw Phase 4 Features Backlog

> **对应主文档**：`docs/specs/openClaw集成方案.md`  
> **对应路线图**：`docs/UE_Editor_Agent/specs/开发路线图.md` 阶段 4：分发、优化与生态

---

## 阶段目标

把 OpenClaw 集成沉淀为可部署、可共享、可审计、可分发的标准方案。

## 并行线设计

阶段 4 拆分为三条可并行推进的工作线，不再是严格串行依赖：

```
分发线:       4.1 技能包封装 → 4.3 团队 Git 同步
基础设施线:   4.2 审计日志（独立）     4.6 健康体检 ✅（独立）
部署线:       4.5 一键部署 ✅（独立）
暂缓:         4.4 社区共享（无近期计划）
```

### 部署线

#### 5. `feature/openclaw-one-click-deploy` ✅

**目标**：让非开发岗位也能零配置使用插件。  
**依赖**：无（独立）  
**状态**：✅ 已完成

**范围**：
- 插件启动时自动检测缺失依赖
- 后台线程自动安装，不阻塞编辑器
- 离线 bundle 降级支持

**交付物**：
- `init_unreal.py` 内置依赖管理流程
- 快速检测 + 异步安装 + 自动启动 MCP

**验收标准**：
- [x] 插件复制到项目即可工作，无需独立安装脚本
- [x] 自动安装依赖（Python包）
- [x] 不阻塞编辑器启动

**与原设计差异**：原计划为独立 `setup.bat`，实际采用启动时自检方案，体验更无感。

**路线图映射**：
- 4.5 极简一键部署方案
