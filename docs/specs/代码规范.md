# AI 协作项目文件规范 (File Standards)

## 1. 代码文件规范 (Code Files)
- **黄金区间**：100 - 300 行。此区间内 AI 的修改逻辑最准，响应最快。
- **硬性上限**：500 行。超过此限制必须进行逻辑拆分。
- **单一职责**：一个文件只处理一件事（一个类、一组相关的纯函数、或一个组件）。
- **禁止省略**：在输出代码时，严禁 AI 使用 `// ... existing code ...` 这种省略方式，除非手动授权。

## 2. 引用与存储路径 (Structure)
- **引用标注**：代码首行建议标注 `// Ref: docs/ARCH.md#Section`，引导 AI 关联上下文。
- **路径深度**：目录层级建议不超过 4 层，方便 AI 快速扫描文件树。
- **配置分离**：环境配置、常量定义、Mock 数据必须独立于逻辑代码。

## 3. AI 交互触发阈值
- **文件 > 500 行**：AI 必须在修改前主动提出重构建议。
- **文件 > 1MB (文本数据)**：禁止 AI 直接读取全量内容，应通过脚本或分段读取。
