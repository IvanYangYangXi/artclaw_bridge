# ArtClaw 项目结构说明

## 项目概述

**ArtClaw** 是一个跨平台的数字内容创作（DCC）工具集，通过MCP（Model Context Protocol）协议为各类3D软件提供AI辅助功能。

### 核心理念
- **统一协议**：所有DCC软件通过MCP与AI通信
- **插件化架构**：每个软件独立实现插件，共享核心逻辑
- **依赖隔离**：各插件拥有独立的Python环境，互不干扰
- **文档驱动**：详细的文档指导开发，确保质量一致

## 各目录说明

### 📁 `docs/specs/` - 通用规范
**作用**：跨项目共享的文档规范和工作流程

**包含内容**：
- AI文档编写规范
- Git工作流规范
- 代码审查标准
- 发布流程

**维护者**：项目负责人

---

### 📁 `docs/{SubProject}/` - 子项目文档
每个DCC软件插件都有独立的文档目录，例如：

#### UE_Editor_Agent（Unreal Engine）
**特点**：
- 使用C++和Python混合开发
- Slate UI框架
- Editor Subsystem架构
- 特有的日志重定向机制

**专属规范**：
- UE命名规范（F/U/A/S前缀）
- Slate UI最佳实践
- 编辑器扩展规范

#### Maya_Editor_Agent（未来）
**预期特点**：
- Python + MEL混合编程
- Maya命令和API
- 节点编辑器集成
- 特有的UI框架（Qt）

---

### 📁 `subprojects/` - 代码实现
每个子项目都有独立的工程目录：

**UEDAgentProj/**：
- 标准的Unreal Engine工程
- 包含插件源码和资源
- 可直接在UE编辑器中打开

**结构说明**：
```
UEDAgentProj/
├── Config/                 # 工程配置
├── Content/                # UE内容（关卡、材质等）
├── Plugins/
│   └── UEEditorAgent/      # 插件主目录
│       ├── UEEditorAgent.uplugin
│       ├── Source/         # C++源码（模块、子系统、UI）
│       ├── Content/        # 插件内容
│       │   └── Python/     # Python脚本（MCP服务器）
│       └── Resources/      # 图标、样式资源
└── UEDAgentProj.uproject   # 工程文件
```

---

### 📁 `openclaw-mcp-bridge/` - MCP桥接（共用）
**作用**：提供MCP协议的Python实现，供各插件使用

**特点**：
- 纯Python实现，跨平台
- 定义标准Tool接口
- 处理通信协议细节
- 各插件通过pip安装使用

**结构**：
```
openclaw-mcp-bridge/
├── openclaw/              # 主包
│   ├── __init__.py
│   ├── server.py         # MCP服务器
│   ├── tools/            # 工具定义
│   └── transport/        # 传输层实现
├── setup.py              # 打包配置
└── tests/                # 单元测试
```

---

### 📁 `docs/{SubProject}/troubleshooting/` - 排错记录
**作用**：记录开发过程中遇到的已知问题、根因分析与修复方案

**格式要求**：
- 每个问题包含：现象、根因、修复、教训
- 顶部维护问题索引表（编号、问题、严重性、状态、日期）
- 修复完成后及时追加记录

**当前文件**（UE_Editor_Agent）：
- `OpenClaw 通信排错记录.md` — Gateway WebSocket 通信相关问题

---

### 📁 `docs/{SubProject}/test_reports/` - 测试报告
**作用**：阶段性测试与验证的执行记录，包含测试结果、缺陷修复说明等

---

### 📁 `docs/features/` - OpenClaw 集成功能
**作用**：OpenClaw Gateway 集成的分阶段功能 backlog（跨子项目共享）

**组织方式**：按 phase 编号，每个文件对应一个集成阶段的 feature 列表
- `openclaw-phase-0.md` ~ `openclaw-phase-4.md`

---

### 📁 `docs/skills/` - Skill 开发文档
**作用**：ArtClaw Skill 管理体系的规范与开发指南

**包含内容**：
- Skill 体系概览与贡献指南
- `manifest.json` 规范定义
- Skill 开发完整指南

## 文档查找指南

### 作为开发者
1. **查看规范**：`docs/specs/AI文档编写规范.md`
2. **开始开发**：查看对应子项目的 `features/` 目录
3. **了解设计**：查看 `specs/` 目录的架构设计文档
4. **历史决策**：查看 `decisions/` 目录的ADR记录
5. **已知问题**：查看 `troubleshooting/` 目录的排错记录

### 作为AI助手
1. **编写文档**：遵循 `docs/specs/AI文档编写规范.md`
2. **生成代码**：参考 `features/` 中的实现文档
3. **验证实现**：使用 `features/` 中的验证清单
4. **跨项目对比**：查看 `specs/跨项目共享机制.md`
5. **排错参考**：查看 `troubleshooting/` 中的历史问题与修复方案
6. **记录问题**：修复完成后及时更新 `troubleshooting/` 排错记录

### 作为项目管理者
1. **整体进度**：查看各子项目的 `specs/开发路线图.md`
2. **项目结构**：查看本文档（`项目结构说明.md`）
3. **质量检查**：使用 `AI文档编写规范.md` 的7.1检查清单
