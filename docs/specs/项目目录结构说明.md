# ArtClaw Bridge 项目结构说明

## 项目概述

**ArtClaw Bridge** 是一个桥接框架，让各类DCC软件（UE、Maya、Max等）通过MCP协议接入AI Agent平台（OpenClaw、Claude Desktop等），获得AI操作能力。

### 核心理念
- **统一协议**：所有DCC软件通过MCP与AI通信
- **插件化架构**：每个软件独立实现插件，共享核心逻辑
- **平台可替换**：Agent 平台（OpenClaw、Claude Desktop 等）通过 `platforms/` 独立管理
- **配置驱动**：路径、端口、Skill 目录等全部由 `~/.artclaw/config.json` 驱动
- **文档驱动**：详细的文档指导开发，确保质量一致

## 各目录说明

### 📁 `core/` — 共享核心模块

**作用**：平台无关的通信核心与基础设施，是**唯一源码**。

**部署方式**：
- `install.py` 安装时复制到各 DCC 插件目录（UE: `Content/Python/`，DCC: `core/`）
- 开发模式下通过 `sys.path` 回溯到项目根 `core/` 导入

**包含内容**：
- `bridge_core.py` — WebSocket RPC 通信协议核心
- `bridge_config.py` — 配置加载，`_PLATFORM_DEFAULTS` 平台默认值映射
- `bridge_diagnostics.py` — 连接诊断与环境检查
- `health_check.py` — 环境健康检查
- `integrity_check.py` — 启动时完整性检查与自动修复
- `memory_core.py` — 记忆管理系统 v2 核心（MemoryManagerV2，68KB）

---

### 📁 `platforms/` — 平台 Bridge 子项目

**作用**：各 Agent 平台的特定适配层，与 DCC 插件完全解耦。

**设计原则**：
- 每个平台一个完整子目录，包含 bridge.py + gateway 插件 + 配置模板
- 安装时选择平台（`install.py --platform openclaw`），运行时不切换
- 切换平台 = 重新运行安装器

**当前平台**：

| 平台 | 状态 | 说明 |
|------|------|------|
| `openclaw/` | ✅ 生产 | OpenClaw Gateway 全链路打通 |
| `claude/` | 🧪 POC | stdio→WebSocket 桥接器概念验证 |
| `workbuddy/` | 📋 预留 | 待调研 |

---

### 📁 `mcp_configs/` — MCP Server 配置模板

**作用**：各 DCC 对应的 MCP Server 注册配置模板（端口、URL、各平台配置格式）。

**注意**：这里只存配置模板和 README，**MCP Server 的实际代码**在各 DCC 插件内部：
- UE: `subprojects/UEDAgentProj/Plugins/UEClawBridge/Content/Python/mcp_server.py`
- DCC: `subprojects/DCCClawBridge/core/mcp_server.py`

---

### 📁 `skills/` — Skill 源码

**作用**：Skill 的项目源码，按 official/marketplace 分层，按 DCC 分目录。

**与运行时的关系**：
- 项目源码 `skills/` = **未安装**，开发用
- `~/.openclaw/skills/` = **已安装** = 运行时（扁平结构）
- `install.py --openclaw` 用 `shutil.copytree` 从源码复制到安装目录
- `skill_hub.py` 通过 `~/.artclaw/config.json` 的 `skills.installed_path` 读取路径

---

### 📁 `subprojects/` — DCC 插件子项目

每个 DCC 软件插件独立工程目录。

#### UEClawBridge（Unreal Engine）
- C++ + Python 混合开发
- C++ 负责 UI（Slate Dashboard）/ 生命周期（EditorSubsystem）/ 主线程调度
- Python 负责 MCP Server / Skill Hub / 通信桥接 / 记忆
- `Content/Python/` 下包含完整的 Python 运行时文件（安装时由 install.py 复制共享模块）

#### DCCClawBridge（Maya / 3ds Max）
- 纯 Python + PySide2/Qt 5.15
- 通用 Qt 界面 + 薄 DCC 适配层（每个 DCC 一个 adapter ~200行）
- `core/` 下包含共享模块副本（安装时由 install.py 从根 `core/` 复制）

- 主文件：`docs/UEClawBridge/troubleshooting/OpenClaw 通信排错记录.md`

## 配置文件

### `~/.artclaw/config.json` — 统一配置中心

安装时由 `install.py` 自动写入，运行时由各模块读取：

```json
{
  "platform": {
    "type": "openclaw",
    "gateway_url": "ws://127.0.0.1:18789",
    "mcp_port": 8080
  },
  "skills": {
    "installed_path": "~/.openclaw/skills",
    "disabled": [],
    "pinned": []
  },
  "mcp": {
    "config_path": "~/.openclaw/openclaw.json",
    "config_key": "mcp.servers"
  },
  "project_root": "D:\\MyProject\\artclaw_bridge"
}
```

## 安装时文件复制规则

`install.py` 安装时执行以下复制：

| 源 | 目标（UE） | 目标（Maya/Max） |
|----|-----------|-----------------|
| `core/*.py` | `Content/Python/` | `DCCClawBridge/core/` |
| `platforms/{platform}/*.py` | `Content/Python/` | `DCCClawBridge/core/` |
| `skills/official/` + `skills/marketplace/` | `~/.openclaw/skills/`（扁平） | 同左 |
| `platforms/{platform}/gateway/` | `~/.openclaw/extensions/mcp-bridge/` | 同左 |

## 文档查找指南

### 作为开发者
1. **查看规范**：`docs/specs/AI文档编写规范.md`
2. **开始开发**：查看对应子项目的 `features/` 目录
3. **了解设计**：查看 `specs/` 目录的架构设计文档
4. **历史决策**：查看 `decisions/` 目录的 ADR 记录
5. **已知问题**：查看 `troubleshooting/` 目录的排错记录

### 作为 AI 助手
1. **编写文档**：遵循 `docs/specs/AI文档编写规范.md`
2. **生成代码**：参考 `features/` 中的实现文档
3. **排错参考**：查看 `troubleshooting/` 中的历史问题与修复方案
4. **记录问题**：修复完成后及时更新 `troubleshooting/` 排错记录

### 作为项目管理者
1. **整体进度**：查看各子项目的 `specs/开发路线图.md`
2. **项目结构**：查看本文档
3. **质量检查**：使用 `AI文档编写规范.md` 的检查清单
