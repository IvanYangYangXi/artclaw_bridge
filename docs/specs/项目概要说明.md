# ArtClaw Bridge 项目概要说明

## 一、项目定位

**ArtClaw Bridge** 是一个桥接框架，让各类DCC软件（UE、Maya、Max等）通过MCP协议接入AI Agent平台（OpenClaw、Claude Desktop等），获得AI操作能力。

**当前阶段**：基于 UE + OpenClaw 完成第一套完整实现，打通整条链路，形成可复用基建。Maya/Max 通过 DCCClawBridge 共享子项目同步支持。

---

```
│  ⚠️ 这是项目价值的核心，也是长期主战场                 │
├─────────────────────────────────────────────────────┤

│  通信层、插件框架、多平台管理                          │
│  ✅ 框架目标：Skill开发者只需关注DCC API，其余透明      │
└─────────────────────────────────────────────────────┘
```

## 三、项目组成

```
ArtClaw Bridge
├── 共享核心层 (core/)           平台无关的通信与基础设施
│   ├── bridge_core.py          WebSocket RPC 通信核心
│   ├── bridge_config.py        配置加载与平台默认值
│   ├── memory_core.py          记忆管理系统 v2
│   └── ...                     诊断、健康检查、完整性检查
│
├── DCC 插件层 (subprojects/)    各DCC软件本地插件
│   ├── UEClawBridge            ✅ 已完成（C++ + Python）
│   ├── DCCClawBridge           ✅ Maya/Max 共享（Python + PySide2）
│   └── 每个 DCC 内嵌 MCP Server + Skill Hub
│
├── 平台 Bridge (platforms/)     可替换的 Agent 平台适配
│   ├── openclaw/               ✅ 当前主力（Gateway 全链路打通）
│   ├── claude/                 🧪 POC（stdio→WS 桥接器）
│   └── 安装时选择，运行时不切换
│
├── MCP 配置 (mcp_configs/)      各 DCC 的 MCP Server 注册配置模板
│
├── Skill 源码 (skills/)         官方/市集/模板 Skill 源码
│   └── 安装时复制到 ~/.openclaw/skills/（运行时）
│
└── 工具 (cli/, install.bat/py)  CLI 与一键安装器
```

## 四、关键概念

| 概念 | 说明 |
|------|------|
| **Skill** | AI 可调用的 DCC 操作能力，是项目核心价值单元。v2.6 后不再注册为 MCP 工具，改为 Python API（`execute_skill()`），AI 通过 `run_python` 调用 |
| **MCP** | Model Context Protocol，AI 平台与 DCC 工具之间的标准通信协议 |
| **DCC 插件** | 运行在 DCC 软件内的本地插件，内嵌 MCP Server，提供唯一 MCP 工具（`run_python` / `run_ue_python`） |
| **Platform Bridge** | 各 Agent 平台的适配层（`platforms/` 下），与 DCC 插件解耦，安装时选择 |
| **共享核心** | `core/` 下的平台无关模块，安装时复制到各 DCC 插件目录 |

---

## 五、核心挑战

| 挑战 | 策略 |
|------|------|
| **Skill数量规模大** | 框架承担所有通信/注册复杂度；v2.6 后每 DCC 仅 1 个 MCP 工具，Skill 通过 Python API 调用 |
| **各DCC Python环境差异** | 共享核心（core/）统一源码，安装时复制到各 DCC；薄适配层隔离差异 |
| **多平台协议差异** | platforms/ 独立子项目，DCC 插件不感知平台；安装时一次性选择 |
| **DCC主线程限制** | 命令队列（UE: tick 消费 / DCC: QTimer 跨线程回调），结果异步回传 |
| **配置散落** | `~/.artclaw/config.json` 统一配置中心，所有路径配置驱动 |
