# ArtClaw文档编写规范

## 0. 项目结构与编写原则

### 0.1 项目结构
**ArtClaw** 总项目包含多个DCC软件插件：
- UEClawBridge (UE)
- Maya_Editor_Agent (未来)
- Blender_Editor_Agent (未来)

本文档适用于总项目及所有子项目。

### 0.2 编写原则
**所有AI生成的文档必须精简、无废话**：
- 使用直接明了的表述方式
- 删除冗余解释和重复说明，在保证核心内容和可操作性的前提下，最大限度减少token消耗。
- 合并相关章节，使用紧凑的表格
- **PRD/架构文档**：单文件控制在 2000 字以内。复杂业务按功能模块拆分（如 `PRD_Auth.md`, `PRD_Order.md`）。
- **实时记忆 (MEMO.md)**：保持在 50 行以内，滚动记录“当前任务”和“最近变更”，旧内容移入 `CHANGELOG.md`。
- **API 契约**：每个模块独立的 API 定义文件，严禁全量 API 堆在一个文件。
- **引用标注**：文件内引用其他文档时，标注相对路径和范围，引导 AI 关联上下文。
- **路径深度**：目录层级建议不超过 4 层，方便 AI 快速扫描文件树。

## 4. 代码规范

### 4.1 C++规范
- **头文件**：必须包含`#pragma once`或`#ifndef`防护
- **类定义**：使用正确前缀（`F`, `U`, `A`, `S`）
- **实现分离**：.h和.cpp文件分开
- **注释**：关键逻辑使用`// NOTE:`, `// TODO:`, `// FIXME:`标注
- **格式化**：遵循UE官方代码规范

### 4.2 Python规范
- **导入语句**：按标准库、第三方库、本地模块分组
- **类型注解**：函数参数和返回值必须添加类型注解
- **文档字符串**：模块、类、函数必须有docstring
- **错误处理**：使用try-except捕获异常并记录日志

### 4.3 配置文件规范
配置采用JSON格式，遵循以下原则：
- 使用有意义的键名
- 保持结构清晰，适当使用嵌套
- 在注释中说明各配置项的用途

## 5. 命名规范

### 5.1 C++命名
| 类型 | 前缀 | 示例 | 说明 |
| :--- | :--- | :--- | :--- |
| Module类 | `F` | `FUEClawBridgeModule` | 继承自IModuleInterface |
| Subsystem类 | `U` | `UUEAgentSubsystem` | 继承自UEditorSubsystem |
| Slate Widget | `S` | `SUEAgentDashboardWidget` | Slate UI类 |
| 结构体 | `F` | `FAgentConfig` | 纯数据容器 |
| 枚举 | `E` | `EConnectionStatus` | 枚举类型 |

### 5.2 Python命名
- **模块名**：`snake_case`（`ue_agent.py`）
- **函数名**：`snake_case`（`setup_python_paths()`）
- **类名**：`PascalCase`（`UELogRedirector`）
- **常量**：`UPPER_SNAKE_CASE`（`LOG_CATEGORY = "UEAgent"`）
- **装饰器**：`@ue_agent.tool`

## 8. 更新与维护

### 8.1 何时更新文档
- [ ] 实现代码的接口或行为变更
- [ ] 新增功能或配置项
- [ ] 修复了文档中描述的错误
- [ ] 发现了新的问题或解决方案
- [ ] 性能优化改变了使用方式

### 8.2 更新流程
1. **标记待更新**：在文档顶部添加 `**状态**：待更新`
2. **创建任务**：在任务管理工具中创建更新任务
3. **同步修改**：同时更新代码和文档
4. **验证更新**：按照7.1检查清单验证
5. **记录变更**：在文档底部"变更历史"中记录
6. **移除标记**：验证完成后移除待更新标记

### 8.3 CHANGELOG格式
```markdown
# UE Editor Agent 变更日志

## [1.0.0-alpha] - 2026-03-14
### 新增
- 插件骨架和模块系统
```

## 10. 常见问题

### 10.1 文档与代码不一致
1. **确认现状**：运行代码确认实际行为
2. **判断优先级**：
   - 代码正确，文档错误 → 更新文档
   - 文档正确，代码错误 → 修复代码
   - 都需改进 → 创建新任务
3. **同步更新**：保持文档和代码一致
4. **通知相关方**：在团队频道或任务管理工具中通知

### 10.2 编号冲突或重复
1. **检查路线图**：对照开发路线图确认正确编号
2. **批量重命名**：使用脚本批量修改文件名和内部引用
3. **更新链接**：检查并更新所有文档间的交叉引用
4. **记录变更**：在CHANGELOG中记录编号变更
