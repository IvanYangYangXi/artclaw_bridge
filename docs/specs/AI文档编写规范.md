# ArtClaw 项目文档编写规范

## 0. 项目概述

### 0.1 项目结构
**ArtClaw** 是总项目，包含多个针对不同DCC（数字内容创作）软件的编辑器插件子项目：

- **UE_Editor_Agent**：Unreal Engine 编辑器插件
- **Maya_Editor_Agent**（未来）：Autodesk Maya 插件
- **Blender_Editor_Agent**（未来）：Blender 插件
- ...（其他DCC软件）

### 0.2 规范适用范围
本文档规范适用于 **ArtClaw总项目** 及所有子项目的文档编写，确保跨项目的一致性和可维护性。

### 1.2 文档分类

#### 1.2.1 通用规范文档（docs/specs/）
- **跨项目共享**：所有子项目共同遵守的规范
- **内容**：文档编写规范、代码规范、Git工作流、发布流程等
- **维护者**：项目负责人
- **更新频率**：低（仅在规范变更时更新）

#### 1.2.2 子项目专属规范（docs/{SubProject}/specs/）
- **单项目专用**：针对特定DCC软件的特有规范
- **内容**：软件专属架构设计、API设计、性能优化策略等
- **示例**：UE的Slate UI规范、Maya的MEL/Python混合编程规范
- **维护者**：子项目负责人

#### 1.2.3 实现文档（docs/{SubProject}/features/）
- **详细步骤**：按开发阶段组织的实现指南
- **内容**：代码示例、配置说明、验证清单
- **特点**：AI助手生成，开发者执行
- **组织方式**：按开发阶段（阶段 X）和功能模块分组

#### 1.2.4 设计决策记录（docs/{SubProject}/decisions/）
- **ADR格式**：Architecture Decision Record
- **命名**：`YYYYMMDD-决策标题.md`
- **内容**：背景、决策、后果、替代方案

## 4. 代码示例规范

### 4.1 C++代码规范
- **头文件**：必须包含`#pragma once`或`#ifndef`防护
- **类定义**：使用正确前缀（`F`, `U`, `A`, `S`）
- **实现分离**：.h和.cpp文件分开，避免头文件包含实现
- **注释**：关键逻辑使用`// NOTE:`, `// TODO:`, `// FIXME:`标注
- **格式化**：遵循UE官方代码规范

```cpp
// 头文件示例
#pragma once

#include "CoreMinimal.h"
#include "EditorSubsystem.h"
#include "UEAgentSubsystem.generated.h"

// NOTE: 编辑器子系统，全局单例
UCLASS()
class UEEDITORAGENT_API UUEAgentSubsystem : public UEditorSubsystem
{
    GENERATED_BODY()

public:
    virtual void Initialize(FSubsystemCollectionBase& Collection) override;
    virtual void Deinitialize() override;

private:
    // TODO: 添加线程安全机制
    bool bIsConnected;
};
```

### 4.2 Python代码规范
- **导入语句**：按标准库、第三方库、本地模块分组
- **类型注解**：函数参数和返回值必须添加类型注解
- **文档字符串**：模块、类、函数必须有docstring
- **错误处理**：使用try-except捕获异常并记录日志

```python
# 导入分组
import sys
import asyncio
from pathlib import Path
from typing import Optional, Dict, Any

import unreal

# 本地导入
from .logger import ue_log

# 函数类型注解
def setup_python_paths(plugin_root: Path) -> bool:
    """设置Python路径，确保依赖可以正确导入
    
    Args:
        plugin_root: 插件根目录路径
        
    Returns:
        成功返回True，失败返回False
        
    Raises:
        FileNotFoundError: 当插件根目录不存在时
    """
    try:
        lib_path = plugin_root / "Content" / "Python" / "Lib"
        if not lib_path.exists():
            raise FileNotFoundError(f"路径不存在: {lib_path}")
            
        sys.path.insert(0, str(lib_path))
        return True
        
    except Exception as e:
        ue_log.error(f"设置Python路径失败: {e}")
        return False
```

### 4.3 配置文件规范
- **JSON格式**：使用双引号，格式化显示
- **路径**：使用正斜杠`/`或双反斜杠`\\`
- **注释**：使用`//`或`/* */`（如果格式支持）

```json
{
  "plugin": {
    "name": "UEEditorAgent",
    "version": "1.0.0",
    // Python依赖配置

      "required_packages": ["websockets", "pydantic"],

    }
  }
```

## 6. 文档一致性要求

### 6.1 路径引用规范


#### 6.1.3 路径变量
使用占位符表示可变的部分：
```markdown

```

### 6.2 类名和函数名一致性
#### 6.2.1 C++命名
| 类型 | 前缀 | 示例 | 说明 |
| :--- | :--- | :--- | :--- |
| Module类 | `F` | `FUEEditorAgentModule` | 继承自IModuleInterface |
| Subsystem类 | `U` | `UUEAgentSubsystem` | 继承自UEditorSubsystem |
| Slate Widget | `S` | `SUEAgentDashboardWidget` | Slate UI类 |
| 结构体 | `F` | `FAgentConfig` | 纯数据容器 |
| 枚举 | `E` | `EConnectionStatus` | 枚举类型 |

#### 6.2.2 Python命名
- **模块名**：小写，使用下划线（`ue_agent.py`）
- **函数名**：snake_case（`setup_python_paths()`）
- **类名**：PascalCase（`UELogRedirector`）
- **常量**：UPPER_SNAKE_CASE（`LOG_CATEGORY = "UEAgent"`）
- **装饰器**：`@ue_agent.tool`保持一致

## 10. 常见问题处理

### 10.1 文档与代码不一致
**问题**：文档描述的功能与实际代码不符

**处理流程**：
1. **确认现状**：运行代码确认实际行为
2. **判断优先级**：
   - 如果代码正确，文档错误 → 更新文档
   - 如果文档正确，代码错误 → 修复代码
   - 如果都需改进 → 创建新任务
3. **同步更新**：保持文档和代码一致
4. **通知相关方**：在团队群或任务管理工具中通知

### 10.2 编号冲突或重复
**问题**：features目录中出现重复编号或编号跳跃

**处理流程**：
1. **检查路线图**：对照开发路线图确认正确编号
2. **批量重命名**：使用脚本批量修改文件名和内部引用
3. **更新链接**：检查并更新所有文档间的交叉引用
4. **记录变更**：在CHANGELOG中记录编号变更

### 10.3 文档缺失或过时
**问题**：新功能没有文档，或文档长时间未更新

**处理流程**：
1. **评估优先级**：确定是否需要立即补充
2. **创建文档**：按照本规范创建新文档
3. **提取信息**：从代码、PR、聊天记录中提取关键信息
4. **标记版本**：明确文档适用的代码版本
5. **计划维护**：定期审查文档的时效性

### 10.4 跨项目文档同步
**问题**：通用规范更新后，如何同步到各子项目

**处理流程**：
1. **发布公告**：在团队频道发布规范更新公告
2. **创建任务**：为各子项目创建应用新规范的任务
3. **渐进式更新**：在新文档中应用新规范，旧文档逐步迁移
4. **验证一致性**：检查各子项目文档的一致性
