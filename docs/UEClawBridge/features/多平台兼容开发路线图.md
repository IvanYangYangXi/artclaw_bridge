## Phase 6：多平台基础设施

**目标**：UE 插件通用化，移除平台硬编码，支持安装时选择平台

### 6.1 移除平台硬编码 (P0)

**任务**：
- 移除代码中对 `~/.openclaw/` 路径的硬编码
- 将平台相关配置改为从 `~/.artclaw/config.json` 读取
- 统一日志输出，不区分平台

**涉及文件**：
- `openclaw_bridge.py` → 改为 `platform_bridge.py`
- MCP 配置读取逻辑
- Skill 目录读取逻辑

**验收标准**：
- 代码中不再出现 `~/.openclaw/` 硬编码
- 所有平台相关配置从统一配置文件读取

### 6.2 平台配置持久化 (P0)

**任务**：
- 创建 `~/.artclaw/config.json` 配置文件
- 定义平台配置结构
- 实现配置的读取和写入

**配置文件结构**：
```json
{
  "platform": {
    "type": "openclaw",
    "gateway_url": "ws://127.0.0.1:18789",
    "mcp_port": 8080
  },
  "project_root": "D:\\MyProject\\artclaw_bridge",
  "skills": {
    "disabled": [],
    "pinned": []
  }
}
```

**验收标准**：
- 配置文件存在且格式正确
- 重启后配置持久化

### 6.3 首次安装平台选择 (P0)

**任务**：
- 检测是否为首次安装（配置文件无 platform.type）
- 首次启动时显示平台选择对话框
- 选择后保存配置并连接

**UI 流程**：
```
首次启动
    │
    ▼
检测无平台配置
    │
    ▼
显示平台选择对话框
    │
    ├─ OpenClaw
    ├─ WorkBuddy
    └─ Claude Desktop
    │
    ▼
保存配置 → 连接平台
```

**验收标准**：
- 首次安装时弹出平台选择
- 选择后自动连接
- 非首次安装直接连接

### 6.4 设置面板扩展 (P1)

**任务**：
- 在现有设置弹窗中添加平台配置显示
- 显示当前平台名称、Gateway 地址
- 添加"测试连接"按钮

**UI 布局**：
```
设置面板
├── [语言] [发送模式] [静默模式] [Plan]
│
├── ── 平台配置 ──
│   当前平台: OpenClaw v1.5.0
│   Gateway: ws://127.0.0.1:18789
│   [测试连接]
│
├── ── Skill ──
│   已安装: 12 个 [查看] [同步]
│
└── ── MCP ──
    ue-editor-agent: 已连接
```

**验收标准**：
- 设置面板显示当前平台信息
- 测试连接按钮功能正常

### 6.5 重新选择平台 (P1)

**任务**：
- 添加"重新选择平台"入口
- 清除当前平台配置
- 重新显示平台选择对话框

**验收标准**：
- 可以切换到其他平台
- 切换后正确连接新平台

## Phase 7：Skill/MCP 查询接口

**目标**：通过 Agent 查询已安装列表，UE 插件不直接读取平台目录

### 7.1 平台侧 Skill 开发 (P0)

**任务**：在平台侧（OpenClaw）开发以下 Skill

| Skill | 功能 | 输出格式 |
|-------|------|---------|
| `artclaw-query-skills` | 查询已安装 Skill | JSON 数组 |
| `artclaw-query-mcps` | 查询已安装 MCP | JSON 数组 |
| `artclaw-sync-skills` | 同步 Skill | 操作结果 |
| `artclaw-mcp-install` | 安装 MCP Server | 操作结果 |

**artclaw-query-skills 输出格式**：
```json
{
  "success": true,
  "skills": [
    {
      "name": "ue54_artclaw_material",
      "version": "1.0.0",
      "status": "enabled",
      "source": "official"
    }
  ]
}
```

**artclaw-query-mcps 输出格式**：
```json
{
  "success": true,
  "mcp_servers": [
    {
      "id": "ue-editor-agent",
      "name": "UE Claw Bridge",
      "status": "connected",
      "port": 8080,
      "tools": ["run_ue_python"]
    }
  ]
}
```

**验收标准**：
- Skill 可以正确返回当前平台已安装的列表
- UE 插件调用后能正确解析结果

### 7.2 UE 插件查询调用 (P0)

**任务**：
- 点击"查看 Skill"时发送查询指令
- 接收并解析 Agent 返回的 JSON
- 显示 Skill 列表

**通信流程**：
```
用户点击"查看 Skill"
        │
        ▼
UE 发送消息:
"查询当前平台已安装的 Skill 列表"
        │
        ▼
Agent 执行 artclaw-query-skills
        │
        ▼
返回 JSON 结果
        │
        ▼
UE 解析并显示列表
```

**验收标准**：
- Skill 列表正确显示
- 状态（启用/禁用）与平台同步

### 7.3 Skill 同步功能 (P1)

**任务**：
- "同步官方"按钮 → 同步 official 层 Skill
- "同步市集"按钮 → 同步 marketplace 层 Skill

**实现方式**：
- UI 按钮点击后发送指令给 Agent
- Agent 执行 `artclaw-sync-skills`

**验收标准**：
- 同步功能正常工作
- Skill 状态正确更新

## Phase 8：MCP Server 源码管理

### 8.1 mcp_servers 目录结构 (P0)

**任务**：
- 创建 `mcp_servers/` 源码目录
- 定义 manifest.json 格式

**目录结构**：
```
mcp_servers/
├── official/
│   ├── ue-editor-agent/
│   │   ├── manifest.json
│   │   ├── config_template.json
│   │   └── README.md
│   └── maya-editor-agent/
├── marketplace/
│   └── custom-mcp/
└── templates/
```

**manifest.json 格式**：
```json
{
  "id": "ue-editor-agent",
  "name": "UE Claw Bridge",
  "version": "1.0.0",
  "dcc": "unreal_engine",
  "description": "Unreal Engine Editor MCP Server",
  "port_range": [8080, 8085],
  "platforms": {
    "openclaw": {
      "config_key": "mcp.servers.ue-editor-agent"
    },
    "workbuddy": {
      "config_key": "mcpServers.ue-editor-agent"
    }
  }
}
```

**验收标准**：
- 目录结构正确
- manifest.json 格式正确

### 8.2 MCP 安装 Skill (P1)
