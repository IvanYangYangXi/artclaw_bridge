## Phase 6：多平台基础设施

**目标**：UE/DCC 插件通用化，目录重组，支持安装时选择平台

### 6.1 移除平台硬编码 (P0) ✅ 已完成

- bridge_config.py 新增平台配置层
- 所有路径改为配置驱动 + fallback
- 三端同步

### 6.2 平台配置持久化 (P0) ✅ 已完成

- `~/.artclaw/config.json` 新增 platform / skills / mcp 配置节

### 6.3 目录重组 (P0) ✅ 已完成

- `openclaw-mcp-bridge/` 拆分为 `core/` + `platforms/openclaw/`
- 所有路径引用全量更新
- install.bat/install.py 使用新路径

### 6.4 install.bat 平台选择 (P0) ✅ 已完成

- install.bat 新增平台选择交互（openclaw/workbuddy/claude）
- install.py 新增 `--platform` CLI 参数
- 根据平台参数决定：复制哪个 bridge.py、写入哪个平台配置、Skills 安装到哪个目录
- `~/.artclaw/config.json` 自动写入 platform/skills/mcp 配置节
- 不指定 `--platform` 时默认 openclaw

## Phase 7：Skill/MCP 列表展示 ✅ 已完成

### 7.1 Skill 列表扫描 (P0) ✅ 已完成

- 读取 `config.skills.installed_path` 扫描 Skill 目录
- 同时支持 manifest.json 和纯 SKILL.md 两种格式

### 7.2 MCP Server 列表读取 (P0) ✅ 已完成

- MCP Tab 改为配置驱动：从 `config.mcp.config_key` 解析 JSON 路径
- `TraverseJsonPath` 通用 dot-separated 路径遍历
- `SetServerEnabled`/`WriteNewServer` 使用缓存的 ConfigKey

### 7.3 UI 列表展示 (P0) ✅ 已完成

- Skill Tab: 完整列表 + 筛选 + 安装状态
- MCP Tab: 列表 + 连接状态探测 + 启用/禁用

### 7.4 Skill 同步功能 (P1) ✅ 已完成

- "同步"按钮触发 Skill 同步（复用 skill_sync API）
- 同步完成后刷新列表

---

## Phase 8：MCP Server 配置模板整理 ✅ 已完成 → 已删除

- 原 `mcp_servers/official/` 整理了配置模板
- 后改名为 `mcp_configs/`
- 最终评估无代码消费者，已删除（配置由 `install.py` 的 `PLATFORM_CONFIGS` + `platforms/*/config/` 管理）
