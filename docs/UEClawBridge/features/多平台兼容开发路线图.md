## Phase 6：多平台基础设施

**目标**：UE/DCC 插件通用化，目录重组，支持安装时选择平台

### 6.1 移除平台硬编码 (P0) ✅ 已完成

- bridge_config.py 新增平台配置层
- 所有路径改为配置驱动 + fallback
- 三端同步

### 6.2 平台配置持久化 (P0) ✅ 已完成

- `~/.artclaw/config.json` 新增 platform / skills / mcp 配置节

### 6.3 目录重组 (P0)

**任务**：
- 创建 `core/` 目录，将共享核心从 `openclaw-mcp-bridge/` 提取出来
- 将 `openclaw-mcp-bridge/` 重命名/迁移到 `platforms/openclaw/`
- OpenClaw 特定文件留在 `platforms/openclaw/`，共享文件移到 `core/`
- 更新 install.bat 的复制路径

**迁移映射**：
```
openclaw-mcp-bridge/bridge_core.py       → core/bridge_core.py
openclaw-mcp-bridge/bridge_config.py     → core/bridge_config.py
openclaw-mcp-bridge/bridge_diagnostics.py→ core/bridge_diagnostics.py
openclaw-mcp-bridge/health_check.py      → core/health_check.py
openclaw-mcp-bridge/integrity_check.py   → core/integrity_check.py
openclaw-mcp-bridge/memory_core.py       → core/memory_core.py

openclaw-mcp-bridge/openclaw_bridge.py   → platforms/openclaw/openclaw_bridge.py
openclaw-mcp-bridge/setup.bat            → platforms/openclaw/setup.bat
openclaw-mcp-bridge/setup_openclaw_config.py → platforms/openclaw/setup_openclaw_config.py
openclaw-mcp-bridge/memory_manager.py    → platforms/openclaw/memory_manager.py
openclaw-mcp-bridge/__init__.py          → platforms/openclaw/__init__.py
openclaw-mcp-bridge/README.md            → platforms/openclaw/README.md
openclaw-mcp-bridge/mcp-bridge/          → platforms/openclaw/gateway/
openclaw-mcp-bridge/templates/           → platforms/openclaw/config/
openclaw-mcp-bridge/openclaw-config-snippet.json → platforms/openclaw/config/
```

**验收标准**：
- `openclaw-mcp-bridge/` 目录删除
- `core/` 和 `platforms/openclaw/` 结构正确
- install.bat 使用新路径正常工作
- 三端部署正常（core/ 复制到 UE/DCC，platform bridge 复制到 UE/DCC）

### 6.4 install.bat 平台选择 (P0)

**依赖**: 6.3

**任务**：
- install.bat 新增 `--platform` 参数（默认 openclaw）
- 根据平台参数决定：
  - 复制哪个 `platforms/{platform}/` 的 bridge.py
  - 写入哪个平台的默认路径到 `~/.artclaw/config.json`
  - 部署到哪个平台的 Skills/MCP 目录
- install.py 同步支持 `--platform` 参数

**验收标准**：
- `install.bat --platform openclaw` 正常安装
- config.json 正确写入平台信息
- 不指定 `--platform` 时默认 openclaw

## Phase 7：Skill/MCP 列表展示

**目标**：已安装 Skill/MCP 列表通过配置路径直接扫描获取并展示

### 7.1 Skill 列表扫描 (P0)

**依赖**: 6.2

**任务**：
- 读取 `config.skills.installed_path` 扫描 Skill 目录
- 同时支持 manifest.json 和纯 SKILL.md 两种格式

**验收标准**：
- 正确扫描并返回已安装 Skill 列表

### 7.2 MCP Server 列表读取 (P0)

**依赖**: 6.2

**任务**：
- 读取 `config.mcp.config_path` + `config.mcp.config_key`
- 解析 MCP Server 配置并返回列表

**验收标准**：
- 正确读取平台配置文件中的 MCP Server 列表

### 7.3 UI 列表展示 (P0)

**依赖**: 7.1, 7.2

**任务**：
- 设置面板中显示 Skill 数量和列表查看入口
- 设置面板中显示 MCP Server 状态

**验收标准**：
- Skill 列表正确显示
- MCP Server 列表正确显示

### 7.4 Skill 同步功能 (P1)

**依赖**: 7.1

**任务**：
- "同步"按钮触发 Skill 同步（复用 skill_sync API）
- 同步完成后刷新列表
