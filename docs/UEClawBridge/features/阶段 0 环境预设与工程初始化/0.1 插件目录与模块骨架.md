# 阶段 0.1：插件目录与模块骨架 (Detailed Setup)

## 1. 物理目录结构创建
请在你的 UE 项目根目录下手动创建以下结构（推荐使用 IDE 或文件管理器）：

```text
YourProject/
└── Plugins/
    └── UEClawBridge/
        ├── UEClawBridge.uplugin         # 插件描述文件
        ├── Resources/
        │   └── Icon128.png               # 插件图标 (128x128)
        ├── Source/
        │   ├── UEClawBridge/
        │   │   ├── UEClawBridge.Build.cs # C++ 编译配置
        │   │   ├── Public/               # 头文件 (.h)
        │   │   └── Private/              # 实现文件 (.cpp)
        └── Content/
            └── Python/
                ├── init_unreal.py        # Python 入口脚本
                └── requirements.txt      # 依赖定义
```

## 3. C++ 编译配置 (UEClawBridge.Build.cs)
**核心细节**：需要包含 `PythonScriptPlugin` 相关的模块，以便 C++ 能与 Python 通信（不同UE版本可能不同）。

```csharp
using UnrealBuildTool;

public class UEClawBridge : ModuleRules
{
	public UEClawBridge(ReadOnlyTargetRules Target) : base(Target)
	{
		PCHUsage = ModuleRules.PCHUsageMode.UseExplicitOrSharedPCHs;

		PublicDependencyModuleNames.AddRange(new string[] { 
			"Core", 
			"CoreUObject", 
			"Engine", 
			"InputCore", 
			"EditorSubsystem", // 阶段 0.2 所需
			"UnrealEd", 
			"PythonScriptPlugin" // 必须包含
		});

		PrivateDependencyModuleNames.AddRange(new string[] {
			"Slate",
			"SlateCore",
			"ToolMenus",      // 阶段 0.3 菜单注册所需
			"Projects"
		});
	}
}
```

## 4. 模块入口定义 (UEClawBridge.h / .cpp)
这是插件在 C++ 层的“心脏”，负责拉起 Python 脚本。

### UEClawBridge.h
```cpp
#pragma once
#include "CoreMinimal.h"
#include "Modules/ModuleManager.h"

class FUEClawBridgeModule : public IModuleInterface
{
public:
	virtual void StartupModule() override;
	virtual void ShutdownModule() override;
};
```

### UEClawBridge.cpp
```cpp
#include "UEClawBridge.h"
#include "Modules/ModuleManager.h"

void FUEClawBridgeModule::StartupModule()
{
	// 模块启动时，UE 会自动扫描 Content/Python 目录
	// 具体的 Python 初始化逻辑将由 init_unreal.py 承载
}

void FUEClawBridgeModule::ShutdownModule() {}

IMPLEMENT_MODULE(FUEClawBridgeModule, UEClawBridge)
```

## 5. 关键细节确认 (Checklist)
1. **模块命名**：文件夹名、`.uplugin` 内的模块名、`.Build.cs` 文件名必须保持一致（均为 `UEClawBridge`）。
2. **图标路径**：如果你没有 `Icon128.png`，UE 会显示默认图标，不影响功能，但建议准备一个。
3. **编译**：完成上述文件创建后，请**右键项目 .uproject -> Generate Visual Studio project files**，然后在 IDE 中进行编译。

**🚩 里程碑**：
1. 成功生成 VS 工程并完成编译。
2. 启动 UE 编辑器，在 **Plugins** 窗口中能找到已启用的 `UE Editor Agent`。

---


关键细节补充：
在 UE 5.x 中，务必在 Project Settings -> Plugins -> Python Script Plugin 中勾选 "Enable Remote Execution"。虽然我们使用 MCP，但开启此选项能极大方便开发阶段的远程调试。
