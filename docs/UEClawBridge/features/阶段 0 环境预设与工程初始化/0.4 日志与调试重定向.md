# 阶段 0.4：日志与调试重定向系统 (Logging System)

**目标**：构建多级日志系统，将Python层的所有输出（AI指令、MCP通信、错误堆栈）实时重定向到UE Output Log，支持按级别过滤和分类查看。

**🚩 里程碑**：
1. Python的`print()`输出实时显示在UE Output Log中
2. 支持DEBUG/INFO/WARNING/ERROR四级日志分类
3. MCP通信日志单独分类，便于调试协议交互
4. 完整的错误堆栈以红色高亮显示
5. 可在Output Log过滤器中单独查看Agent日志

---

### 1. 完整验证清单

#### 1.1 基础功能验证
- [ ] 在Python中执行`print("test")`，UE Output Log显示`[INFO] test`
- [ ] 使用`UELogger.debug()`，日志以VERBOSE级别显示
- [ ] 使用`UELogger.error()`，日志以ERROR级别显示（红色）
- [ ] 在Output Log过滤器中勾选`LogUEAgent`，只显示Agent相关日志
- [ ] 在Output Log过滤器中勾选`LogUEAgent_MCP`，只显示MCP通信日志

#### 1.2 异常处理验证
- [ ] Python代码发生异常时，完整堆栈显示在Output Log
- [ ] 堆栈信息包含文件名、行号、函数名
- [ ] 错误信息以红色高亮显示
- [ ] 未捕获的异常也能被正确记录

#### 1.3 MCP通信验证
- [ ] MCP请求和响应都记录到`LogUEAgent_MCP`类别
- [ ] 日志包含完整的JSON数据（调试用）
- [ ] MCP错误单独记录到`LogUEAgent_Error`
- [ ] 使用`@log_mcp_call`装饰器能自动记录调用

#### 1.4 性能验证
- [ ] 高频日志输出（1000+条/秒）不卡顿
- [ ] 大量日志不会导致内存泄漏
- [ ] 日志系统初始化耗时<100ms
- [ ] 日志重定向不影响Python代码执行速度
