## 阶段 0.1-1：OpenClaw 环境集成与依赖管理 (OpenClaw & Dependencies)
**目标**：安装并配置 OpenClaw 客户端，确保其能识别并连接到 UE 插件暴露的 MCP 接口。

## 3. OpenClaw 配置与操作步骤 (OpenClaw Side)

在阶段 1 的网关逻辑编写完成前，你需要先确保 OpenClaw 能够识别该节点。

### 3.1 详细操作步骤
1. **启动项目**：将上述文件放好后，打开 UE 项目。
2. **确认安装**：检查插件目录是否生成了 `site-packages` 文件夹。
3. **启用远程执行**：前往 `Project Settings -> Plugins -> Python Script Plugin`，勾选 **Enable Remote Execution**。
4. **安装 OpenClaw**：
   - 访问 [OpenClaw 官网](https://openclaw.io) 下载客户端。
   - 在 OpenClaw 设置中找到 `mcp-servers` 配置。

### 3.2 OpenClaw MCP 配置模板
OpenClaw MCP Bridge 插件 — 完整部署文档：./openclaw-mcp-bridge/README.md

- [ ] **4. 编写依赖自动安装脚本**
    - **操作**：在 `init_unreal.py` 中编写逻辑，检查插件目录是否已存在 `site-packages`。
    - **细节设置**：若缺失依赖，自动调用 `subprocess.run([py_path, "-m", "pip", "install", "-r", ...])` 定向安装到插件内部目录。

**🚩 里程碑**：
1. OpenClaw 客户端能够正常启动并进入等待连接状态。
2. UE 插件启动时，自动在插件目录下生成 `site-packages` 文件夹并装载所需库。
3. Python `sys.path` 已正确包含插件私有依赖路径，执行 `import websockets` 不报错。
