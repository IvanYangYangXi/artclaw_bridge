# 阶段 0.2：UUEAgentSubsystem 详细实现步骤与参数配置

本阶段的目标是建立插件的“核心状态真值源”。通过继承 `UEditorSubsystem`，我们实现了一个随编辑器启动自动创建、随关闭自动销毁的全局单例。它将作为 C++ UI 层与 Python 通信层之间的状态同步桥梁。

---

### 1. 详细实现步骤

#### 第一步：创建 C++ 类 (Header)
在 `Source/UEAgent/Public/UEAgentSubsystem.h` 中定义类。

```cpp
#pragma once

#include "CoreMinimal.h"
#include "EditorSubsystem.h"
#include "UEAgentSubsystem.generated.h"

// 声明动态多播委托：当连接状态改变时通知 UI 刷新颜色
DECLARE_DYNAMIC_MULTICAST_DELEGATE_OneParam(FOnAgentConnectionStatusChanged, bool, bNewStatus);

/**
 * UUEAgentSubsystem
 * 全局单例，负责协调 OpenClaw 与 UE 编辑器的连接状态
 */
UCLASS(BlueprintType)
class UEAGENT_API UUEAgentSubsystem : public UEditorSubsystem
{
    GENERATED_BODY()

public:
    // --- 框架钩子 ---
    virtual void Initialize(FSubsystemCollectionBase& Collection) override;
    virtual void Deinitialize() override;

    // --- 核心接口 ---

    /** 供 Python 网关调用：更新连接状态 */
    UFUNCTION(BlueprintCallable, Category = "UEAgent")
    void SetConnectionStatus(bool bInIsConnected);

    /** 供 UI 绑定使用：查询当前是否在线 */
    UFUNCTION(BlueprintPure, Category = "UEAgent")
    bool GetConnectionStatus() const { return bIsConnected; }

    // --- 暴露属性 ---

    /** 连接状态真值 */
    UPROPERTY(BlueprintReadOnly, Category = "UEAgent")
    bool bIsConnected = false;

    /** 状态变更委托：UI 层将绑定此事件以实现图标变色 */
    UPROPERTY(BlueprintAssignable, Category = "UEAgent")
    FOnAgentConnectionStatusChanged OnConnectionStatusChanged;
};
```

#### 第二步：实现逻辑 (CPP)
在 `Source/UEAgent/Private/UEAgentSubsystem.cpp` 中编写状态切换逻辑。

```cpp
#include "UEAgentSubsystem.h"

void UUEAgentSubsystem::Initialize(FSubsystemCollectionBase& Collection)
{
    Super::Initialize(Collection);
    bIsConnected = false; // 初始化为断开状态
    UE_LOG(LogTemp, Log, TEXT("UEAgent: Subsystem Initialized."));
}

void UUEAgentSubsystem::Deinitialize()
{
    UE_LOG(LogTemp, Log, TEXT("UEAgent: Subsystem Deinitializing."));
    Super::Deinitialize();
}

void UUEAgentSubsystem::SetConnectionStatus(bool bInIsConnected)
{
    if (bIsConnected != bInIsConnected)
    {
        bIsConnected = bInIsConnected;
        
        // 广播状态变更通知，驱动图标变色
        OnConnectionStatusChanged.Broadcast(bIsConnected);
        
        UE_LOG(LogTemp, Warning, TEXT("UEAgent: Connection Status Updated -> %s"), 
            bIsConnected ? TEXT("CONNECTED") : TEXT("DISCONNECTED"));
    }
}
```

### 2. 核心参数配置表 (Parameter Config)


| 配置项 | 推荐设置 | 说明 |
| :--- | :--- | :--- |
| **Module Type** | `Editor` | 必须在 .uplugin 中设置为 Editor 类型，确保不在发布版游戏中运行。 |
| **BlueprintType** | `true` | 必须标记，否则 Python 的 `unreal` 模块无法识别此 C++ 类。 |
| **Setter 函数** | `BlueprintCallable` | 必须标记，否则 Python 脚本无法跨界调用此函数。 |
| **连接真值** | `BlueprintReadOnly` | 保护数据，确保 Python 必须通过 `SetConnectionStatus` 接口来修改状态。 |

### 3. Python 端的交互示例

当你的 OpenClaw 通信网关接收到连接状态变化时，通过以下代码同步给 C++：

在你的插件目录下创建 Content/Python 文件夹（这是 UE 自动识别的路径）。
文件名：init_unreal.py (推荐) 


```python
import unreal

def sync_connection_state(is_online: bool):
    # 获取 C++ 子系统的单例对象
    subsystem = unreal.get_editor_subsystem(unreal.UEAgentSubsystem)
    
    if subsystem:
        # 调用 C++ 接口，触发状态更新和 UI 刷新委托
        subsystem.set_connection_status(is_online)
        unreal.log(f"Agent Status Synced: {is_online}")

# 测试：模拟连接成功
sync_connection_state(True)
```
