# 剪贴板图片与文件发送功能设计

## 概述

在 UE 聊天面板中支持从剪贴板粘贴图片或文件路径，作为附件随消息发送给 AI。

## 用户交互

### 图片粘贴 (Ctrl+V)
1. 用户复制截图/图片到剪贴板
2. 在聊天输入框按 Ctrl+V
3. 输入框上方出现**附件预览栏**：缩略图 + 文件名 + 删除按钮
4. 用户可选填文字说明，点击发送
5. 图片以 base64 编码随消息发送

### 文件粘贴/拖入
1. 用户复制文件路径（或从资源管理器复制文件）
2. 在输入框按 Ctrl+V
3. 检测剪贴板内容类型：
   - **图片数据**: 直接读取 bitmap，编码为 PNG/JPEG base64
   - **文件路径**: 显示为附件（图片文件做缩略图预览，其他文件显示图标+文件名）
   - **普通文本**: 正常粘贴到输入框（不拦截）

### 附件按钮（备选）
- 输入框旁添加 📎 按钮，点击打开文件选择器

## 技术方案

### 1. C++ 层 (UEAgentDashboard)

#### 数据模型
```cpp
struct FPendingAttachment
{
    FString FilePath;      // 本地文件路径（图片/文件）
    FString MimeType;      // "image/png", "image/jpeg", "application/octet-stream"
    FString DisplayName;   // 显示名称
    TArray<uint8> Data;    // 文件二进制数据（图片直接从剪贴板读取时使用）
    bool bIsImage = false; // 是否为图片类型
};
```

#### 键盘事件拦截
在 `OnInputKeyDown` 中检测 Ctrl+V：
- 调用 `FPlatformApplicationMisc::ClipboardPaste()` 获取文本
- 调用 `FSlateApplication::Get().GetPlatformApplication()->GetClipboardImage()` 尝试获取图片
- 优先检测图片 → 文件路径 → 普通文本

**注意**: UE5 的 `FGenericPlatformApplicationMisc` 没有直接获取剪贴板图片的 API。
需要使用 Windows API `GetClipboardData(CF_DIBV5)` 或 `CF_BITMAP` 读取图片数据。

#### 附件预览 UI
在输入框上方（工具栏和输入框之间）显示附件预览栏：
```
┌─────────────────────────────────────────┐
│ [🖼 screenshot.png 245KB] [✕]          │  ← 附件预览栏
├─────────────────────────────────────────┤
│ 请分析这张截图中的 UI 布局...           │  ← 输入框
└─────────────────────────────────────────┘
```

#### 发送流程修改
`OnSendClicked()`:
1. 收集 `PendingAttachments` 列表
2. 每个附件：读取文件/数据 → base64 编码
3. 生成 JSON 附件数组 `[{mimeType, content, name}]`
4. 写入独立附件文件 `_openclaw_attachments.json`
5. 调用 `PlatformBridge->SendMessageWithAttachments(Message, Attachments, ResponseFile)`
6. 清空 `PendingAttachments`

### 2. IAgentPlatformBridge 接口扩展

```cpp
virtual void SendMessageWithAttachments(
    const FString& Message,
    const FString& AttachmentsFile,  // JSON 文件路径
    const FString& ResponseFile) = 0;
```

### 3. Python 层 (openclaw_chat.py)

新增 `send_chat_with_attachments_async(msg_file, attachments_file, response_file)`:
1. 读取 msg_file（文本消息）
2. 读取 attachments_file（JSON 数组）
3. 构建带 attachments 的 chat.send params

### 4. WebSocket 层 (openclaw_ws.py)

`do_chat()` 新增 `attachments` 参数：
```python
await ws.send(json.dumps({
    "type": "req", "id": req_id, "method": "chat.send",
    "params": {
        "sessionKey": session_key,
        "message": message,
        "attachments": [
            {"mimeType": "image/png", "content": "<base64>"},
        ],
        "idempotencyKey": str(uuid.uuid4()),
    },
}))
```

## 附件类型处理

| 来源 | 检测方式 | 处理方式 |
|------|----------|----------|
| 剪贴板图片 | Win32 CF_DIBV5/CF_BITMAP | 转 PNG → base64 |
| 剪贴板文件路径 | 检测路径格式 + 文件存在 | 读取文件 → base64 |
| 剪贴板文本中的图片路径 | 正则匹配 .png/.jpg/.jpeg/.bmp | 读取文件 → base64 |
| 非图片文件路径 | 路径存在但非图片 | 发送路径文本给 AI（AI 通过工具读取）|

## 文件大小限制
- 图片: 最大 10MB（base64 后约 13MB）
- 超限提示用户

## 实现顺序

1. **Phase A**: C++ 剪贴板图片检测 + Win32 API 读取 + 附件预览 UI
2. **Phase B**: Python/WebSocket 层传递附件
3. **Phase C**: C++ 发送流程集成 + 端到端测试
4. **Phase D**: 文件路径支持 + 附件按钮
