# 对话框重构 — 消息筛选与工具调用展示

> 版本: v1.0 | 日期: 2026-03-28 | 状态: 设计完成

## 问题描述

当前 UE Dashboard 的对话显示存在以下问题：

1. **消息丢失**: 当前会话中 AI 的中间消息（tool 调用、多轮思考）未被接收和展示，用户只能看到最终回复
2. **缺少工具调用信息**: AI 使用 MCP 工具时（如 `run_ue_python`、`read`、`exec`），用户看不到工具调用过程
3. **跨 session 干扰**: bridge_core.py 的事件处理没有严格按 session key 过滤，可能接收到其他 session（如 cron、其他 DCC）的消息

## 目标

1. 当前会话的**所有内容**都被接收并展示（包括 tool 调用信息）
2. 其他会话的内容**不被接收**
3. 对话框增加 **tool 调用信息展示**（可折叠）

## 架构设计

### 消息流向 (当前)

```
User Input → Python Bridge → Gateway → AI → Gateway Events → Python Bridge → Stream File → C++ Poll → UI
```

### 事件过滤 (改进)

**当前问题**: `_handle_chat_event` 和 `_handle_agent_event` 仅通过 `_active_run_id` 过滤，不过滤 session key。

**改进**: 在事件处理中增加 session key 过滤，确保只处理当前 session 的事件。

### 工具调用事件

OpenClaw Gateway 通过 chat event 推送工具调用信息，格式：

```json
{
  "event": "chat",
  "payload": {
    "state": "delta",
    "sessionKey": "qi/ue-editor:1774629649110",
    "runId": "xxx",
    "message": {
      "content": [
        {
          "type": "toolCall",
          "id": "read:0",
          "name": "read",
          "arguments": {"file_path": "..."}
        }
      ]
    }
  }
}
```

工具调用结果：

```json
{
  "event": "chat",
  "payload": {
    "state": "delta",
    "message": {
      "content": [
        {
          "type": "toolResult",
          "toolCallId": "read:0",
          "toolName": "read",
          "content": [{"type": "text", "text": "..."}],
          "isError": false
        }
      ]
    }
  }
}
```

## 实现方案

### Phase 1: Python — 事件过滤增强 + 工具调用事件传递

#### bridge_core.py 修改

1. **Session Key 过滤**: 在 `_handle_chat_event` 和 `_handle_agent_event` 中增加 session key 比对
2. **工具调用回调**: 新增 `on_tool_call` 和 `on_tool_result` 回调
3. **事件类型扩展**: `_write_stream_event` 增加 `tool_call` 和 `tool_result` 类型

```python
# 新增回调
self.on_tool_call: Optional[Callable[[str, str, dict], None]] = None  # (tool_name, tool_id, arguments)
self.on_tool_result: Optional[Callable[[str, str, str, bool], None]] = None  # (tool_name, tool_id, content, is_error)
```

#### openclaw_bridge.py 修改

`_write_stream_event` 增加写入 tool 事件到 stream file：

```json
{"type": "tool_call", "tool_name": "read", "tool_id": "read:0", "arguments": {"file_path": "..."}}
{"type": "tool_result", "tool_name": "read", "tool_id": "read:0", "content": "...", "is_error": false}
```

### Phase 2: C++ — 流式文件解析 + UI 展示

#### UEAgentDashboard.cpp 修改

1. **流式文件解析**: 在 `SendToOpenClaw` 的轮询 lambda 中，解析 `tool_call` 和 `tool_result` 事件
2. **消息模型扩展**: `FChatMessage` 增加工具调用字段

```cpp
struct FChatMessage
{
    FString Sender;      // "user", "assistant", "system", "tool"
    FString Content;
    FDateTime Timestamp;
    bool bIsCode = false;
    
    // 工具调用信息 (Sender == "tool" 时使用)
    FString ToolName;       // 工具名称
    FString ToolId;         // 工具调用 ID
    FString ToolArguments;  // 参数 JSON (展开显示)
    FString ToolResult;     // 结果内容
    bool bToolError = false; // 是否报错
    bool bToolCollapsed = true; // 默认折叠
};
```

3. **UI 渲染**: `RebuildMessageList` 中为 `Sender == "tool"` 的消息渲染折叠式工具调用卡片：

```
[🔧 read] file_path: "..."           [▶ 展开]
```

展开后：

```
[🔧 read] file_path: "..."           [▼ 折叠]
├── 参数: {"file_path": "..."}
└── 结果: (200 行文本) ...前100字符...
```

4. **颜色方案**:
   - 工具名称: 橙色 (FLinearColor(1.0f, 0.6f, 0.2f))
   - 工具结果成功: 暗绿 (FLinearColor(0.3f, 0.6f, 0.3f))
   - 工具结果失败: 红色 (FLinearColor(0.9f, 0.3f, 0.3f))

### Phase 3: Session 隔离

#### bridge_core.py

在 `_handle_chat_event` 和 `_handle_agent_event` 中增加 session key 过滤：

```python
def _handle_chat_event(self, payload: dict):
    session_key = payload.get("sessionKey", "")
    
    # 如果有 session key，且不是当前 session，忽略
    if session_key and self._session_key and session_key != self._session_key:
        return
    
    # ... 其余逻辑不变
```
