## 🎯 目标与里程碑

### 核心目标
构建**主线程调度器**，确保所有UE编辑器操作都在游戏主线程执行，解决**线程安全问题**。

### 验收标准

## 📚 背景与上下文

### 为什么需要GameThread Marshalling？

UE编辑器API**只能在主线程调用**，如果在其他线程调用会导致：

```python
# ❌ 错误示例：在Python线程中直接调用UE API
import threading
import unreal

def bad_idea():
    # 这是在Python线程，不是UE主线程！
    actor = unreal.EditorLevelLibrary.spawn_actor_from_class(
        unreal.StaticMeshActor, unreal.Vector(0, 0, 0)
    )  # 💥 崩溃或断言失败！

thread = threading.Thread(target=bad_idea)
thread.start()
```

典型错误：
- `Assertion failed: IsInGameThread()`
- 随机崩溃
- 数据竞争
- 编辑器无响应

### 设计原则

1. **透明化**：调用方无需关心线程问题
2. **异步支持**：支持回调和Future模式
3. **超时控制**：防止死锁
4. **错误传播**：异常正确传递回调用方

## ❓ 常见问题

### Q1: 任务长时间不执行怎么办？
A:
1. 检查调度器是否启动：`scheduler.is_running`
2. 检查队列长度：`scheduler.get_queue_status()`
3. 确认在主循环中：`unreal.register_slate_pre_tick_callback`

### Q2: 如何取消已提交的任务？
A: Future模式支持取消：
```python
future = scheduler.run_async(...)
future.cancel()  # 取消任务
```

### Q3: 内存泄漏？
A: 确保调用`scheduler.stop()`释放资源，或使用弱引用：
```python
import weakref
scheduler_ref = weakref.ref(scheduler)
```
