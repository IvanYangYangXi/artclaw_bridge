# 3.1 Skill热加载器（统一管理）

**阶段**：阶段3 - 智能增强与Skill系统  
**重要性**：⭐⭐⭐⭐⭐ (P0)  
**依赖**：2.7 错误感知与自修复  
**预估工时**：2-3天

---

## 🎯 目标与里程碑

### 核心目标
构建**Skill热加载器**，无需重启编辑器即可加载/更新/卸载AI技能（Skill），实现插件功能的动态扩展。UE插件作为**Skill管理中心**，向所有MCP客户端（WorkBuddy、OpenClaw等）提供统一的Skill访问接口。

### 验收标准
- [ ] Skill加载时间<2秒
- [ ] 热更新不中断当前会话
- [ ] 支持版本回滚（保留最近3个版本）
- [ ] 加载失败不影响系统稳定性
- [ ] MCP接口`list_skills`可访问
- [ ] MCP接口`execute_skill`可执行
- [ ] Skill重载后自动通知所有连接的MCP客户端
- [ ] WorkBuddy/OpenClaw可实时感知Skill列表变化
- [ ] 向MCP客户端暴露Skill列表和执行接口
- [ ] 客户端无需重启即可使用新加载的Skill

## 🏗️ 架构设计

### Skill管理中心模式

```
┌─────────────────────────────────────────────────────────────┐
│                    UE Editor Agent插件                        │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐  │
│  │        Skill管理中心 (Skill Hot Loader)                 │  │
│  │                                                        │  │
│  │  - Skill加载/卸载/热重载                              │  │
│  │  - 版本管理                                           │  │
│  │  - 权限控制                                           │  │
│  │  - 向所有客户端暴露统一接口                            │  │
│  └──────────────────────┬─────────────────────────────────┘  │
│                         │                                    │
│  ┌──────────────────────▼─────────────────────────────────┐  │
│  │           MCP服务器接口层                               │  │
│  │                                                        │  │
│  │  - tools/unreal://skills/list (列出所有Skill)          │  │
│  │  - tools/unreal://skills/execute (执行Skill)           │  │
│  │  - notifications/skills/reloaded (Skill重载通知)       │  │
│  └──────────────┬──────────────────┬───────────────────────┘  │
│                 │                  │                          │
└─────────────────┼──────────────────┼──────────────────────────┘
                  │                  │
        ┌─────────▼─────────┐  ┌────▼──────────┐
        │   WorkBuddy       │  │   OpenClaw    │
        │   (MCP客户端)      │  │   (MCP客户端)  │
        └───────────────────┘  └───────────────┘
```

**优势**：
- ✅ 统一管理：所有Skill在UE插件侧管理，避免多客户端重复管理
- ✅ 平台无关：WorkBuddy、OpenClaw等所有MCP客户端使用同一套Skill
- ✅ 热加载即时生效：Skill重载后，所有连接的客户端无需重启即可使用
- ✅ 权限控制集中：安全策略在UE插件侧统一实施

## 🤝 与MCP客户端协作流程

### 场景1：WorkBuddy使用UE插件的Skill

```
1. 用户输入: "帮我优化选中的材质"
   ↓
2. WorkBuddy分析意图 → 需要调用材质优化Skill
   ↓
3. WorkBuddy通过MCP调用: unreal://skills/execute
   {
     "skill_name": "batch_material_optimizer",
     "parameters": {"target_actors": [...]}
   }
   ↓
4. UE插件接收调用 → 执行Skill → 返回结果
   ↓
5. WorkBuddy展示结果给用户
```

### 场景2：Skill热重载通知所有客户端

```
1. 开发者修改Skill文件: batch_material_optimizer.py
   ↓
2. UE插件文件监控检测到变化
   ↓
3. UE插件热重载Skill
   ↓
4. UE插件通过MCP推送通知给所有客户端:
   {
     "method": "notifications/skills/reloaded",
     "params": {"skill_name": "batch_material_optimizer"}
   }
   ↓
5. WorkBuddy接收通知 → 更新Skill列表缓存
   ↓
6. OpenClaw接收通知 → 更新Skill列表缓存
```

### 场景3：从远程市场安装Skill

```
1. 用户在WorkBuddy浏览Skill市场
   ↓
2. 用户选择安装: batch-material-optimizer-1.0.clawpkg
   ↓
3. WorkBuddy下载Skill包 → 发送到UE插件:
   unreal://skills/install
   {"package_path": ".../batch-material-optimizer-1.0.clawpkg"}
   ↓
4. UE插件解压并安装Skill → 热加载
   ↓
5. UE插件通知所有客户端: Skill列表已更新
   ↓
6. WorkBuddy和OpenClaw同步更新Skill列表
```
