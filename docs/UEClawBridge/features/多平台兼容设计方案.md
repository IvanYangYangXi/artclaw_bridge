# 多平台兼容设计方案

> **版本**: v1.1  
> **日期**: 2026-03-30  
> **状态**: 已确认

---

## 一、背景与目标

### 1.1 项目愿景

ArtClaw Bridge 的定位是：**让各种 DCC 软件（UE、Maya、Max）能快速接入到各种 Agent 平台（OpenClaw、WorkBuddy、Claude Desktop 等）**

### 1.2 当前问题

| 问题 | 说明 |
|------|------|
| 平台硬编码 | 当前只支持 OpenClaw，代码中硬编码了平台相关逻辑 |
| 配置读取 | MCP 配置读取固定路径 `~/.openclaw/openclaw.json` |
| Skill 目录 | Skills 安装位置硬编码为 `~/.openclaw/skills/` |
| 获取已安装列表 | Skill/MCP 管理面板硬编码读取平台目录 |

### 1.3 设计目标

1. **UE 插件通用化** - 不针对任何特定平台，插件代码不随平台变化而修改
2. **平台侧适配** - 新增平台时，只需在平台侧开发适配器，UE 插件无需改动
3. **安装时选择平台** - 第一次安装时选择目标平台，之后固定使用
4. **配置路径驱动** - 已安装 Skill/MCP 列表通过配置路径直接扫描获取

## 二、核心架构

### 2.1 架构概述

```
┌─────────────────────────────────────────────────────────┐
│                   UE Editor Agent                        │
│                   (DCC 插件，通用)                        │
│  ┌─────────────────────────────────────────────────┐    │
│  │  MCP Server (run_ue_python)                     │    │
│  │  Skill Hub (热加载)                              │    │
│  │  Memory Manager (记忆)                           │    │
│  │  Knowledge Base (知识)                           │    │
│  └─────────────────────────────────────────────────┘    │
└────────────────────────┬────────────────────────────────┘
                         │
                    长连接 (WebSocket)
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│               Agent 平台层 (可替换)                       │
│                                                         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │  OpenClaw   │  │  WorkBuddy  │  │   其他平台  │     │
│  │             │  │             │  │             │     │
│  │ + mcp-bridge│  │ + xxx-bridge│  │ + xxx-bridge│     │
│  │ + 平台配置   │  │ + 平台配置   │  │ + 平台配置  │     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
│         ▲                                     ▲         │
│         │ 平台侧适配器 (平台方提供)              │         │
└─────────┴─────────────────────────────────────┴─────────┘
```

### 2.2 职责划分

| 层级 | 职责 | 是否需要跨平台修改 |
|------|------|------------------|
| **UE 插件** | MCP Server、Skill 执行、记忆管理、UI | **不需要** |
| **长连接通信** | WebSocket 连接管理 | **不需要** |
| **平台侧适配器** | 协议转换、配置管理、平台特定功能 | **每个平台单独开发** |
| **项目源码** | Skill/MCP 源文件管理（official/marketplace） | **不需要** |

### 2.3 核心原则

1. **UE 插件不感知平台** - 插件代码不包含任何平台特定逻辑
2. **平台侧负责连接** - 平台提供适配器插件，负责与 UE 插件建立连接
3. **配置路径驱动** - 已安装 Skill/MCP 列表通过配置路径直接扫描获取，不绕 Agent 查询
4. **安装时选择平台** - 首次安装时选择目标平台，配置持久化

## 三、平台配置管理

### 3.1 安装时选择平台

**流程**：

```
用户安装 UE 插件
       │
       ▼
显示平台选择界面（首次安装时）
       │
       ├─ 选择 OpenClaw
       ├─ 选择 WorkBuddy
       └─ 选择 Claude Desktop
       │
       ▼
保存平台配置到本地（含 Skill/MCP 路径）
       │
       ▼
连接到选定的平台
```

**配置存储**：

```json
// ~/.artclaw/config.json
{
  "platform": {
    "type": "openclaw",
    "gateway_url": "ws://127.0.0.1:18789",
    "mcp_port": 8080
  },
  "skills": {
    "installed_path": "~/.openclaw/skills",
    "disabled": [],
    "pinned": []
  },
  "mcp": {
    "config_path": "~/.openclaw/openclaw.json",
    "config_key": "mcp.servers"
  },
  "project_root": "D:\\MyProject\\artclaw_bridge"
}
```

### 3.2 平台配置项

| 配置项 | 说明 | 来源 |
|--------|------|------|
| `platform.type` | 平台类型 | 安装时选择 |
| `platform.gateway_url` | Gateway WebSocket 地址 | 平台默认值，可修改 |
| `platform.mcp_port` | MCP Server 端口 | 平台默认值，可修改 |
| `skills.installed_path` | 已安装 Skill 目录 | 安装时按平台自动设置 |
| `mcp.config_path` | 平台 MCP 配置文件路径 | 安装时按平台自动设置 |
| `mcp.config_key` | 配置文件中 MCP Server 的 JSON key | 安装时按平台自动设置 |

### 3.3 不同平台的默认路径

| 平台 | skills.installed_path | mcp.config_path | mcp.config_key |
|------|----------------------|-----------------|----------------|
| OpenClaw | `~/.openclaw/skills` | `~/.openclaw/openclaw.json` | `mcp.servers` |
| WorkBuddy | `~/.workbuddy/skills` | `~/.workbuddy/config.json` | `mcpServers` |
| Claude Desktop | `~/.claude/skills` | `~/.claude/config.json` | `mcpServers` |

### 3.4 重新选择平台

在设置面板提供"重新选择平台"入口，切换时保留 Skill/MCP 状态，只修改平台连接信息和路径。

## 四、Skill 管理

### 4.1 源码管理（保持不变）

项目源码统一管理，保持 official/marketplace 结构：

```
skills/
├── official/           # 官方 Skill
│   ├── universal/      # 通用（无 DCC 前缀）
│   ├── unreal/         # UE 专用
│   ├── maya/           # Maya 专用
│   └── max/            # Max 专用
├── marketplace/        # 市集 Skill
└── user/               # 用户自定义
```

### 4.2 安装到平台

Skill 安装到平台私有目录，路径由 `config.skills.installed_path` 决定。

### 4.3 通过配置路径获取已安装列表

**核心思路**：安装时将获取路径写入配置，UE 插件从配置读取路径后直接扫描文件系统。

**获取列表逻辑**：

1. 读取 `config.skills.installed_path`
2. 扫描该目录下的所有 Skill 子目录
3. 解析每个 Skill 的 `manifest.json`（代码包）或 `SKILL.md`（文档型）
4. 返回并显示列表

**优势**：
- UE 插件通用，不含平台硬编码
- 直接读文件系统，快速可靠，不依赖 Agent 查询
- 新增平台只需定义默认路径
- 用户可自定义路径（非标准安装位置）

### 4.4 Skill 同步

同步操作通过已有的 `artclaw-skill-manage` Skill（skill_sync API）执行，安装脚本（install.bat）也会自动同步。

## 五、MCP Server 源码管理

### 5.1 目标

将 MCP Server 的配置模板和文档统一到项目源码中管理，与 Skill 源码结构对齐。**仅做目录整理和配置模板化，不做额外的 manifest 规范或自动安装体系。**

### 5.2 目录结构

```
mcp_servers/
├── official/
│   ├── ue-editor-agent/
│   │   ├── config_template.json   # 各平台配置模板
│   │   └── README.md              # 使用说明
│   ├── maya-editor-agent/
│   │   ├── config_template.json
│   │   └── README.md
│   └── max-editor-agent/
│       ├── config_template.json
│       └── README.md
└── marketplace/                    # 未来第三方 MCP Server
```

### 5.3 config_template.json 示例

```json
{
  "_comment": "UE Editor Agent MCP Server 配置模板",
  "platforms": {
    "openclaw": {
      "config_key": "mcp.servers.ue-editor-agent",
      "config_value": {
        "url": "ws://127.0.0.1:8080/ws",
        "type": "streamable-http"
      }
    },
    "workbuddy": {
      "config_key": "mcpServers.ue-editor-agent",
      "config_value": {
        "url": "ws://127.0.0.1:8080/ws"
      }
    }
  }
}
```

### 5.4 MCP 已安装列表获取

通过 `config.mcp.config_path` + `config.mcp.config_key` 直接读取平台配置文件：

1. 读取 `config.mcp.config_path`（如 `~/.openclaw/openclaw.json`）
2. 按 `config.mcp.config_key`（如 `mcp.servers`）定位配置节点
3. 提取 MCP Server 列表并返回

## 六、平台侧适配器

### 6.1 适配器职责

每个 Agent 平台需要提供适配器，负责：

1. **与 UE 插件建立长连接** - WebSocket 连接管理
2. **协议转换** - 将平台协议转换为 MCP 协议
3. **配置管理** - 读写平台配置文件

### 6.2 现有 OpenClaw 适配器

当前 OpenClaw 的 `mcp-bridge` 插件即为此适配器：

| 组件 | 职责 |
|------|------|
| `mcp-bridge/index.ts` | Gateway 侧 MCP 桥接 |
| `openclaw_bridge.py` | UE 侧与 Gateway 通信 |
| `mcp_server.py` | UE 侧 MCP Server |

> **注意**：`openclaw_bridge.py` 保持现有命名。通用逻辑已在 `bridge_core.py`，各平台特定逻辑各自独立文件（如未来 `workbuddy_bridge.py`），不合并为 `platform_bridge.py`。

### 6.3 新平台适配器要求

| 要求 | 说明 |
|------|------|
| **MCP 桥接插件** | 将平台协议转换为 MCP 协议 |
| **Gateway 连接** | 与 UE 插件建立 WebSocket 长连接 |
| **配置格式** | 定义 MCP Server 配置格式 |
| **平台 bridge.py** | 继承 bridge_core.py 的平台适配层 |
