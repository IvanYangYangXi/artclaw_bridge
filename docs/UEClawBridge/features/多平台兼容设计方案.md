# 多平台兼容设计方案

> **版本**: v1.0  
> **日期**: 2026-03-30  
> **状态**: 设计稿，待确认

---

## 一、背景与目标

### 1.1 项目愿景

ArtClaw Bridge 的定位是：**让各种 DCC 软件（UE、Maya、Max）能快速接入到各种 Agent 平台（OpenClaw、WorkBuddy、Claude Desktop 等）**

### 1.2 当前问题

| 问题 | 说明 |
|------|------|
| 平台硬编码 | 当前只支持 OpenClaw，代码中硬编码了平台相关逻辑 |
| 配置读取 | MCP 配置读取固定路径 `~/.openclaw/openclaw.json` |
| Skill 目录 | Skills 安装位置硬编码为 `~/.openclaw/skills/` |
| 获取已安装列表 | Skill/MCP 管理面板硬编码读取平台目录 |

### 1.3 设计目标

1. **UE 插件通用化** - 不针对任何特定平台，插件代码不随平台变化而修改
2. **平台侧适配** - 新增平台时，只需在平台侧开发适配器，UE 插件无需改动
3. **安装时选择平台** - 第一次安装时选择目标平台，之后固定使用
4. **通过 Agent 查询** - 获取已安装 Skill/MCP 列表通过 Agent 查询，而非直接读目录

## 二、核心架构

### 2.1 架构概述

```
┌─────────────────────────────────────────────────────────┐
│                   UE Editor Agent                        │
│                   (DCC 插件，通用)                        │
│  ┌─────────────────────────────────────────────────┐    │
│  │  MCP Server (run_ue_python)                     │    │
│  │  Skill Hub (热加载)                              │    │
│  │  Memory Manager (记忆)                           │    │
│  │  Knowledge Base (知识)                           │    │
│  └─────────────────────────────────────────────────┘    │
└────────────────────────┬────────────────────────────────┘
                         │
                    长连接 (WebSocket)
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│               Agent 平台层 (可替换)                       │
│                                                         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │  OpenClaw   │  │  WorkBuddy  │  │   其他平台  │     │
│  │             │  │             │  │             │     │
│  │ + mcp-bridge│  │ + xxx-bridge│  │ + xxx-bridge│     │
│  │ + 平台配置   │  │ + 平台配置   │  │ + 平台配置  │     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
│         ▲                                     ▲         │
│         │ 平台侧适配器 (平台方提供)              │         │
└─────────┴─────────────────────────────────────┴─────────┘
```

### 2.2 职责划分

| 层级 | 职责 | 是否需要跨平台修改 |
|------|------|------------------|
| **UE 插件** | MCP Server、Skill 执行、记忆管理、UI | **不需要** |
| **长连接通信** | WebSocket 连接管理 | **不需要** |
| **平台侧适配器** | 协议转换、配置管理、平台特定功能 | **每个平台单独开发** |
| **项目源码** | Skill/MCP 源文件管理（official/marketplace） | **不需要** |

### 2.3 核心原则

1. **UE 插件不感知平台** - 插件代码不包含任何平台特定逻辑
2. **平台侧负责连接** - 平台提供适配器插件，负责与 UE 插件建立连接
3. **通过 Agent 查询状态** - 已安装的 Skill/MCP 列表通过 Agent 查询返回
4. **安装时选择平台** - 首次安装时选择目标平台，配置持久化

## 三、平台配置管理

### 3.1 安装时选择平台

**流程**：

```
用户安装 UE 插件
       │
       ▼
显示平台选择界面（首次安装时）
       │
       ├─ 选择 OpenClaw
       ├─ 选择 WorkBuddy
       └─ 选择 Claude Desktop
       │
       ▼
保存平台配置到本地
       │
       ▼
连接到选定的平台
```

**配置存储**：

```json
// ~/.artclaw/config.json
{
  "platform": {
    "type": "openclaw",  // openclaw / workbuddy / claude
    "gateway_url": "ws://127.0.0.1:18789",
    "mcp_port": 8080
  },
  "project_root": "D:\\MyProject\\artclaw_bridge"
}
```

### 3.2 平台配置项

| 配置项 | 说明 | 来源 |
|--------|------|------|
| `platform.type` | 平台类型 | 安装时选择 |
| `platform.gateway_url` | Gateway WebSocket 地址 | 平台默认值，可修改 |
| `platform.mcp_port` | MCP Server 端口 | 平台默认值，可修改 |

### 3.3 重新选择平台

在设置面板提供"重新选择平台"入口：

```
设置面板
├── 平台配置
│   ├── 当前平台: OpenClaw v1.5.0
│   ├── Gateway: ws://127.0.0.1:18789
│   ├── [重新选择平台]
│   └── [测试连接]
```

## 四、Skill 管理

### 4.1 源码管理（保持不变）

项目源码统一管理，保持 official/marketplace 结构：

```
skills/
├── official/           # 官方 Skill
│   ├── universal/    # 通用（无 DCC 前缀）
│   ├── unreal/        # UE 专用
│   ├── maya/         # Maya 专用
│   └── max/          # Max 专用
├── marketplace/       # 市集 Skill
└── user/              # 用户自定义
```

### 4.2 安装到平台

**当前设计**：Skill 安装到平台私有目录

| 平台 | Skills 目录 |
|------|------------|
| OpenClaw | `~/.openclaw/skills/` |
| WorkBuddy | `~/.workbuddy/skills/` |
| Claude Desktop | `~/.claude/skills/` |

### 4.3 通过配置路径获取已安装列表

**核心思路**：安装时将获取路径写入配置，UE 插件从配置读取路径后直接获取

**流程**：

```
安装到平台时
       │
       ▼
写入配置:
  skills.installed_path = "~/.openclaw/skills"
       │
       ▼
UE 插件启动时读取配置
       │
       ▼
从 installed_path 扫描 Skill 目录
       │
       ▼
解析 manifest.json / SKILL.md
       │
       ▼
返回 Skill 列表
```

**获取列表逻辑**：

```
1. 用户点击 "Skills 管理"
2. UE 插件读取 config.skills.installed_path
3. 扫描该目录下的所有子目录
4. 解析每个 Skill 的元信息:
   - manifest.json (代码包 Skill)
   - SKILL.md (文档型 Skill)
5. 返回并显示列表
```

**需要的 Skill（用于同步操作）**：

| Skill 名称 | 功能 |
|------------|------|
| `artclaw-skill-install` | 安装 Skill 到当前平台 |
| `artclaw-skill-uninstall` | 卸载当前平台的 Skill |
| `artclaw-sync-skills` | 同步 Skill 到当前平台 |

## 五、MCP Server 管理

### 5.1 MCP Server 源码结构

新增 MCP Server 源码管理，与 Skill 类似：

```
mcp_servers/
├── official/
│   ├── ue-editor-agent/
│   │   ├── manifest.json        # MCP Server 元信息
│   │   ├── config_template.json # 配置模板
│   │   └── README.md
│   └── maya-editor-agent/
├── marketplace/
└── templates/
```

### 5.2 manifest.json 结构

```json
{
  "id": "ue-editor-agent",
  "name": "UE Claw Bridge",
  "version": "1.0.0",
  "dcc": "unreal_engine",
  "description": "Unreal Engine Editor MCP Server",
  "port_range": [8080, 8085],
  "platforms": {
    "openclaw": {
      "config_key": "mcp.servers.ue-editor-agent",
      "install_command": "安装 UE Editor Agent 到 OpenClaw"
    },
    "workbuddy": {
      "config_key": "mcpServers.ue-editor-agent",
      "install_command": "安装 UE Editor Agent 到 WorkBuddy"
    }
  }
}
```

### 5.3 通过配置路径获取已安装 MCP

**流程**：

```
安装到平台时
       │
       ▼
写入配置:
  mcp.config_path = "~/.openclaw/openclaw.json"
  mcp.config_key = "mcp.servers"
       │
       ▼
UE 插件启动时读取配置
       │
       ▼
从 config_path 解析 MCP Server 配置
       │
       ▼
返回 MCP Server 列表
```

**获取列表逻辑**：

```
1. 用户点击 "MCP 管理"
2. UE 插件读取 config.mcp.config_path
3. 解析配置文件
4. 根据 config.mcp.config_key 定位 MCP Server 配置节点
5. 提取 MCP Server 信息并返回
```

**需要的 Skill（用于安装操作）**：

| Skill 名称 | 功能 |
|------------|------|
| `artclaw-mcp-install` | 安装 MCP Server 到当前平台 |
| `artclaw-mcp-uninstall` | 卸载当前平台的 MCP Server |

## 六、平台侧适配器

### 6.1 适配器职责

每个 Agent 平台需要提供适配器，负责：

1. **与 UE 插件建立长连接** - WebSocket 连接管理
2. **协议转换** - 将平台协议转换为 MCP 协议
3. **配置管理** - 读写平台配置文件
4. **查询接口实现** - 提供查询已安装 Skill/MCP 的接口

### 6.2 现有 OpenClaw 适配器

当前 OpenClaw 的 `mcp-bridge` 插件即为此适配器：

| 组件 | 职责 |
|------|------|
| `mcp-bridge/index.ts` | Gateway 侧 MCP 桥接 |
| `openclaw_bridge.py` | UE 侧与 Gateway 通信 |
| `mcp_server.py` | UE 侧 MCP Server |

### 6.3 新平台适配器要求

新增平台需要提供：

| 要求 | 说明 |
|------|------|
| **MCP 桥接插件** | 将平台协议转换为 MCP 协议 |
| **Gateway 连接** | 与 UE 插件建立 WebSocket 长连接 |
| **配置格式** | 定义 MCP Server 配置格式 |
| **Skill 实现** | 实现 `artclaw-query-skills`、`artclaw-query-mcps` 等 Skill |

## 十、已安装列表获取方式

### 10.1 核心思路

**安装时写入获取路径**，而非硬编码平台目录：

```
安装到 Agent 平台时
       │
       ▼
将 Skill/MCP 的获取路径写入配置
       │
       ▼
UE 插件从配置读取路径
       │
       ▼
直接从路径获取已安装列表
```

### 10.2 配置结构

```json
// ~/.artclaw/config.json
{
  "platform": {
    "type": "openclaw",
    "gateway_url": "ws://127.0.0.1:18789",
    "mcp_port": 8080
  },
  "skills": {
    "installed_path": "~/.openclaw/skills",
    "disabled": [],
    "pinned": []
  },
  "mcp": {
    "config_path": "~/.openclaw/openclaw.json",
    "config_key": "mcp.servers"
  },
  "project_root": "D:\\MyProject\\artclaw_bridge"
}
```

### 10.3 安装时写入路径

**安装脚本/流程**：

```
用户选择目标平台 (如 OpenClaw)
       │
       ▼
安装脚本根据平台类型设置默认路径:
  - skills.installed_path = "~/.openclaw/skills"
  - mcp.config_path = "~/.openclaw/openclaw.json"
       │
       ▼
写入配置文件
       │
       ▼
UE 插件启动时读取配置
       │
       ▼
从 skills.installed_path 获取 Skill 列表
从 mcp.config_path 获取 MCP 配置
```

### 10.4 不同平台的默认路径

| 平台 | skills.installed_path | mcp.config_path |
|------|----------------------|-----------------|
| OpenClaw | `~/.openclaw/skills` | `~/.openclaw/openclaw.json` |
| WorkBuddy | `~/.workbuddy/skills` | `~/.workbuddy/config.json` |
| Claude Desktop | `~/.claude/skills` | `~/.claude/config.json` |

### 10.5 UE 插件获取列表逻辑

**Skill 列表**：
1. 读取 `config.skills.installed_path`
2. 扫描该目录下的所有 Skill 目录
3. 解析每个 Skill 的 `manifest.json` 或 `SKILL.md`
4. 返回列表

**MCP Server 列表**：
1. 读取 `config.mcp.config_path`
2. 解析配置文件中的 MCP Server 配置
3. 返回列表

### 10.6 优势

| 优势 | 说明 |
|------|------|
| **UE 插件通用** | 不包含任何平台硬编码，只读取配置中的路径 |
| **灵活扩展** | 新增平台只需定义默认路径，UE 插件无需修改 |
| **可自定义** | 用户可以修改配置中的路径（如非标准安装位置） |
| **简单可靠** | 直接读取文件系统，不依赖 Agent 查询接口 |
