**版本**: v2.0  
**日期**: 2026-03-31  
**状态**: Phase 6-8 完成，Phase 9 进行中

---

## 一、背景与目标

ArtClaw Bridge 定位：**让各种 DCC 软件（UE、Maya、Max）能快速接入各种 Agent 平台（OpenClaw、WorkBuddy、Claude Desktop 等）**

### 设计原则

1. **DCC 插件不感知平台** — 只依赖共享核心层
2. **平台 Bridge 独立管理** — 每个平台一个完整子项目
3. **安装时选择平台** — install.bat `--platform` 参数，不在运行时切换
4. **配置路径驱动** — 所有路径从 `~/.artclaw/config.json` 读取

## 二、核心架构

```
┌──────────────────────────────────────────────┐
│         DCC 插件 (UE / Maya / Max)            │
│         MCP Server / Skill Hub / Memory       │
└──────────────┬───────────────────────────────┘
               │ 调用共享核心
┌──────────────┴───────────────────────────────┐
│              共享核心层 core/                   │
│  bridge_core / bridge_config / memory_core    │
└──────────────┬───────────────────────────────┘
               │ WebSocket 长连接
┌──────────────┴───────────────────────────────┐
│         平台 Bridge (可替换)                    │
│  openclaw/ │ workbuddy/ │ claude/ │ ...       │
└──────────────────────────────────────────────┘
```

## 四、平台配置

```json
// ~/.artclaw/config.json
{
  "platform": { "type": "openclaw", "gateway_url": "ws://127.0.0.1:18789" },
  "skills": { "installed_path": "~/.openclaw/skills" },
  "mcp": { "config_path": "~/.openclaw/openclaw.json", "config_key": "mcp.servers" },
  "project_root": "D:\\MyProject_D\\artclaw_bridge"
}
```

| 平台 | skills 路径 | mcp config_key |
|------|------------|----------------|
| OpenClaw | `~/.openclaw/skills` | `mcp.servers` |
| WorkBuddy | `~/.workbuddy/skills` | `mcpServers` |
| Claude | `~/.claude/skills` | `mcpServers` |

## 五、共享模块同步保障

**问题**: core/ 和 platforms/ 源码与 DCC/UE 副本容易漂移

**方案**: `verify_sync.py` 校验工具
- 检查模式: `python verify_sync.py`（仅报告差异）
- 修复模式: `--fix`（源码→副本）/ `--reverse`（副本→源码）
- CI 模式: `--ci`（不一致时退出码 1）
- 覆盖范围: core/ 6 模块 × 2 端 + platforms/openclaw/ 3 模块 × 1 端 = 15 对
- install.py 安装后自动运行校验
