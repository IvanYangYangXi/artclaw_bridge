# 多平台兼容设计方案

> **版本**: v1.2  
> **日期**: 2026-03-30  
> **状态**: 已确认

---

## 一、背景与目标

### 1.1 项目愿景

ArtClaw Bridge 的定位是：**让各种 DCC 软件（UE、Maya、Max）能快速接入到各种 Agent 平台（OpenClaw、WorkBuddy、Claude Desktop 等）**

### 1.2 当前问题

| 问题 | 说明 |
|------|------|
| 平台硬编码 | 当前只支持 OpenClaw，代码中硬编码了平台相关逻辑 |
| 目录混乱 | `openclaw-mcp-bridge/` 里混着共享核心和 OpenClaw 特定代码 |
| Skill 目录 | Skills 安装位置硬编码为 `~/.openclaw/skills/` |
| MCP Server | 配置模板散落在各处 |

### 1.3 设计目标

1. **UE/DCC 插件通用化** — 不针对任何特定平台，插件代码不随平台变化而修改
2. **平台 Bridge 独立管理** — 每个平台一个完整的 Bridge 子项目，与引擎/DCC 插件解耦
3. **安装时选择平台** — 通过 install.bat 一次性选择，不在引擎/DCC 内切换
4. **配置路径驱动** — 已安装 Skill/MCP 列表通过配置路径直接扫描获取

## 二、核心架构

### 2.1 架构概述

```
┌──────────────────────────────────────────────────────────┐
│              DCC 插件 (UE / Maya / Max)                    │
│              (通用，不感知平台)                              │
│  ┌────────────────────────────────────────────────────┐  │
│  │  MCP Server (run_ue_python / run_python)            │  │
│  │  Skill Hub / Memory / Knowledge Base                │  │
│  └────────────────────────────────────────────────────┘  │
└─────────────────────────┬────────────────────────────────┘
                          │ 调用共享核心 (bridge_core.py 等)
                          │
┌─────────────────────────┴────────────────────────────────┐
│                    共享核心层                               │
│  core/ (bridge_core / bridge_config / memory_core / ...)  │
│  安装时复制到各 DCC 插件目录                                 │
└─────────────────────────┬────────────────────────────────┘
                          │ 长连接 (WebSocket)
                          │
┌─────────────────────────┴────────────────────────────────┐
│              平台 Bridge (可替换的子项目)                    │
│                                                           │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐    │
│  │ openclaw/    │  │ workbuddy/   │  │   其他平台/  │    │
│  │              │  │              │  │              │    │
│  │ bridge.py    │  │ bridge.py    │  │ bridge.py    │    │
│  │ gateway/     │  │ gateway/     │  │ gateway/     │    │
│  │ setup.*      │  │ setup.*      │  │ setup.*      │    │
│  │ config/      │  │ config/      │  │ config/      │    │
│  └──────────────┘  └──────────────┘  └──────────────┘    │
└──────────────────────────────────────────────────────────┘
```

### 2.2 职责划分

| 层级 | 职责 | 修改频率 |
|------|------|---------|
| **DCC 插件** | MCP Server、Skill 执行、记忆、UI | 功能迭代时 |
| **共享核心 `core/`** | 通信协议、配置读取、诊断、记忆核心 | 很少 |
| **平台 Bridge** | 平台特定连接、配置部署、Gateway 插件 | 每平台独立 |

### 2.3 核心原则

1. **DCC 插件不感知平台** — 只依赖共享核心层
2. **平台 Bridge 是独立子项目** — 各自完整，与引擎/DCC 插件解耦
3. **安装时一次性选择** — install.bat 参数 `--platform openclaw|workbuddy|claude`
4. **切平台 = 重新安装** — 不在运行时切换，简单可靠

### 3.2 与现状对比

| 现状 | 新结构 | 说明 |
|------|--------|------|
| `openclaw-mcp-bridge/bridge_core.py` | `core/bridge_core.py` | 共享核心提取到根 `core/` |
| `openclaw-mcp-bridge/openclaw_bridge.py` | `platforms/openclaw/openclaw_bridge.py` | 平台特定代码归到平台目录 |
| `openclaw-mcp-bridge/mcp-bridge/index.ts` | `platforms/openclaw/gateway/index.ts` | Gateway 插件归到平台目录 |
| `openclaw-mcp-bridge/templates/` | `platforms/openclaw/config/` | 配置模板归到平台目录 |
| 三端手动 copy 同步 | `core/` 是唯一源码，install 时复制 | 消除同步问题 |

### 3.3 安装时复制规则

install.bat 安装时：
1. 将 `core/` 下所有 `.py` 复制到 DCC 插件的对应目录
2. 将选定平台的 `platforms/{platform}/` 特定文件复制到对应位置
3. 写入 `~/.artclaw/config.json` 的 platform 配置

## 四、平台配置管理

### 4.1 配置文件

```json
// ~/.artclaw/config.json
{
  "platform": {
    "type": "openclaw",
    "gateway_url": "ws://127.0.0.1:18789",
    "mcp_port": 8080
  },
  "skills": {
    "installed_path": "~/.openclaw/skills",
    "disabled": [],
    "pinned": []
  },
  "mcp": {
    "config_path": "~/.openclaw/openclaw.json",
    "config_key": "mcp.servers"
  },
  "project_root": "D:\\MyProject\\artclaw_bridge"
}
```

### 4.2 不同平台的默认路径

| 平台 | skills.installed_path | mcp.config_path | mcp.config_key |
|------|----------------------|-----------------|----------------|
| OpenClaw | `~/.openclaw/skills` | `~/.openclaw/openclaw.json` | `mcp.servers` |
| WorkBuddy | `~/.workbuddy/skills` | `~/.workbuddy/config.json` | `mcpServers` |
| Claude Desktop | `~/.claude/skills` | `~/.claude/config.json` | `mcpServers` |

### 4.3 平台选择

- **install.bat --platform openclaw** → 安装时选择平台
- 选择后写入配置，后续所有路径由配置驱动
- 切平台 = 重新运行 install.bat 选择不同平台

## 五、Skill 管理

### 5.1 已安装列表获取

通过 `config.skills.installed_path` 直接扫描文件系统（统一安装目录，扁平结构）：

1. 读取配置中的 `installed_path`（如 `~/.openclaw/skills`）
2. 扫描目录下所有 Skill 子目录（扁平结构，无分层）
3. 解析每个子目录的 `manifest.json` 或 `SKILL.md`
4. 返回已安装 Skill 列表

**安装目录变更**：Skills 现已统一安装到外部配置目录，采用扁平结构，不再使用 `00_official/`, `01_team/` 等分层目录。

### 5.2 Skill 安装流程

通过 `install.py --openclaw` 执行：
1. 扫描项目源码 `skills/official/` 和 `skills/marketplace/`
2. 用 `shutil.copytree` 复制整个 Skill 目录（含代码和文档）到 `skills.installed_path`
3. 支持扁平和分层两种目录结构扫描，向后兼容

## 六、平台 Bridge 开发规范

### 6.1 最低要求

每个平台 Bridge 需要提供：

| 文件 | 职责 |
|------|------|
| `{platform}_bridge.py` | 继承 bridge_core.py，实现平台特定连接逻辑 |
| `setup.bat` / `setup.py` | 平台配置部署脚本 |
| `gateway/` | Gateway 侧插件（如有） |
| `config/` | 配置模板 |
| `README.md` | 使用说明 |

### 6.2 与 DCC 插件的关系

- 平台 Bridge 和 DCC 插件**完全解耦**
- DCC 插件只依赖 `core/` 共享核心
- 平台 Bridge 的 `{platform}_bridge.py` 在安装时复制到 DCC 插件目录

## 七、已确认事项

| 问题 | 决定 |
|------|------|
| 平台配置存储 | `~/.artclaw/config.json` 统一管理 |
| 已安装列表获取 | 配置路径直接扫描 |
| 平台切换方式 | 重新运行 install.bat，不在引擎/DCC 内切换 |
| 共享核心位置 | 根目录 `core/`，安装时复制到各 DCC |
| 平台 Bridge 位置 | `platforms/{platform}/` |
| MCP Server 配置 | 各平台配置由 `install.py` 的 `PLATFORM_CONFIGS` 管理，`platforms/*/config/` 存模板 |
| `openclaw_bridge.py` 命名 | 保持，各平台各自独立文件 |
