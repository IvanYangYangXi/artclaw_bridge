# 5.8 多会话管理 — 详细规划文档

> 版本: v1.0 | 日期: 2026-03-28 | 状态: 规划中

---

## 1. 需求分析

### 用户痛点
- 当前只能看到一个活跃对话，新建对话后旧对话消失
- 无法回顾之前的对话内容
- 无法在多个主题之间切换

### 核心功能
1. **会话列表**: 显示所有历史会话，按时间排序
2. **会话切换**: 点击列表项切换到对应会话，加载历史消息
3. **会话删除**: 删除不需要的会话
4. **当前会话标识**: 明确标识当前活跃会话

---

## 2. 技术方案

### 2.1 数据来源

**方案: 本地维护会话列表 + Gateway transcript 加载消息历史** ✅ 已确认
- C++ 侧维护 `TArray<FSessionEntry>` 列表（仅 label + key，不保存消息）
- 每次 `/new` 时新增条目（时间戳 + 标签 + session key）
- 切换会话时从 Gateway transcript 文件加载历史消息（不做本地消息持久化）
- 会话列表本身也不需要本地持久化（进程内有效即可）

### 2.2 会话历史消息加载

**切换会话时如何加载消息?**

方案: **从 Gateway transcript 文件读取** ✅ 已确认
- 会话的 JSONL transcript 存储在 `~/.openclaw/agents/<agentId>/sessions/<sessionId>.jsonl`
- Python 侧新增方法 `load_session_history(session_key)` → 读取并解析 transcript JSONL
- 解析出 user/assistant 消息，忽略 system/tool 内部条目
- 返回给 C++ 侧渲染
- 不做本地消息缓存，每次切换都从 transcript 读取

### 2.3 UI 设计

**布局方案**: 下拉菜单（Combo Box）+ 会话管理按钮

位置: 在 `/new` 按钮旁，添加会话选择下拉菜单

```
[▼ 对话 03-28 09:15] [+ 新对话] [管理...]
```

- 下拉菜单显示最近 N 个会话（默认 10）
- "管理..." 按钮弹窗显示完整列表（可搜索、可删除）

**不用侧边栏的原因**: UE Editor 的面板空间有限，侧边栏会占用太多水平空间

### 2.4 数据模型

```cpp
struct FSessionEntry
{
    FString SessionKey;       // "agent/ue-editor:1711612345000"
    FString SessionId;        // transcript 文件对应的 session id
    FString Label;            // "对话 03-28 09:15"
    FDateTime CreatedAt;
    bool bIsActive = false;
    // 消息不存本地，切换时从 transcript 读取
};
```

### 2.5 核心流程

1. **新建会话** (`OnNewChatClicked`):
   - 保存当前会话的 Messages 到 SessionEntries
   - PlatformBridge->ResetSession()（清 session key）
   - 创建新 FSessionEntry，设为 active
   - 清空显示 + 发送 `/new`

2. **切换会话** (`OnSessionSelected`):
   - 保存当前 session 信息到 SessionEntries
   - Python 侧 `load_session_history(session_key)` → 读 transcript JSONL → 返回消息列表
   - 更新 Messages 数组
   - PlatformBridge->SetSessionKey(key) — 需要新增此接口
   - RebuildMessageList()

3. **删除会话** (`OnDeleteSession`):
   - 从 SessionEntries 移除
   - 不需要通知 Gateway（不影响 Gateway 侧的 session）
