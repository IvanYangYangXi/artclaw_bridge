# WorkBuddy侧实现指南：工具与资源集成

**对应UE侧阶段**：阶段1.2-2.2（核心基础设施 + 感知增强）  
**前置条件**：基础连接已完成  
**预估工时**：2-3天

---

## 目标

WorkBuddy能调用UE插件暴露的Tools，并通过Resources获取编辑器上下文。

**验收标准：**
- Tools列表自动同步，AI能调用任意工具
- 编辑器关键状态变化（模式切换、关卡加载）能通知到WorkBuddy
- System Prompt根据编辑器当前模式动态调整

---

## 实现步骤

### 步骤1：Tools自动发现

WorkBuddy连接后自动发送 `tools/list`，拿到UE插件注册的所有工具。

UE插件响应示例：
```json
{
  "tools": [
    {
      "name": "run_ue_python",
      "description": "执行Python代码",
      "inputSchema": { "properties": { "code": { "type": "string" } }, "required": ["code"] }
    },
    {
      "name": "spawn_actor",
      "description": "在场景中生成Actor",
      "inputSchema": { "properties": { "class_path": {"type": "string"}, "location": {"type": "object"} } }
    }
  ]
}
```

无需额外配置，WorkBuddy会自动将这些工具提供给AI使用。

### 步骤2：Resources上下文同步

采用**混合模式**，避免token浪费：

**按需拉取（主要方式）**：AI在处理用户请求时，根据意图主动读取相关资源：
- 用户问"当前选了什么" → 读取 `unreal://level/selected_actors`
- 用户问"关卡里有什么" → 读取 `unreal://level/all_actors`

**关键事件推送（补充）**：只订阅高价值、低频的编辑器事件：
- `unreal://editor/mode_changed`（模式切换：关卡编辑器/蓝图/材质）
- `unreal://level/loaded`（关卡加载完成）
- `unreal://transaction/committed`（用户做了一次可撤销操作）

UE插件在这些事件发生时，通过MCP通知推送：
```json
{
  "method": "notifications/resources/updated",
  "params": { "uri": "unreal://editor/mode", "contents": "blueprint_editor" }
}
```

### 步骤3：动态System Prompt

WorkBuddy接收到编辑器模式变化通知后，更新System Prompt前缀：

| 编辑器模式 | System Prompt附加内容 |
|-----------|---------------------|
| 关卡编辑器 | 当前在关卡编辑器，可操作Actor、地形、光照 |
| 蓝图编辑器 | 当前在蓝图编辑器，可创建节点、连接逻辑 |
| 材质编辑器 | 当前在材质编辑器，可编辑材质节点和参数 |

实现方式：WorkBuddy订阅模式变化通知，在构建对话上下文时动态拼入对应描述。

### 步骤4：Tool调用确认

危险操作（含 `delete`/`destroy`/`remove`/`batch` 关键词的工具）调用前，WorkBuddy应向用户展示确认提示，明确说明将要执行的操作和影响范围。

## 典型交互流程

```
用户："在原点放一个立方体"
  ↓
WorkBuddy 生成调用：tools/call → spawn_actor
  参数：{ class_path: "StaticMeshActor", location: [0,0,0], mesh: "Cube" }
  ↓
UE插件执行，返回：{ actor_name: "StaticMeshActor_001", location: [0,0,0] }
  ↓
WorkBuddy 回复用户："已在原点生成立方体，名称 StaticMeshActor_001"
```

---

## 常见问题

**Tools列表为空**：UE插件注册逻辑未执行，查看UE Output Log。

**Resources不更新**：检查UE插件是否实现了关键事件推送；手动调用 `resources/read` 验证能否拉取。

**Tool调用无响应**：查看UE插件日志，通常是主线程调度问题（操作未进入命令队列）。
