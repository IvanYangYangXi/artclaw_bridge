# WorkBuddy侧实现指南：高级功能与智能增强

**对应UE侧阶段**：阶段2.3-3.3  
**前置条件**：工具与资源集成已完成  
**预估工时**：3-4天

---

## 目标

完善WorkBuddy侧的安全确认机制、RAG知识库支持、Skill热加载接入。

**验收标准：**
- 高风险操作触发确认对话框
- RAG能检索UE API文档并注入上下文
- UE插件热加载Skill后，WorkBuddy能立即感知并使用新工具

---

## 实现步骤

### 步骤1：风险分级确认

**风险判断规则**（在WorkBuddy侧实现）：

| 风险级别 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 低 | 只读查询类工具 | 直接执行 |
| 中 | 修改单个对象 | 执行前提示 |
| 高 | 批量操作 | 弹出确认框，列出受影响对象 |
| 危险 | 包含 delete/destroy/remove | 弹出醒目确认框，需明确点击确认 |

WorkBuddy在调用工具前，根据工具名称和参数评估风险等级，高风险操作暂停执行、等待用户确认。

### 步骤2：RAG知识库集成

目的：让AI在处理UE相关问题时，能引用项目文档和UE API参考，减少幻觉。

**知识库内容来源：**
- UE Python API文档（从官方文档整理为Markdown）
- 项目自身的编码规范文档
- 已有Skill的使用说明

**集成方式**：WorkBuddy内置RAG能力，配置知识库目录路径即可。具体配置参数以WorkBuddy官方文档为准。

**检索注入时机**：用户提问涉及UE API调用或项目规范时，自动检索相关片段注入System Prompt。

### 步骤3：Skill热加载接入

**Skill在UE插件侧管理**，WorkBuddy作为MCP客户端只需：

1. **感知Skill列表变化**：订阅 `notifications/skills/reloaded` 通知，收到后刷新本地tools缓存

2. **调用Skill**：Skill被注册为普通MCP工具，调用方式与其他工具一致：
   ```
   tools/call → skill_name → parameters
   ```

3. **无需WorkBuddy侧额外管理**：Skill的加载、卸载、版本管理全部由UE插件负责

**验证方式**：修改UE插件侧某个Skill文件，保存后在WorkBuddy调用该工具，确认新逻辑已生效。

## 常见问题

**确认框不弹出**：检查WorkBuddy是否支持工具调用前拦截；或在System Prompt里指示AI对高风险操作必须先向用户描述再执行。

**RAG检索结果不相关**：知识库文档质量问题，精简文档内容，去掉与UE操作无关的内容；调高相关性阈值。

**Skill热加载后WorkBuddy未感知**：确认UE插件在热加载完成后发送了 `notifications/skills/reloaded` 通知；或手动断开重连MCP。
