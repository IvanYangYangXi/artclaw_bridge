# WorkBuddy侧实现指南：基础连接与配置

**对应UE侧阶段**：阶段0（工程初始化）+ 阶段1.1（MCP WebSocket网关）  
**前置条件**：UE插件完成阶段1.1，MCP服务器已在某端口监听  
**预估工时**：1天

---

## 目标

在WorkBuddy中配置MCP服务器连接，建立与UE插件的WebSocket通信。

**验收标准：**
- WorkBuddy成功连接UE插件MCP服务器
- 断线后能自动重连
- MCP初始化握手成功（能拿到tools/list响应）

---

## 实现步骤

### 步骤1：配置MCP服务器

WorkBuddy通过配置文件或设置界面添加MCP服务器。核心参数：

| 参数 | 值 | 说明 |
|------|-----|------|
| 连接类型 | WebSocket | UE插件暴露WebSocket端点 |
| 地址 | `ws://localhost:8080/mcp` | 端口与UE插件保持一致 |
| 自动连接 | 是 | IDE启动时自动连接 |
| 重连次数 | 5 | 断线后重试 |

> **注意**：WorkBuddy实际支持的配置字段以其官方文档为准，此处仅描述需要配置的能力。

### 步骤2：配置System Prompt

为UE Editor Agent配置专用System Prompt，告知AI当前可用能力：

```
你是一个UE编辑器助手，通过MCP协议操作Unreal Engine编辑器。
可用工具来自连接的UE插件，执行高风险操作（删除、批量修改）前需向用户确认。
```

### 步骤3：验证连接

连接成功的标志：
- WorkBuddy状态栏显示已连接
- 调用 `tools/list` 能返回UE插件注册的工具列表
- 调用 `resources/list` 能返回编辑器资源列表

**快速验证方式**：在WorkBuddy对话框输入"列出当前可用的UE工具"，AI应能返回工具清单。

---

## 常见问题

**连接失败（ECONNREFUSED）**：确认UE插件已启动，端口号一致，防火墙未拦截。

**连接成功但tools/list为空**：UE插件侧工具注册逻辑有问题，查看UE Output Log。

**多个UE项目并行**：为每个项目配置不同端口（如8080、8081），在WorkBuddy中切换活动服务器。
