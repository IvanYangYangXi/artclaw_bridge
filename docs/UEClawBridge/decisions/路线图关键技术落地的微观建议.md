# UE Editor Agent 关键技术落地微观建议 (Technical Deep Dive)

针对你路线图中的核心挑战，以下是从底层实现角度给出的微观策略，旨在确保系统在虚幻引擎中的**稳定性**与**高响应性**。

---

### 1. 线程调度与指令同步 (Phase 1.5)
**痛点**：WebSocket 消息在非游戏线程到达，而 `unreal` 模块（如修改 Actor）必须在 GameThread 执行。
- **微观建议**：使用 **Task Graph** 或 **AsyncTask** 封装执行器。
- **实现细节**：
    - 创建一个 `FUEAgentTaskQueue` 类。
    - 接收到 JSON-RPC 指令后，不直接运行，而是将其封装为 `TFunction<void()>` 丢入 `AsyncTask(ENamedThreads::GameThread, [Task](){ Task(); });`。
    - **进阶**：对于涉及渲染或物理的重操作，建议在 `PreTick` 或 `PostTick` 钩子中执行，避免破坏引擎的帧同步。

### 2. 事务保护与回滚机制 (Phase 1.4)
**痛点**：AI 一次性生成 100 个光源，用户想撤销时需要手动点 100 次 Undo。
- **微观建议**：强制开启 **Scoped Transaction**。
- **实现细节**：
    - 在 Python 万能执行器的入口处，必须调用 `unreal.EditorAssetLibrary.begin_transaction("AI_Action")`。
    - 执行完毕后调用 `end_transaction()`。
    - **风险预警**：如果 Python 脚本中途 Crash，务必在 `finally` 块中关闭 Transaction，否则会造成 Undo 栈损坏或编辑器 UI 卡死。

### 3. 感知增强的数据精简 (Phase 2.6)
**痛点**：将整个项目上万个资产名塞给 LLM 会导致 Token 爆炸且响应极慢。
- **微观建议**：引入 **语义视锥（Semantic Viewport）** 过滤。
- **实现细节**：
    - **视口优先**：仅将用户当前相机视野（Camera Frustum）内的 Actor 列表传给 Agent。
    - **选中优先**：默认只注入 `unreal.EditorLevelLibrary.get_selected_level_actors()` 的元数据。
    - **按需索取**：实现一个 `search_assets(query)` Tool，当 AI 明确需要新模型时，再调用 `AssetRegistry` 进行搜索，而非预加载。

### 4. 隔离的 Python 环境与热加载 (Phase 0.4 & 3.1)
**痛点**：第三方库（如 `pydantic`, `mcp`）与 UE 自带库版本冲突。
- **微观建议**：利用 `site.addsitedir()` 动态切入 **Private Site-Packages**。
- **实现细节**：
    - 在插件目录下创建 `Binaries/ThirdParty/Python/Lib/site-packages`。
    - 在 `init_unreal.py` 中，使用 `sys.path.insert(0, ...)` 确保插件私有库优先级高于系统全局库。
    - **热加载技巧**：使用 `importlib.reload()` 配合文件系统观察者（FileWatcher），监听 Skill 文件夹。一旦脚本保存，立即刷新 Agent 的工具字典。

### 5. RAG 知识库的“影子”索引 (Phase 3.6)
**痛点**：UE Python API 庞杂且版本变动频繁，AI 经常生成 4.27 时代的过期代码。
- **微观建议**：建立 **Version-Specific API Snippets**。
- **实现细节**：
    - 不要索引整份文档。优先索引 `Engine/Intermediate/PythonStub/unreal.py`（这是当前版本最准确的存根）。
    - 针对常用操作（如材质修改、关卡流送），人工标注 50 个左右的高质量 **黄金 Prompt 模版** 放入向量库，RAG 检索时优先匹配模版而非原始 API。
