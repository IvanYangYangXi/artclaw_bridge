# UE Editor Agent 开发路线图 (Development Roadmap)

## 阶段 0：环境预设与工程初始化 (Environment & Bootstrap)
**目标**：构建稳定的插件开发骨架，解决 Python 依赖隔离与 UE 内部环境的兼容性问题。

- [ ] **0.1 插件目录与模块骨架 (Plugin Scaffold)**
    - 创建 `UEEditorAgent` 插件，配置 `Editor` 模块及其 `uplugin` 定义。
    - 建立 `Content/Python` 目录，并将其加入 UE 的 Python 搜索路径。
    - 安装并配置 OpenClaw 客户端，确保其能识别并连接到 UE 插件暴露的 MCP 接口。
    - **操作**：创建 `UEEditorAgent.uplugin`，启用 `CanContainContent`。
    - **细节**：声明依赖 `PythonScriptPlugin` 和 `EditorScriptingUtilities`；设置加载阶段为 `PostEngineInit`。
- [ ] **0.2 C++ 编辑器子系统 (UUEAgentSubsystem)** 
    - 建立一个全局唯一的单例对象，作为 OpenClaw 连接状态的“真值来源”，并协调 C++ 与 Python 的通讯。
    - **自动生命周期**：继承自 `UEditorSubsystem` 的类会在编辑器启动时自动创建，关闭时自动销毁，无需手动管理内存。
    - **状态中心**：C++ 顶部的工具栏图标颜色（绿/红）可以直接绑定到这个 Subsystem 的 `bIsConnected` 变量。
    - **Python 访问桥梁**：Python 可以通过 `unreal.get_editor_subsystem(unreal.UEAgentSubsystem)` 轻松获取该单例，从而同步连接状态。
    - **操作**：继承 `UEditorSubsystem` 创建全局单例类。
    - **细节**：
        - 接口：`SetConnectionStatus(bool)` (供 Python 调用)。
        - 属性：`bIsConnected` (作为插件状态的唯一真值)。
        - 委托：`OnConnectionStatusChanged` (用于驱动工具栏图标颜色切换)。
- [ ] **0.3 开发者菜单与入口 (Developer UI Entrance)**
    - 在 UE 顶部菜单栏（或工具栏）注册 Agent 插件图标。
    - 实现点击图标弹出简单的“Agent 状态检查”原生 Slate 窗口。
    - **操作**：使用 `UToolMenus` 在编辑器顶部注入图标。
    - **细节**：图标颜色绑定到 `UUEAgentSubsystem` 的状态（断开为灰/红，连接成功为绿）。
- [ ] **0.4 依赖隔离管理机制 (Dependency Management)**
    - 实现插件启动时的 `site-packages` 检测逻辑。
    - 编写自动化脚本，支持在插件目录下私有安装 `websockets`, `pydantic` 等必要库，避免污染引擎自带的 Python 环境。
    - **操作**：在插件启动时检测 `site-packages` 目录。
    - **细节**：若缺失 `websockets` 或 `pydantic`，自动调用 `pip install --target` 定向安装至插件内部目录，确保不污染引擎环境。
- [ ] **0.5 日志与调试重定向 (Logging System)**
    - 将 Python 层的标准输出（STDOUT/STDERR）重定向至 UE `Output Log`。
    - 定义 `LogUEAgent` 分类，实现带颜色区分的 AI 指令执行日志。
    - **操作**：编写 `init_unreal.py`。
    - **细节**：
        - 将插件私有库路径加入 `sys.path`。
        - 重定向 Python `stdout/stderr` 至 UE `Output Log`，并添加 `LogUEAgent` 前缀。

**🚩 里程碑**：
1. 插件在虚幻引擎中成功加载且无报错。
2. 插件能自动识别并补全运行所需的第三方 Python 库。
3. 点击插件图标能看到反馈，且 Output Log 能实时打印 AI 相关的调试信息。


## 阶段 1：开放网关与安全沙盒 (Gateway & Safety Sandbox)
**目标**：实现“万能执行器”，在保持高度自由（直接运行 Python）的同时，通过机制保障引擎不 Crash。

- [ ] **1.1 MCP WebSocket 通信网关**
    - 实现基于 `websockets` 库的服务器。
    - **细节**：支持自动端口探测（8080+）与断线自动重置。
- [ ] **1.2 万能执行器 (Universal Proxy)**
    - 开发核心 Tool `run_ue_python(code: str)`。
    - **细节**：使用 `exec()` 函数，并内置 `try-except` 捕获所有 Python 层异常，将 Traceback 转为字符串返回给 AI。
- [ ] **1.3 静态指令预审 (Static Guard)**
    - **操作**：使用 Python `ast` (抽象语法树) 模块扫描代码。
    - **细节**：在执行前拦截黑名单（如 `os.system`, `subprocess`, `__import__`），从源头切断系统级破坏。
- [ ] **1.4 事务保护 (Undo Guard)**
    - **操作**：强制 `ScopedEditorTransaction` 包装。
    - **细节**：确保 AI 生成的临时 Python 变量不污染持久数据，且所有场景变更一键撤销。
- [ ] **1.5 主线程调度器 (GameThread Marshalling)** 
    - **操作**：利用 `unreal.get_interpreter().execute_on_main_thread()`。
    - **细节**：解决 WebSocket 线程调用 `unreal` API 导致的即时崩溃问题。
- [ ] **1.6 快捷上下文注入 (Context Injection)** 
    - **操作**：在 `exec` 的 `globals` 参数中预置常用对象。
    - **细节**：自动注入 `S` (选中的 Actor 列表), `W` (当前世界), `L` (常用库简写)，极大减少 AI 编写的代码量和出错率。

**🚩 里程碑**：
1. OpenClaw 发送一段复杂的 `unreal` 代码（如：循环生成 10 个球体并随机变色）。
2. UE 成功执行，Output Log 打印完整执行链路。
3. 如果代码语法错误，AI 能收到报错并自动修正再次发送。
4. 全程无闪退，按下 Ctrl+Z 场景瞬间恢复原状。

# 阶段 2：感知增强与原生交互 (Perception & Native UI)
**目标**：解决 AI 的“盲目性”与“高危性”，通过实时上下文反馈和原生 UI 拦截，建立安全、闭环的人机协作环境。

- [ ] **2.1 编辑器内对话面板 (In-Editor Chat Tab)**
    - **操作**：通过 C++ 在 Slate 中嵌入 `SWebBrowser` 控件。
    - **细节**：实现面板的停靠（Docking）功能，并同步 UE 编辑器的主题配色（背景色、滚动条样式），确保 UI 融合度。

- [ ] **2.2 动态上下文感知与资源映射 (Resources)**
    - **操作**：将 UE 编辑器状态映射为 MCP 资源 URI（如 `unreal://level/selection`）。
    - **细节**：AI 通过读取资源实时获取当前选中项、活动资产路径及视口相机位置。

- [ ] **2.3 风险分级确认 UI (Risk-Aware Confirmation)**
    - **操作**：C++ 实现原生 Slate 模态对话框。
    - **细节**：对高危操作（包含 `delete` 关键字、修改超过 50 个 Actor、或保存资产）强制触发物理确认。

- [ ] **2.4 视口同步与视觉反馈 (Viewport Feedback)**
    - **操作**：实现 `focus_on_actor` 和 `highlight_actors` 工具。
    - **细节**：AI 执行脚本后，自动高亮受影响的 Actor，并将视口相机平滑平移至目标区域。

- [ ] **2.5 编辑器模式智能过滤 (Context Filter)**
    - **操作**：实时监测用户是在材质编辑器、蓝图编辑器还是关卡编辑器。
    - **细节**：根据当前模式动态调整 AI 的 System Prompt，使其生成的代码优先使用当前环境下的 API。

- [ ] **2.6 运行时数据精简 (Data Pruning)** 
    - **操作**：实现 Actor 属性过滤器。
    - **细节**：将复杂的 `UObject` 转换为极简 JSON（仅保留 Name, Class, Transform, Tags）。
    - **原因**：防止将海量无效底层属性传给 AI 导致 Token 溢出或 AI 产生 API 调用幻觉。

- [ ] **2.7 错误感知与自修复循环 (Self-Healing Loop)** 
    - **操作**：将 `run_ue_python` 的执行报错（Traceback）自动作为上下文回传。
    - **细节**：在 Prompt 中引导 AI：“如果执行失败，请根据返回的错误日志分析原因，并生成修复后的代码重新执行”。

**🚩 里程碑**：
1. **感知**：AI 能够主动“看到”你选中的资产，不再需要你手动输入路径。
2. **安全**：AI 生成删除脚本时，UE 界面立即弹出确认框：“AI 计划删除 3 个物体，确认吗？”。
3. **反馈**：AI 修改材质后，视口自动对焦该物体并闪烁提示，用户无需寻找改动位置。

# 阶段 3：自进化 Skill 与 RAG 知识库 (Intelligence & Optimization)
**目标**：构建本地知识底座，解决 AI 编写 API 时的幻觉问题，并实现跨端记忆同步与版本兼容，使 Agent 具备专业开发者的认知深度。

- [ ] **3.1 Skill 热加载器 (Dynamic Skill Hub)**
    - **操作**：实现对 `Plugins/UEAgent/Skills/` 目录的实时文件监控。
    - **细节**：利用 Python `importlib.reload` 机制。保存 `.py` 脚本后，UE 内部的 AI 立即获得新技能，无需重启引擎。

- [ ] **3.2 本地 RAG 知识库集成 (Local Knowledge RAG)**
    - **操作**：集成轻量级向量库（如 `faiss-cpu` 或 `ChromaDB`）。
    - **细节**：索引 UE Python API 官方文档、项目特定的命名规范（Naming Convention）及常用代码范例（Snippets）。

- [ ] **3.3 语义检索与 Prompt 动态注入 (Smart Retrieval)**
    - **操作**：在 `run_ue_python` 执行前，根据用户指令语义检索相关 API 片段。
    - **细节**：将检索到的正确代码范例作为 `Context` 喂给 OpenClaw，极大提高“万能执行器”编写代码的一次性成功率。

- [ ] **3.4 分层记忆与偏好存储 (Tiered Memory Management)** 🧩 *深度整合*
    - **操作**：在项目 `Saved/UEAgent/` 目录下建立结构化 JSON 存储。
    - **细节**：
        - **事实记忆**：记录本项目常用材质路径、Actor 命名后缀等“硬知识”。
        - **用户偏好**：通过 MCP 接口 `update_user_preference` 记录用户对光照强度、布局间距的个人审美偏好。
        - **跨端同步**：将本地总结出的“项目规范”通过 MCP Resource 定时同步给 OpenClaw 的向量记忆，确保 AI “入乡随俗”。

- [ ] **3.5 接口影子化与版本兼容层 (API HAL)**
    - **操作**：编写 `ue_version_adapter.py` 适配层。
    - **细节**：针对 5.3/5.4/5.5 差异化 API（如物理模拟、渲染设置等接口变动）进行抽象包装，确保 AI 生成的代码能跨版本运行。

- [ ] **3.6 向量化索引自动化 (Auto-Indexing Pipeline)**
    - **操作**：编写文档预处理工具。
    - **细节**：支持一键将项目 Wiki、美术规范（Markdown/PDF）转换为向量索引。AI 遇到模糊指令时会主动说：“根据项目规范，此资产应归类到 XXX 文件夹...”。

**🚩 里程碑**：
1. **进化能力**：开发者将一个新脚本丢入文件夹，AI 立即学会了如何“一键清理关卡冗余光影”。
2. **准确性**：AI 不再猜 API，它会说：“在 5.5 版本中我应使用 X 接口，且符合项目 SM_ 前缀规范”。
3. **长期记忆**：重启编辑器后，AI 依然记得你之前对“赛博朋克风灯光颜色”的调整偏好。

# 阶段 4：分发、优化与生态 (Phase 4: Ecosystem & Polish)
**目标**：降低部署门槛，建立标准化的技能分发体系，确保在大型团队研发环境下的稳定性与可追溯性。

- [ ] **4.1 OpenClaw 技能包标准封装 (Skill Packaging)**
    - **操作**：定义标准的 `manifest.json` 与插件结构。
    - **细节**：制作包含 **System Prompt 增强、MCP 配置、Python 逻辑、预览图** 的完整 Skill 包，实现“一键导入 OpenClaw”。

- [ ] **4.2 性能监控与全量审计日志 (Performance & Audit)**
    - **操作**：建立 `Saved/Logs/UEAgent_Audit.log`。
    - **细节**：记录 AI 执行的**每一行原始代码**、受影响的 **Actor UID** 及**撤销状态**。提供性能热图，分析哪些 AI 指令最耗时（如海量资产扫描）。

- [ ] **4.3 团队分发与 Git 同步机制 (Team Collaboration)**
    - **操作**：适配 `Plugins/UEAgent/Skills` 的 Git 工作流。
    - **细节**：通过 `.gitignore` 保护本地 Memory，但同步核心 Skill 库。实现插件启动时的 **“冲突检测”**：若本地 Skill 与团队版本冲突，弹出交互界面由用户选择。

- [ ] **4.4 社区共享与远程更新接口 (ClawHub Integration)**
    - **操作**：对接 **ClawHub** 或企业内部静态服务器。
    - **细节**：支持在 UE 插件面板内直接浏览、预览并一键下载远程 Skill 包。支持 **热更新** 机制，无需拉取 Git 即可更新 AI 的逻辑脚本。

- [ ] **4.5 极简一键部署方案 (One-Click Deploy)** 🧩 *新增*
    - **操作**：制作 `setup.bat/sh` 引导脚本。
    - **细节**：针对非开发人员（如美术/策划），一键完成 **UE 插件安装 + OpenClaw 自动关联 + 离线 Python 依赖解压**。

- [ ] **4.6 稳定性“体检”工具 (Health Check)** 🧩 *新增*
    - **操作**：在插件面板提供“环境诊断”按钮。
    - **细节**：自动检测端口占用、WebSocket 延迟、Python 路径污染及 RAG 索引完整性，生成诊断报告。

**🚩 里程碑**：
1. **开箱即用**：新入职的美术人员只需运行一个脚本，即可在 UE 里唤起 AI 助手。
2. **安全合规**：所有 AI 行为可追溯至具体的 Python 代码行，满足大厂审计需求。
3. **生态联动**：团队沉淀的“一键场景灯光优化”技能可瞬间同步给所有关卡设计师。

## 进度管理建议
1. **周迭代**：每周完成 1-2 个子任务，优先保证 **1.1** 和 **1.2** 的稳定性。
2. **文档同步**：每完成一个核心 Tool，自动更新其在 OpenClaw 端的系统提示词（System Prompt）。
3. **安全第一**：在阶段 2 完善之前，限制 AI 对磁盘物理文件的写权限。
