# UE Editor Agent 开发路线图 (Development Roadmap)

## 阶段 0：环境预设与工程初始化 (Environment & Bootstrap)
**目标**：构建稳定的插件开发骨架，解决 Python 依赖隔离与 UE 内部环境的兼容性问题。

- [ ] **0.1 插件目录与模块骨架 (Plugin Scaffold)**
    - 创建 `UEEditorAgent` 插件，配置 `Editor` 模块及其 `uplugin` 定义。
    - 建立 `Content/Python` 目录，并将其加入 UE 的 Python 搜索路径。
- [ ] **0.2 依赖隔离管理机制 (Dependency Management)**
    - 实现插件启动时的 `site-packages` 检测逻辑。
    - 编写自动化脚本，支持在插件目录下私有安装 `websockets`, `pydantic` 等必要库，避免污染引擎自带的 Python 环境。
- [ ] **0.3 开发者菜单与入口 (Developer UI Entrance)**
    - 在 UE 顶部菜单栏（或工具栏）注册 Agent 插件图标。
    - 实现点击图标弹出简单的“Agent 状态检查”原生 Slate 窗口。
- [ ] **0.4 日志与调试重定向 (Logging System)**
    - 将 Python 层的标准输出（STDOUT/STDERR）重定向至 UE `Output Log`。
    - 定义 `LogUEAgent` 分类，实现带颜色区分的 AI 指令执行日志。

**🚩 里程碑**：
1. 插件在虚幻引擎中成功加载且无报错。
2. 插件能自动识别并补全运行所需的第三方 Python 库。
3. 点击插件图标能看到反馈，且 Output Log 能实时打印 AI 相关的调试信息。


## 阶段 1：核心基础设施构建 (Phase 1: Foundation)
**目标**：打通 OpenClaw 与 UE 的双向通信，实现首个“Hello World”式场景操作。

- [ ] **1.1 MCP 通信网关开发 (C++/Python)**
    - 在 UE 插件中集成 WebSocket Server。
    - 实现 MCP 协议的基础握手与 JSON-RPC 消息路由。
- [ ] **1.2 Python 自动映射装饰器**
    - 编写 `@ue_agent.tool` 装饰器。
    - 实现基于 `inspect` 的函数签名向 MCP Schema 的自动转换。
- [ ] **1.3 基础原子工具集 (Atomic Tools)**
    - 封装 `spawn_actor`, `get_selected`, `set_transform` 等核心 Python 接口。
- [ ] **1.4 Undo/Redo 事务集成**
    - 确保所有 MCP 调用自动包裹在 `ScopedEditorTransaction` 中。

**🚩 里程碑**：用户在 OpenClaw 对话框输入指令，UE 场景中即时生成并移动方块，且支持 Ctrl+Z 撤销。

---

## 阶段 2：感知能力与原生交互 (Phase 2: Perception & UI)
**目标**：让 AI “看得见”场景内容，并能通过 UE 原生界面与用户确认。

- [ ] **2.1 编辑器内对话界面 (In-Editor UI) 集成计划**
    - 在完成基础通信（阶段 1）后，立即启动编辑器内 UI 开发，开发者可以直接在 UE 内部看到反馈。
- [ ] **2.2 MCP Resources 实时检索**
    - 实现场景大纲（Outliner）与资产浏览器（Content Browser）的资源映射。
    - 开发按需加载（Lazy Loading）机制，防止海量资产导致通信卡顿。
- [ ] **2.3 原生 UI 管理器 (Native UI Manager)**
    - C++ 层实现标准的 Slate/UMG 弹出确认框。
    - 提供 Python 异步回调接口，支持 AI 操作的“二次确认”流。
- [ ] **2.4 错误感知与自修复**
    - 捕获 Python 运行时异常并将其作为 MCP Error 返回给 LLM，引导 AI 修正指令。

**🚩 里程碑**：AI 能识别选中物体的名称和材质，并在修改关键参数前弹出 UE 原生对话框请求许可。

---

## 3. 动态 Skill 系统与高级功能 (Phase 3: Advanced Skills)
**目标**：实现能力的模块化扩展，突破 Python 性能限制。

- [ ] **3.1 Skill 热加载器 (Skill Hub)**
    - 实现文件系统监听，自动 `reload` 变更的 Python 模块。
    - 定义标准的 `manifest.json` 结构。
- [ ] **3.2 C++ 性能扩展层**
    - 封装 C++ 接口处理高频任务（如批量修改 1000+ 物体属性）。
    - 穿透 Python 无法访问的 RHI/贴图通道底层 API。
- [ ] **3.3 复杂工作流 Skill 示例**
    - 开发“一键程序化森林散布”或“自动化关卡灯光配置”参考 Skill。

**🚩 里程碑**：开发者无需修改插件源码，仅通过放入一个文件夹即可为 Agent 增加全新的行业专用技能。

---

## 4. 分发、优化与生态 (Phase 4: Ecosystem & Polish)
**目标**：降低部署门槛，支持团队协作与技能分享。

- [ ] **4.1 OpenClaw 插件包封装**
    - 制作符合 OpenClaw 规范的发布包，包含提示词模板与 MCP 配置。
- [ ] **4.2 性能监控与日志**
    - 建立操作审计日志，记录 AI 所有的资产变更路径。
- [ ] **4.3 团队分发机制**
    - 适配 Git 工作流，确保 Skill 库在多人协作项目中自动同步。
- [ ] **4.4 社区共享接口**
    - 对接 ClawHub，实现 Skill 的在线下载与版本更新。

**🚩 里程碑**：发布首个稳定版本 .uplugin，支持在不同机器上通过简单配置即可激活 AI 辅助开发环境。

---

## 进度管理建议
1. **周迭代**：每周完成 1-2 个子任务，优先保证 **1.1** 和 **1.2** 的稳定性。
2. **文档同步**：每完成一个核心 Tool，自动更新其在 OpenClaw 端的系统提示词（System Prompt）。
3. **安全第一**：在阶段 2 完善之前，限制 AI 对磁盘物理文件的写权限。
