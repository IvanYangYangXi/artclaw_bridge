## 🎯 测试环境要求

### 硬件环境
- Windows 10/11 64位操作系统
- 16GB RAM及以上
- 支持DirectX 12的显卡

### 软件环境
- Unreal Engine 5.7
- Python 3.9+ (UE内置)
- websockets库 (已包含在插件中)

### 前置条件
- [ ] UE项目已配置UEEditorAgent插件
- [ ] 插件已编译并通过
- [ ] PythonScriptPlugin已启用
- [ ] EditorScriptingUtilities已启用

#### 2.2 端口探测测试

**测试目的**: 验证端口占用时能自动切换到可用端口

**测试步骤**:
1. 打开命令行，运行: `netstat -ano | findstr :8080`
2. 如果8080被占用，记录PID并终止进程，或修改测试用例
3. 如果8080空闲，手动启动一个程序占用8080端口
4. 重启UE编辑器
5. 观察Output Log中的端口选择日志

**预期结果**:
- ✅ 如果8080被占用，日志显示尝试8081
- ✅ 如果8081也被占用，继续尝试8082、8083...
- ✅ 最终使用第一个可用端口
- ✅ Dashboard中显示正确的服务器地址

**测试状态**: ⬜ 未测试 | ⬜ 通过 | ⬜ 失败

- ✅ 客户端断开后，Dashboard显示 "Status: Offline"（红色）
- ✅ 状态切换无延迟，实时刷新

#### 2.4 消息回显测试

**测试目的**: 验证AI客户端发送的消息能被正确处理并返回响应

**测试步骤**:
1. 启动UE编辑器，等待MCP服务器启动
2. 使用WebSocket测试工具连接到 ws://localhost:8080 (或其他端口)
3. 发送测试消息: `{"jsonrpc":"2.0","id":1,"method":"test"}`
4. 观察Output Log和响应结果
5. 发送多条消息，验证稳定性

**预期结果**:
- ✅ 可以成功连接到WebSocket服务器
- ✅ 发送消息后，Output Log显示 "[MCPServer] 收到消息: {...}"
- ✅ 收到响应: `{"jsonrpc":"2.0","id":1,"result":{"status":"received","message":"UE Editor 已收到: test"}}`
- ✅ 响应的id与请求一致
- ✅ 多次发送消息都能正确处理

**测试状态**: ⬜ 未测试 | ⬜ 通过 | ⬜ 失败  
**备注**:

#### 2.5 断线重连测试

**测试目的**: 验证断线后无需重启服务器，客户端可以重新连接

**测试步骤**:
1. 启动UE编辑器，等待MCP服务器启动
2. 使用WebSocket测试工具连接
3. 验证连接状态和消息通信正常
4. 关闭WebSocket测试工具（模拟意外断开）
5. 等待1-2秒
6. 重新打开WebSocket测试工具并连接
7. 验证是否可以重新连接

**预期结果**:
- ✅ 第一次连接成功
- ✅ 通信正常
- ✅ 关闭客户端后，服务器日志显示 "AI客户端断开连接"
- ✅ 重新连接成功，无需重启UE编辑器
- ✅ 第二次连接后通信仍然正常

**测试状态**: ⬜ 未测试 | ⬜ 通过 | ⬜ 失败
