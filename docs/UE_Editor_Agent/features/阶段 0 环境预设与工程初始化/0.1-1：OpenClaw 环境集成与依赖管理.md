## 阶段 0.1-1：OpenClaw 环境集成与依赖管理 (OpenClaw & Dependencies)
**目标**：安装并配置 OpenClaw 客户端，确保其能识别并连接到 UE 插件暴露的 MCP 接口。

- [ ] **1. 安装 OpenClaw 桌面端/CLI**
    - **操作**：前往 [OpenClaw 官网](https://openclaw.io) 或 GitHub 下载最新的 OpenClaw 客户端。
    - **细节设置**：安装完成后，通过命令行执行 `claw --version` 确认安装成功。
- [ ] **2. 配置本地 MCP 测试环境**
    - **操作**：在 OpenClaw 的配置文件（通常是 `openclaw.config.json`）中预留 UE 插件的连接信息。
    - **细节设置**：
        - 传输协议选择：`WebSocket` (因为 UE 编辑器是持久运行的，长连接最稳定)。
        - 默认地址：`ws://127.0.0.1:8080` (这是我们在阶段 1.1 中要实现的端口)。
- [ ] **3. 插件私有 Python 依赖库准备**
    - **操作**：在 UE 插件目录 `Content/Python/` 下创建一个 `requirements.txt`。
    - **细节设置**：
        - 必须包含：`websockets` (用于 MCP 通信), `pydantic` (用于数据模型校验)。
        - 进阶可选：`aiohttp` (如果需要访问外部网络 API)。
- [ ] **4. 编写依赖自动安装脚本**
    - **操作**：在 `init_unreal.py` 中编写逻辑，检查插件目录是否已存在 `site-packages`。
    - **细节设置**：若缺失依赖，自动调用 `subprocess.run([py_path, "-m", "pip", "install", "-r", ...])` 定向安装到插件内部目录。

**🚩 里程碑**：
1. OpenClaw 客户端能够正常启动并进入等待连接状态。
2. UE 插件启动时，自动在插件目录下生成 `site-packages` 文件夹并装载所需库。
3. Python `sys.path` 已正确包含插件私有依赖路径，执行 `import websockets` 不报错。
