# UE Editor Agent - 依赖隔离管理实现 (Phase 0.2)

## 1. 核心逻辑实现 (init_unreal.py)
将以下代码放置在插件的 `Content/Python/init_unreal.py` 中。它会在 UE 启动时自动检查并补全环境。

```python
import unreal
import os
import sys
import subprocess

def setup_plugin_dependencies():
    # 1. 获取当前插件的 Python 目录
    # 假设插件名为 UEEditorAgent
    plugin_python_dir = os.path.dirname(__file__)
    site_packages_dir = os.path.join(plugin_python_dir, "site-packages")
    requirements_path = os.path.join(plugin_python_dir, "requirements.txt")

    # 2. 将私有 site-packages 加入 sys.path
    if site_packages_dir not in sys.path:
        sys.path.append(site_packages_dir)

    # 3. 检查核心库是否已安装 (以 websockets 为例)
    try:
        import websockets
        import pydantic
        unreal.log("UEEditorAgent: 所有依赖库已就绪。")
    except ImportError:
        unreal.log_warning("UEEditorAgent: 缺失依赖，准备开始自动安装...")
        
        # 4. 获取 UE 内置的 Python 解析器路径
        python_exe = sys.executable
        
        # 5. 执行静默安装到目标目录 (--target)
        try:
            subprocess.check_call([
                python_exe, "-m", "pip", "install", 
                "--target", site_packages_dir, 
                "-r", requirements_path
            ])
            unreal.log("UEEditorAgent: 依赖安装成功，请重启编辑器或手动重载 Python。")
        except Exception as e:
            unreal.log_error(f"UEEditorAgent: 自动安装失败: {str(e)}")

# 执行初始化
setup_plugin_dependencies()
```

## 2. 配置文件准备 (requirements.txt)
在 `Content/Python/` 目录下创建 `requirements.txt`，写入以下内容：

```text
websockets>=12.0
pydantic>=2.0
```

## 3. 技术细节说明
- **`--target` 参数**：这是实现隔离的关键。它告诉 `pip` 不要安装到全局目录，而是安装到插件指定的 `site-packages` 文件夹。
- **`sys.path.append`**：确保 UE 的 Python 解释器在查找库时，优先查找我们插件内部的目录。
- **持久化**：一旦安装完成，下次启动将直接跳过安装逻辑，不影响编辑器启动速度。

**🚩 验证操作**：
1. 重启虚幻引擎。
2. 查看 **Output Log**（输出日志）。
3. 如果看到 `UEEditorAgent: 依赖安装成功`，检查插件目录是否多出了一个 `site-packages` 文件夹。

---
**提示**：在某些受限的内网环境下，自动安装可能会失败。此时可以手动将本地安装好的库复制到 `site-packages` 中，该机制依然有效。
