## 阶段 0.4：日志与调试重定向 (Logging System)
我们将通过 Python 的 sys.stdout 重定向机制，将 AI 智能体的所有后台逻辑（如 MCP 协议的 JSON 交互、Python 报错、OpenClaw 的指令回馈）实时同步到虚幻引擎的 Output Log 面板中。
如果没有这一步，AI 的报错将只能在外部命令行查看，极大影响开发效率。

**目标**：将 Python 运行环境的输出与虚幻引擎原生日志系统深度绑定，实现透明化调试。

- [ ] **0.4.1 自定义 Log 类别注册 (C++)**
    - **操作**：在 `UEEditorAgent.h` 中定义 `DECLARE_LOG_CATEGORY_EXTERN(LogUEAgent, Log, All);`。
    - **细节设置**：在 `.cpp` 中使用 `DEFINE_LOG_CATEGORY(LogUEAgent);` 实例化。这能让日志在 Output Log 中拥有独立的颜色和分类过滤器。
- [ ] **0.4.2 Python 流重定向 (Redirector)**
    - **操作**：在 `init_unreal.py` 中编写一个继承自 `io.TextIOBase` 的类。
    - **细节设置**：重写 `write` 方法，内部调用 `unreal.log()`。
- [ ] **0.4.3 区分日志级别**
    - **操作**：将 Python 的 `print` 映射为 `Log`，将 `warnings.warn` 映射为 `Warning`，将 `Exception` 堆栈映射为 `Error`。
- [ ] **0.4.4 MCP 通信截获 (Interceptor)**
    - **操作**：在 WebSocket 收到消息时，强制打印 `[OpenClaw >> UE] {json_content}`。
    - **细节设置**：方便在不打开外部抓包工具的情况下，直接在 UE 内部观察 AI 的思考路径。

## 核心代码实现 (init_unreal.py 增强版)

```python
import unreal
import sys

class UELogRedirector:
    """将 Python 的 print 输出重定向到 UE Output Log"""
    def write(self, message):
        # 过滤掉纯换行符，避免日志出现大量空行
        clean_msg = message.strip()
        if clean_msg:
            # 使用 unreal.log 将内容推送到 UE 日志系统
            # 也可以根据前缀判断使用 unreal.log_error 或 unreal.log_warning
            unreal.log(f"LogUEAgent: [Python] {clean_msg}")

    def flush(self):
        pass

# 替换标准输出和错误输出
sys.stdout = UELogRedirector()
sys.stderr = UELogRedirector()

unreal.log("LogUEAgent: Python 日志重定向已激活。")
```

**🚩 里程碑**：
1. 在 Python 脚本中执行 `print("Hello AI")`，UE 的 Output Log 立即出现 `LogUEAgent: [Python] Hello AI`。
2. 当 Python 脚本发生语法错误或运行时崩溃时，完整的堆栈信息会以**红色字体**显示在 UE 日志中。
3. 用户可以在 Output Log 的过滤器（Filters）中单独勾选 `LogUEAgent`，屏蔽引擎其他无关干扰信息。

---
**提示**：在 UE 5.4+ 中，建议配合 `unreal.log_warning()` 使用，因为 AI 生成的代码有时会触发非致命警告，将其标黄有助于快速定位潜在的逻辑风险。
