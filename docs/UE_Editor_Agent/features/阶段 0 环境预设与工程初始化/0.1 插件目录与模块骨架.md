## 阶段 0.1：插件目录与模块骨架 (Plugin Scaffold)
**目标**：构建一个标准的 UE 5.x 混合插件，确保 C++ 模块能拉起 Python 环境，并为 MCP 通信预留接口。

- [ ] **1. 插件定义与模块配置 (.uplugin)**
    - **操作**：创建 `UEEditorAgent.uplugin` 文件。
    - **细节设置**：
        - `Type`: 设置为 `Editor`。
        - `LoadingPhase`: 必须设为 `PostEngineInit`。原因：Python 脚本系统在引擎初始化后期才完全就绪，过早加载会导致 Python API 调用失败。
        - `Plugins`: 声明依赖 `PythonScriptPlugin` 和 `EditorScriptingUtilities`。

- [ ] **2. 目录结构规范 (Directory Structure)**
    - **操作**：手动建立以下核心目录。
    - **细节设置**：
        - `Source/UEEditorAgent/`: 存放 C++ 核心逻辑（WebSocket Server, Slate UI）。
        - `Content/Python/`: **关键目录**。UE 会自动将此目录加入 `sys.path`。
        - `Resources/`: 存放插件图标（Icon128.png）。

- [ ] **3. C++ 模块启动逻辑 (Module Startup)**
    - **操作**：在 `FUEEditorAgentModule::StartupModule()` 中注入逻辑。
    - **细节设置**：
        - **执行 Python 入口**：通过 `IPythonScriptPlugin::Get()->ExecPythonCommand(TEXT("import init_unreal"))` 触发 Python 层的初始化。
        - **单例初始化**：初始化一个全局单例（如 `UUEAgentSubsystem`），用于维护与 OpenClaw 的连接状态。

- [ ] **4. Python 初始化脚本 (init_unreal.py)**
    - **操作**：在 `Content/Python/` 下创建 `init_unreal.py`。
    - **细节设置**：
        - 这是 UE 官方指定的 Python 自动启动文件名。
        - 在此脚本中设置 `unreal.log("UE Editor Agent Python Initialized")`，用于验证环境。

- [ ] **5. 编辑器 UI 入口注册 (Menu/Toolbar Extension)**
    - **操作**：利用 `FUICommandList` 和 `FExtender` 在编辑器顶部菜单栏添加按钮。
    - **细节设置**：
        - 建议路径：`Window -> AI Tools -> UE Editor Agent`。
        - 关联一个简单的 `SCompoundWidget` (Slate)，作为后续对话界面的容器。

**🚩 里程碑**：
1. 启动 UE 后，在 **Output Log** 中能看到 `UE Editor Agent Python Initialized` 的字样。
2. 菜单栏出现插件图标，点击后能打开一个空白的插件面板。
3. `uplugin` 文件正确声明了对 Python 插件的强依赖，确保分发时不会因环境缺失报错。


关键细节补充：
在 UE 5.x 中，务必在 Project Settings -> Plugins -> Python Script Plugin 中勾选 "Enable Remote Execution"。虽然我们使用 MCP，但开启此选项能极大方便开发阶段的远程调试。
