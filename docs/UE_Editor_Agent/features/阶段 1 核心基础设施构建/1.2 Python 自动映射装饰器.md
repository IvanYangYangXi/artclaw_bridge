在 阶段 1.2，我们将实现整个插件的“灵魂”机制：自动能力发现。通过 Python 的装饰器和内省（Introspection）技术，你可以直接将普通的 Python 函数转化为 OpenClaw 大脑可识别的 MCP Tool，无需手动编写复杂的 JSON 定义

# 阶段 1.2：Python 自动映射装饰器 (Auto-Mapping Decorator)
**目标**：通过装饰器自动提取 Python 函数的参数、类型和文档字符串，生成符合 MCP 标准的 Tool Schema，并实现自动分发。

- [ ] **1.2.1 定义 Tool 注册表 (Registry)**
    - **操作**：创建一个全局单例类 `ToolRegistry`，用于存储所有被标记的函数及其元数据。
    - **细节设置**：使用字典结构 `{ "tool_name": function_object }` 映射，方便后续根据 MCP 指令快速查找执行。
- [ ] **1.2.2 编写 `@mcp_tool` 装饰器**
    - **操作**：利用 Python 的 `functools.wraps` 保持原始函数签名。
    - **细节设置**：在函数定义阶段，自动触发 `inspect` 模块解析参数，并将其存入注册表。
- [ ] **1.2.3 类型转换器 (Type Mapper)**
    - **操作**：将 Python 类型（`int`, `str`, `float`, `bool`）映射为 JSON Schema 类型。
    - **细节设置**：支持 `unreal.Vector` 到 `{"x": 0, "y": 0, "z": 0}` 的自定义结构化映射。
- [ ] **1.2.4 动态生成 ListTools 响应**
    - **操作**：当 OpenClaw 发送 `list_tools` 请求时，遍历注册表生成完整的 JSON 数组。

## 核心实现代码 (tool_manager.py)

```python
import inspect
import functools
import unreal

class ToolRegistry:
    _tools = {}

    @classmethod
    def register_to_mcp(cls, name=None):
        def decorator(func):
            tool_name = name or func.__name__
            # 使用 inspect 获取函数签名和文档
            sig = inspect.signature(func)
            doc = func.__doc__ or "无描述"
            
            # 构建 MCP 标准的 JSON Schema
            properties = {}
            required = []
            for param_name, param in sig.parameters.items():
                # 简单的类型映射逻辑
                py_type = param.annotation
                json_type = "string"
                if py_type == int: json_type = "integer"
                elif py_type == float: json_type = "number"
                elif py_type == bool: json_type = "boolean"
                
                properties[param_name] = {
                    "type": json_type,
                    "description": f"参数 {param_name}" # 可通过解析 docstring 进一步细化
                }
                if param.default == inspect.Parameter.empty:
                    required.append(param_name)

            cls._tools[tool_name] = {
                "func": func,
                "schema": {
                    "name": tool_name,
                    "description": doc.strip(),
                    "inputSchema": {
                        "type": "object",
                        "properties": properties,
                        "required": required
                    }
                }
            }
            unreal.log(f"LogUEAgent: 已注册 MCP 工具 -> {tool_name}")
            return func
        return decorator

    @classmethod
    def get_all_schemas(cls):
        return [tool["schema"] for tool in cls._tools.values()]

    @classmethod
    def call_tool(cls, name, arguments):
        if name in cls._tools:
            # 这里的调用需要注意主线程安全（详见 1.3）
            return cls._tools[name]["func"](**arguments)
        raise ValueError(f"Tool {name} not found")
```

## 关键技术细节
1. **Docstring 解析**：建议使用 `docstring_parser` 库（可在阶段 0.2 加入 requirements）来自动从注释中提取每个参数的具体含义，这能显著提升 AI 使用工具的准确度。
2. **默认值处理**：如果 Python 函数定义了默认值，对应的 JSON Schema 应标记该参数为非必填（Optional）。
3. **热更新兼容**：在 **阶段 3.1 (Skill 热加载)** 中，当脚本重载时，需先清理 `_tools` 注册表，防止旧版本的函数残留在内存中。
