# 1.4 事务保护(Undo Guard)

**阶段**：阶段1 - 核心基础设施构建  
**重要性**：⭐⭐⭐⭐⭐ (P0)  
**依赖**：1.3 静态指令预审(Static Guard)  
**预估工时**：2-3天

---

## 🎯 目标与里程碑

### 核心目标
构建**事务保护系统**，将所有AI生成的操作包装在可撤销的事务中，实现**操作可回滚**，防止AI误操作导致的数据丢失。

### 验收标准
- [ ] 100% AI操作可被撤销（Ctrl+Z）
- [ ] 支持事务嵌套（最多5层）
- [ ] 批量操作原子性（全部成功或全部回滚）
- [ ] 崩溃恢复后仍可撤销（编辑器未关闭）
- [ ] 性能开销<10%（与直接操作对比）

---

## 📚 背景与上下文

### 为什么需要Undo Guard？

即使通过了Static Guard的代码，AI仍可能产生**逻辑错误**：

```python
# AI生成的代码可能逻辑错误
import unreal

# 错误：删除了所有静态网格体，而不是特定的那个
all_actors = unreal.EditorLevelLibrary.get_all_level_actors()
for actor in all_actors:
    if isinstance(actor, unreal.StaticMeshActor):
        actor.destroy_actor()  # 😱 删除了所有静态网格体！
```

没有Undo Guard的后果：
- 开发者只能手动重做所有工作
- 可能导致数小时工作丢失
- 降低开发者对AI助手的信任

### 设计原则

1. **透明化**：对AI代码无侵入，自动包装
2. **完整性**：捕获所有副作用（文件、资产、场景）
3. **可嵌套**：支持复杂操作的事务组合
4. **可审计**：记录所有操作日志

        actor = unreal.EditorLevelLibrary.spawn_actor_from_class(

    # 检查撤销栈

    # 测试2：异常回滚

            )

        pass
    
    # 检查Actor是否被回滚删除
    all_actors = unreal.EditorLevelLibrary.get_all_level_actors()
    # 应该没有新创建的Actor
    
    unreal.log("Undo Guard 测试通过!")

## ❓ 常见问题

### Q1: 事务回滚失败怎么办？
A:
1. 检查是否在事务中调用了非回滚操作（如文件IO）
2. 确保所有Actor操作都在事务内进行
3. 查看日志：`Saved/Logs/UndoGuard.log`

### Q2: 如何部分回滚？
A: UndoGuard不支持部分回滚。如需部分回滚，拆分为多个小事务：
```python
guard = UndoGuard()

guard.wrap_actor_operation("第一部分", op1)
if condition:
    guard.wrap_actor_operation("第二部分", op2)  # 这部分可单独撤销
```

### Q3: 事务嵌套太多导致性能下降？
A:
- 减少嵌套层数（最大5层）
- 合并小事务为大事务
- 禁用自动保存（`AutoSaveInterval=0`）
