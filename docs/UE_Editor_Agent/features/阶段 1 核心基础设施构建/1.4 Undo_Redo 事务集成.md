在 阶段 1.4，我们将为 Agent 的所有操作接入虚幻引擎的原生 撤销/重做（Undo/Redo） 系统。这是生产级插件的必备功能：它能确保 AI 的错误操作（如误删模型或错误的场景布局）可以被开发者一键恢复，从而极大提升 AI 在实际工作流中的安全性。

# 阶段 1.4：Undo/Redo 事务集成 (Transaction Guard)
**目标**：将 AI 的每个原子指令或 Skill 逻辑自动封装进 UE 的 `ScopedEditorTransaction`，确保所有变更均可回滚。

- [ ] **1.4.1 定义事务装饰器 (Transaction Decorator)**
    - **操作**：编写一个名为 `@with_transaction` 的 Python 装饰器。
    - **细节设置**：利用 `unreal.ScopedEditorTransaction` 上下文管理器，在函数执行前开启事务，执行后自动关闭。
- [ ] **1.4.2 动态生成事务名称 (Naming)**
    - **操作**：根据调用的 MCP Tool 名称和参数动态生成撤销记录的描述（如：“AI: 生成 Cube”）。
    - **细节设置**：方便用户在编辑器的 `Undo History` 窗口中清晰辨认哪些操作是由 AI 完成的。
- [ ] **1.4.3 异常回滚处理 (Rollback)**
    - **操作**：捕获 Python 执行过程中的任何异常。
    - **细节设置**：如果脚本报错，确保事务能被正确取消（Cancel），防止产生残留的半成品操作数据。
- [ ] **1.4.4 与主线程调度器集成 (Unified Integration)**
    - **操作**：将事务装饰器整合进 **1.3.1** 的 `run_on_main_thread` 流程中。

## 核心实现代码 (transaction_helper.py)

```python
import unreal
import functools

def with_transaction(note="AI Agent Operation"):
    """
    装饰器：自动为函数包裹 UE 撤销事务。
    注意：此装饰器必须在主线程内运行才有效。
    """
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            # 获取动态描述：尝试从参数中提取更多信息
            transaction_name = f"AI: {func.__name__}"
            
            # 开启 UE 事务上下文
            with unreal.ScopedEditorTransaction(transaction_name) as trans:
                try:
                    result = func(*args, **kwargs)
                    # 执行成功，事务会自动在退出 with 时提交
                    return result
                except Exception as e:
                    # 如果发生异常，事务通常会自动回滚（取决于 UE 版本）
                    # 显式记录日志以便调试
                    unreal.log_error(f"Transaction Failed: {str(e)}")
                    raise e
        return wrapper
    return decorator

# --- 综合应用示例 ---

@ToolRegistry.register_to_mcp(name="transform_actor")
@run_on_main_thread  # 先切回主线程
@with_transaction()  # 再开启事务
def transform_actor(name: str, x: float, y: float, z: float):
    """移动指定 Actor 到新位置并支持撤销。"""
    actor = unreal.EditorLevelLibrary.get_all_level_actors().filter(lambda a: a.get_name() == name)
    if actor:
        actor[0].set_actor_location(unreal.Vector(x, y, z), False, True)
        return f"Moved {name} to {x},{y},{z}"
    raise ValueError("Actor not found")
```

## 关键技术细节
1. **嵌套事务**：UE 支持嵌套事务，但为了清晰起见，建议一个 MCP Tool 指令对应一个顶层事务。
2. **UI 线程同步**：`ScopedEditorTransaction` **必须在 GameThread 执行**。这就是为什么 `@with_transaction` 必须放在 `@run_on_main_thread` 之后（内部）。
3. **撤销记录可见性**：执行完操作后，你可以打开 UE 菜单 `Edit -> Undo History` 观察。你应该能看到以 `AI:` 开头的操作记录。

**🚩 里程碑**：
1. 通过 OpenClaw 让 AI 生成一个物体，然后在 UE 编辑器中按 `Ctrl+Z`，物体应消失。
2. 通过 OpenClaw 执行一个批量操作（如移动 10 个物体），按一次 `Ctrl+Z` 应该能同时撤销这 10 个物体的位移。
3. 在 `Undo History` 面板中能清楚地看到 AI 操作的痕迹。
