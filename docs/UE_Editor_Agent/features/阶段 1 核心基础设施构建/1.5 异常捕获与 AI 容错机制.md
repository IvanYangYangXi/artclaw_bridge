当 AI 指令执行失败（如模型路径写错、坐标溢出）时，UE 内部会报错，但 OpenClaw 的 AI 却不知道发生了什么，会导致它陷入“复读机”或卡死状态。# 阶段 1.5：异常捕获与 AI 容错机制 (Error Handling & Feedback)
**目标**：构建闭环反馈，将 UE 内部的 Python/C++ 错误转化为 AI 能理解的“修改建议”，实现自动纠错。

- [ ] **1.5.1 结构化错误返回 (Structured Error)**
    - **操作**：定义标准的 MCP Error JSON 响应体。
    - **细节设置**：不只是返回 "Error"，而是返回 `{"code": 404, "message": "资产路径 /Game/NotExist 不存在", "suggestion": "请检查 Content Browser 路径"}`。
- [ ] **1.5.2 运行时权限校验 (Safety Sandbox)**
    - **操作**：在执行前检查指令是否超限。
    - **细节设置**：例如单次生成 Actor 数量限制在 100 以内，防止 AI 逻辑死循环导致 UE 内存溢出挂掉。
- [ ] **1.5.3 线程状态看门狗 (Watchdog)**
    - **操作**：监控主线程（GameThread）的响应耗时。
    - **细节设置**：如果 AI 请求的操作导致 UE 冻结（卡住超过 5 秒），强制中断当前 Python 协程并向 OpenClaw 发送 Timeout 警告。

## 核心实现代码 (error_handler.py)

```python
import unreal
import json

def safe_mcp_execute(func):
    """装饰器：捕获执行错误并包装为标准的 MCP 响应"""
    def wrapper(*args, **kwargs):
        try:
            result = func(*args, **kwargs)
            return {"status": "success", "data": result}
        except unreal.PythonException as pe:
            # 捕获 UE 特有的 Python 报错
            return {
                "status": "error",
                "type": "UnrealEngineError",
                "message": str(pe),
                "suggestion": "请确认引擎 API 调用参数或资产路径是否正确。"
            }
        except Exception as e:
            return {
                "status": "error",
                "type": "GeneralError",
                "message": str(e)
            }
    return wrapper
```

**🚩 里程碑**：
1.故意给 AI 下达一个错误的指令（如：把模型移动到太阳系外），AI 能收到 UE 返回的错误描述。
2.AI 根据返回的错误信息，能够自动回复：“抱歉，坐标超出范围，我已尝试为您修正至场景边界”。
3.即便脚本崩溃，UE 插件的主服务（WebSocket Server）依然运行正常。
