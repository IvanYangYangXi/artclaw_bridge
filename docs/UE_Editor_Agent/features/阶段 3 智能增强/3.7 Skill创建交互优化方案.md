# 3.7 Skill 创建交互优化方案

**版本**: 1.0  
**日期**: 2026-03-17  
**作者**: 小优 (ArtClaw AI)  
**状态**: ✅ 已完成

---

## 1. 背景与问题

### 1.1 当前流程

目前 UE 编辑器内创建 ArtClaw Skill 有两个入口：

1. **自然语言触发** — 在聊天框输入含 "artclaw" + 创建意图词 + "skill/技能" 的文本
2. **Create Skill 按钮** — 点击后弹出模态对话框

两种入口最终都走向 `OpenSkillCreationDialog()` → 模态对话框，要求用户手动填写：
- Description（必填）
- Category（选填）
- Software（选填，默认 unreal_engine）

填写完毕后调用 `StartSkillGeneration()` → Python `_handle_skill_generate()` → 基于关键词推断生成 scaffold。

### 1.2 现存问题

| # | 问题 | 影响 |
|---|------|------|
| P1 | **模态对话框打断工作流** — 用户需离开聊天上下文，填写表单式弹窗 | 体验割裂，不像在跟 AI 对话 |
| P2 | **信息应自动获取但要求手动填** — software 字段在 UE 里默认就是 unreal_engine，UE 版本也可自动检测 | 多余操作 |
| P3 | **缺少 AI 推断能力** — Skill 名称 (name)、display_name 需要 AI 自动生成而非用户填写 | 用户还要想 snake_case 命名 |
| P4 | **Category 输入不友好** — 文本框 + hint text，用户需要记忆枚举值 | 容易填错或留空 |
| P5 | **无多轮对话能力** — 如果 AI 需要补充信息（如 Skill 是否需要读写操作、目标软件是否非 UE），没有对话式追问机制 | 一次填写不够灵活 |
| P6 | **生成结果展示不够清晰** — 直接在聊天中打印代码片段，没有确认/编辑/拒绝的交互 | 用户没有"确认安装"的明确操作 |

---

## 2. 设计目标

> **核心理念**: 创建 Skill 应该是一次自然对话，而不是填表单。

- **用户只需说一句话**（自然语言描述想要什么 Skill）
- **AI 自动推断一切能推断的**（name、category、software、UE 版本、risk_level）
- **需要用户确认的通过聊天对话追问**（不弹模态窗口）
- **最终确认在聊天中完成**（"我准备好了，确认创建吗？"）

## 3. 方案设计

### 3.1 总体流程（新）

```
用户: "帮我创建一个 artclaw 技能，批量重命名场景中的 Actor 并加前缀"
        ↓
[C++ 层] IsArtclawSkillTrigger → true
        ↓
[C++ 层] 不再弹模态对话框，直接把用户输入作为普通消息发送给 AI
        ↓（通过 OpenClaw MCP 通道）
[AI 层] 收到消息，识别为 Skill 创建意图
        ↓
[AI 层] 自动收集上下文:
  • 调用 get_editor_context() → 获取 UE 版本、当前关卡名
  • 分析用户描述 → 推断 category (scene), risk_level (medium)
  • 生成 name (batch_rename_actors_prefix), display_name ("批量重命名 Actor 并加前缀")
        ↓
[AI 层] 组织确认消息，在聊天中展示:
  "我来帮你创建这个 Skill，以下是我的理解：
   📦 名称: batch_rename_actors_prefix
   📂 分类: scene
   🎯 软件: unreal_engine (UE 5.5)
   ⚠️ 风险: medium (涉及重命名操作)
   📝 描述: 批量重命名场景中的 Actor 并加前缀
   
   确认创建？(yes/no) 或者告诉我需要调整什么。"
        ↓
用户: "好的" / "分类改成 workflow"
        ↓
[AI 层] 调用 skill_generate / skill_manage(action=create) 执行创建
        ↓
[AI 层] 创建成功，展示结果:
  "✅ Skill 'batch_rename_actors_prefix' 已创建并激活！
   📁 路径: Skills/02_user/batch_rename_actors_prefix/
   📄 文件: manifest.json, __init__.py, README.md
   
   你可以试试说 '重命名所有 Actor 加上 SM_ 前缀' 来使用它。"
```

### 3.2 AI 自动获取/推断的字段

| 字段 | 获取方式 | 备注 |
|------|----------|------|
| `software` | 自动: 当前运行环境为 UE → `"unreal_engine"` | 如描述涉及 Maya/Max 则推断对应值 |
| `software_version` | 自动: 调用 `get_editor_context()` 获取 UE 版本号 | 填入 manifest 的 `software_version.min` |
| `name` | AI 推断: 从用户描述提取关键词，生成 snake_case | 替代当前 `_infer_skill_name()` 的简陋逻辑 |
| `display_name` | AI 推断: 取用户描述的精简版 | |
| `category` | AI 推断: 基于语义理解分类 | 比关键词匹配更准确 |
| `risk_level` | AI 推断: 根据操作性质判断（读/写/删除） | |
| `author` | 自动: 使用配置中的用户名或默认值 | |
| `version` | 自动: `"1.0.0"` | |
| `target_layer` | 默认 `"user"`，AI 可根据上下文调整 | 团队 Skill 需用户明确指定 |

### 3.3 需要用户确认的信息

通过 **聊天对话** 向用户追问（非弹窗），仅在 AI 无法确定时才追问：

| 场景 | AI 追问示例 |
|------|-------------|
| 描述太模糊，无法确定功能 | "你希望这个 Skill 具体做什么？比如只是查询信息，还是会修改场景？" |
| 涉及多个 DCC 软件 | "这个 Skill 是只用在 UE 里，还是也需要支持 Maya？" |
| 可能需要 C++ 接口 | "这个功能可能需要 C++ 支持（比如自定义编辑器面板），你确定要加吗？" |
| 目标层级不确定 | "这个 Skill 是你个人用的，还是要发给团队？" |

### 3.4 交互入口变更

#### 3.4.1 Create Skill 按钮（保留但行为变更）

**现在**: 点击 → 弹出模态对话框  
**改为**: 点击 → 在聊天输入框自动填充提示文本，引导用户用自然语言描述

```cpp
// 改前
FReply SUEAgentDashboard::OnCreateSkillClicked()
{
    OpenSkillCreationDialog();  // 弹窗
    return FReply::Handled();
}

// 改后
FReply SUEAgentDashboard::OnCreateSkillClicked()
{
    // 在输入框填入引导文本，光标定位到末尾让用户继续输入
    InputTextBox->SetText(FText::FromString(
        TEXT("用 artclaw 创建一个技能: ")));
    FSlateApplication::Get().SetKeyboardFocus(InputTextBox);
    return FReply::Handled();
}
```

#### 3.4.2 自然语言触发（简化判断逻辑）

**现在**: `IsArtclawSkillTrigger()` 匹配后 → 提取描述 → 弹窗或直接本地生成  
**改为**: `IsArtclawSkillTrigger()` 匹配后 → 将完整输入作为普通消息发给 AI → AI 接管后续流程

```cpp
// 改前
if (IsArtclawSkillTrigger(InputText))
{
    FString SkillDesc = ExtractSkillDescription(InputText);
    if (SkillDesc.IsEmpty())
        OpenSkillCreationDialog();      // 弹窗
    else
        StartSkillGeneration(...);      // 本地 Python 生成
    return FReply::Handled();
}

// 改后: 删除 IsArtclawSkillTrigger 拦截
// 所有消息统一发给 AI，由 AI 端通过 skill_generate MCP Tool 的 system prompt 
// 识别创建意图并走 Skill 创建流程
```

> **注意**: 这意味着 Skill 创建的意图识别从 C++ 硬编码移到 AI 侧。AI 通过 MCP Tool 的描述（`skill_generate` tool description）自然理解何时该触发。

### 3.5 AI 侧实现（skill_generate 增强）

#### 3.5.1 新增 `get_ue_environment` 辅助信息

在 `skill_mcp_tools.py` 的 `skill_generate` handler 中，自动收集环境信息：

```python
def _collect_environment_context() -> dict:
    """自动收集当前 UE 环境上下文"""
    ctx = {}
    try:
        import unreal
        # UE 版本
        ctx["ue_version"] = f"{unreal.SystemLibrary.get_engine_version()}"
        # 项目名
        ctx["project_name"] = str(unreal.Paths.get_project_file_path()).split("/")[-1].replace(".uproject", "")
        # 当前关卡
        world = unreal.EditorLevelLibrary.get_editor_world()
        if world:
            ctx["current_level"] = world.get_name()
    except Exception:
        pass
    ctx["software"] = "unreal_engine"  # 在 UE 内运行，这个是确定的
    return ctx
```

#### 3.5.2 skill_generate 返回值增强

skill_generate 的返回结果增加环境上下文，方便 AI 组织确认消息：

```python
return {
    "success": True,
    "skill_name": name,
    "display_name": display_name,
    "inferred_category": category,
    "inferred_risk_level": risk_level,
    "environment": env_context,      # 新增
    "target_layer": target_layer,
    "generated_files": { ... },
    "needs_confirmation": True,       # 新增: 标记需要用户确认
}
```

### 3.6 删除的代码

以下 C++ 函数可以**删除或简化**：

| 函数 | 处置 |
|------|------|
| `OpenSkillCreationDialog()` | **删除** — 不再需要模态对话框 |
| `StartSkillGeneration()` | **删除** — 不再由 C++ 发起本地 Python 调用 |
| `PollSkillGenerationProgress()` | **删除** — 不再轮询文件 |
| `OpenSkillPreviewDialog()` | **删除** — 预览在聊天中展示 |
| `IsArtclawSkillTrigger()` | **删除** — 意图识别移到 AI 侧 |
| `ExtractSkillDescription()` | **删除** — 由 AI 处理 |
| `ShowCppRequirementDialog()` | **保留但改为聊天确认** — C++ 需求通过 AI 追问 |

**预计减少 C++ 代码**: ~400 行

## 6. 风险与兼容性

| 风险 | 缓解措施 |
|------|----------|
| AI 不在线时无法创建 Skill | Create Skill 按钮检查连接状态，未连接时提示 "请先连接 AI (/connect)" |
| AI 推断不准确 | 确认步骤让用户有机会修改；AI 不确定时主动追问 |
| 删除 C++ 本地生成能力 | 保留 `skill_manage(action=create)` MCP Tool 作为 AI 调用入口，能力未丢失 |
| 旧版自然语言触发失效 | AI 侧通过 tool description 自然识别，不依赖 C++ 硬编码关键词 |

## 7. 验收标准

- [ ] 点击 Create Skill 按钮 → 输入框填充引导文本（无弹窗）
- [ ] 用户自然语言描述 → AI 自动推断所有可推断字段
- [ ] AI 展示推断结果并请求确认 → 用户确认后创建成功
- [ ] 描述模糊时 → AI 通过聊天追问补充信息
- [ ] AI 未连接时 → 给出友好提示
- [ ] 创建成功后 → Skill 自动注册到 MCP，立即可用
- [ ] 删除 ~400 行 C++ 弹窗代码
