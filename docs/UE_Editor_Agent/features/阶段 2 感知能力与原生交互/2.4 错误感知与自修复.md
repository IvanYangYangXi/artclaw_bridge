在 阶段 2.4，我们要让 Agent 具备“自我诊断”能力。当 AI 执行指令失败（如：设置了不存在的材质属性、移动了被锁定的 Actor）时，它不再只是报一个冷冰冰的 Error，而是能主动分析 UE Output Log 中的报错信息，并尝试自动修正指令重新执行。

# 阶段 2.4：错误感知与自修复 (Error Perception & Self-Healing)
**目标**：构建闭环反馈系统，捕获 UE 运行时的底层报错，将其转化为 AI 可理解的上下文，引导 AI 自动修正逻辑错误。

- [ ] **2.4.1 增强型错误截获 (Error Interceptor)**
    - **操作**：在 `safe_mcp_execute` 装饰器中接入堆栈跟踪（Traceback）。
    - **细节设置**：将复杂的 Python 报错简化为 AI 友好的描述（例如：将 `AttributeError: 'NoneType' ...` 转化为 `错误：无法找到名为 'Chair_1' 的 Actor`）。
- [ ] **2.4.2 属性启发式匹配 (Fuzzy Matching)**
    - **操作**：当 AI 设置材质参数失败时，返回相近的可用参数列表。
    - **细节设置**：若 AI 尝试设置 `BaseColor` 但实际参数名是 `Base_Color`，系统返回提示：“参数不匹配，该材质包含：[Base_Color, Metallic...]”。
- [ ] **2.4.3 自动重试机制 (Auto-Retry Loop)**
    - **操作**：在 OpenClaw 端配置“纠错 Prompt”。
    - **细节设置**：当收到带有 `suggestion` 字段的错误响应时，AI 自动根据建议修改参数并重新发起 `call_tool` 请求（上限 3 次）。
- [ ] **2.4.4 状态完整性校验 (State Validation)**
    - **操作**：执行后对比 Resources 状态。
    - **细节设置**：AI 移动了物体后，通过 Resource 确认坐标是否真的变了。若没变（可能被物理模拟或锁定限制），主动汇报原因。

## 核心实现代码 (self_healing.py)

```python
import unreal
import difflib

def get_best_match(target: str, possibilities: list):
    """启发式匹配：寻找最接近的属性名称"""
    matches = difflib.get_close_matches(target, possibilities, n=1, cutoff=0.6)
    return matches[0] if matches else None

@mcp_tool
@run_on_main_thread
def set_material_parameter(actor_name: str, param_name: str, value: float):
    """
    尝试设置材质参数，带自动纠错提示。
    """
    actor = find_actor(actor_name)
    mesh = actor.get_component_by_class(unreal.StaticMeshComponent)
    mid = mesh.get_material(0).create_dynamic_material_instance()
    
    # 检查参数是否存在
    all_params = [str(p.parameter_info.name) for p in mid.scalar_parameter_values]
    if param_name not in all_params:
        suggestion = get_best_match(param_name, all_params)
        error_msg = f"参数 '{param_name}' 不存在。"
        if suggestion:
            error_msg += f"你是不是想找 '{suggestion}'？"
        
        # 抛出带结构的异常，由网关层转为 MCP Error
        raise ValueError(json.dumps({
            "error": error_msg,
            "available_params": all_params,
            "can_retry": True if suggestion else False
        }))
    
    mid.set_scalar_parameter_value(param_name, value)
    return "Success"
```

## 关键技术细节
1. **区分用户错误与 AI 错误**：如果是 AI 逻辑错误（参数拼错），触发自动重试；如果是环境错误（插件未安装），则弹窗告知用户。
2. **上下文注入**：将最近 5 条执行成功的 Log 和 1 条失败的 Log 作为“短期记忆”喂给 OpenClaw，帮助它理解当前的执行环境。
3. **静默修复 vs 告知用户**：微小的拼写纠正可以静默完成，但如果涉及逻辑变更（如 AI 找不到 A 资产而尝试用 B 资产代替），必须触发 **2.2 的确认 UI**。

**🚩 里程碑**：
1. 故意给 AI 一个错误的材质参数名，AI 在收到报错后能回复：“检测到参数名有误，已自动修正为 'Base_Color' 并执行成功”。
2. 当 AI 尝试生成一个无效路径的资产时，它能通过报错信息自动去 `AssetRegistry` 搜索正确的路径。
3. 插件日志中清晰记录了“错误 -> 诊断 -> 修复”的完整链路。
