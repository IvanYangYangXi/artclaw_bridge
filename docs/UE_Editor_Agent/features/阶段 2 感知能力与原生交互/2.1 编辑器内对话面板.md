# 阶段 2.1：编辑器内对话面板 (In-Editor Chat Bridge)
**目标**：在 UE 界面中嵌入一个可停靠的 Dockable Tab，集成 OpenClaw 的交互界面，实现“边看场景边聊天”。

- [ ] **2.1.1 引入 WebBrowser 模块 (C++ Setup)**
    - **操作**：在 `UEEditorAgent.Build.cs` 中添加 `WebBrowser` 和 `WebBrowserWidget` 模块依赖。
    - **细节设置**：这是在 Slate 中渲染 HTML5 界面（OpenClaw 视图）的基础。
- [ ] **2.1.2 创建对话面板类 (Slate/UMG)**
    - **操作**：继承 `SCompoundWidget` 创建一个名为 `SAgentChatPanel` 的类。
    - **细节设置**：内部封装 `SWebBrowser` 控件，指向 OpenClaw 的本地 Web 服务地址（如 `http://localhost:3000`）。
- [ ] **2.1.3 Tab 注册与单例管理**
    - **操作**：在 `FUEEditorAgentModule` 中使用 `FGlobalTabmanager` 注册 Tab 标识符。
    - **细节设置**：关联阶段 0.3 中定义的菜单按钮，点击即唤起此面板。
- [ ] **2.1.4 双向 JS 桥接 (JS Bridge)**
    - **操作**：利用 `WebBrowser` 的 `OnConsoleMessage` 或 `BindUObject` 功能。
    - **细节设置**：让网页端的 AI 能够通过 JS 触发 UE 原生逻辑（如：点击网页上的 Actor 名称，UE 视口自动聚焦该 Actor）。

## 核心 C++ 代码参考 (SAgentChatPanel.cpp)

void SAgentChatPanel::Construct(const FArguments& InArgs)
{
    ChildSlot
    [
        SNew(SVerticalBox)
        + SVerticalBox::Slot()
        .FillHeight(1.0f)
        [
            // 嵌入 WebBrowser 控件
            SAssignNew(WebBrowserWidget, SWebBrowser)
            .InitialURL(TEXT("http://localhost:3000/ue-chat-view")) // 指向 OpenClaw 视图
            .ShowControls(false) // 隐藏浏览器导航栏，使其看起来像原生 UI
            .OnConsoleMessage(this, &SAgentChatPanel::HandleConsoleMessage)
        ]
    ];
}

## 关键技术细节
1. **DPI 缩放**：UE 的 Slate 界面有缩放机制，确保 Web 界面在 4K 屏幕下不会模糊（可通过 `FGlobalTabmanager` 的 DPI 设置同步）。
2. **本地认证**：为了安全，可以在 URL 中注入一个临时 Token（如 `?auth=ue_token_123`），确保 OpenClaw 只响应来自本插件的请求。
3. **性能开销**：`WebBrowser` 控件基于 CEF (Chromium Embedded Framework)，在不使用时应支持休眠以节省显存。
