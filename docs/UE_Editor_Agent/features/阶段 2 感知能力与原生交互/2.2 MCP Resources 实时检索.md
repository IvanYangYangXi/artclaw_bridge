在 阶段 2.2，我们要赋予 AI “观察”的能力。通过 MCP 的 Resources 机制，AI 不再是盲目执行指令，而是能像人类开发者一样，先查看场景里有什么（Outliner），再决定怎么操作。
我们将利用虚幻引擎的 离散化扫描（Scattered Scanning） 技术，将复杂的引擎对象转化为 AI 可读的结构化 JSON。

# 阶段 2.2：MCP Resources 实时检索 (Scene Perception)
**目标**：将 UE 的编辑器状态映射为 MCP 资源 URI，使 AI 能够主动拉取场景层级、资产属性和当前选中项。

- [ ] **2.2.1 定义资源 URI 规范 (URI Schema)**
    - **操作**：约定一套标准路径，如 `unreal://level/actors` (所有物体), `unreal://editor/selection` (选中项)。
    - **细节设置**：AI 通过 `read_resource(uri)` 指令即可获取对应的实时快照。
- [ ] **2.2.2 场景大纲序列化 (Outliner Serializer)**
    - **操作**：编写 Python 函数遍历 `unreal.EditorLevelLibrary.get_all_level_actors()`。
    - **细节设置**：仅提取核心字段：`Name`, `Class`, `Location`, `Tags`。避免返回冗余的底层指针数据，防止 Token 爆炸。
- [ ] **2.2.3 资产搜索接口 (Content Search)**
    - **操作**：封装 `unreal.AssetRegistry` 搜索功能。
    - **细节设置**：支持 URI 参数过滤，例如 `unreal://assets/search?type=StaticMesh&name=Chair`。
- [ ] **2.2.4 订阅与变更通知 (Subscribe/Notify)**
    - **操作**：利用 `unreal.OnLevelActorListChanged` 委托。
    - **细节设置**：当用户在场景里手动删除了物体，通过 MCP 发送 `resource_updated` 通知给 OpenClaw，确保 AI 的记忆与视口同步。

## 核心实现代码 (resource_manager.py)

```python
import unreal
import json

class UEMCPResourceManager:
    @staticmethod
    def get_scene_hierarchy():
        """获取当前场景的层级结构 JSON"""
        actors = unreal.EditorLevelLibrary.get_all_level_actors()
        scene_data = []
        for actor in actors:
            scene_data.append({
                "name": actor.get_name(),
                "class": actor.get_class().get_name(),
                "location": {
                    "x": actor.get_actor_location().x,
                    "y": actor.get_actor_location().y,
                    "z": actor.get_actor_location().z
                },
                "tags": list(actor.tags)
            })
        return json.dumps(scene_data, indent=2)

    @staticmethod
    def get_current_selection():
        """获取用户当前选中的物体详情"""
        selected = unreal.EditorLevelLibrary.get_selected_level_actors()
        return json.dumps([a.get_name() for a in selected])

# 在 MCP 协议层注册 Resource 响应
# 当 OpenClaw 请求 unreal://level/actors 时，调用 get_scene_hierarchy()
```

## 关键技术细节
1. **数据脱敏与精简**：不要把整个 `UObject` 的所有属性都塞给 AI。只给它对布局和逻辑有意义的数据（如 Transform, Material Name, Socket 等）。
2. **频率控制 (Throttling)**：如果场景有几万个物体，全量扫描会卡顿。建议实现“局部更新”或限制 AI 只能读取它感兴趣的区域。
3. **只读安全性**：Resources 在 MCP 规范中是 **Read-Only** 的。修改操作必须通过阶段 1 的 **Tools** 完成，从而维持清晰的读写分离架构。

**🚩 里程碑**：
1. 用户在场景中选中一个桌子，问 AI：“我选了什么？它在哪？”，AI 能准确回答：“你选中了 StaticMeshActor_1，位于坐标 (100, 200, 0)”。
2. AI 能够发现场景中重叠的物体（通过对比 Location 资源），并主动建议：“我发现有两个椅子重叠了，是否需要我清理？”。
3. 通过 OpenClaw 能够检索 `Content Browser` 中的模型路径，AI 不再需要用户提供完整的资源路径即可自动加载。
