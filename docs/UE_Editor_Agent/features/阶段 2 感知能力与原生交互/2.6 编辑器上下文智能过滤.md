# 阶段 2.6：编辑器上下文智能过滤 (Context-Aware Filtering)
**目标**：实时获取用户当前的“编辑焦点”（是在材质编辑器、蓝图、还是关卡视口），动态调整 AI 的 Tool 权重和 Prompt。

- [ ] **2.6.1 焦点窗口感知 (Focus Detection)**
    - **操作**：调用 `GEditor->GetActiveViewport()` 和 `GEditor->GetEditorSubsystem<UAssetEditorSubsystem>()`。
    - **细节设置**：通过 C++ 监控当前处于活动状态的编辑器窗口类型（Level, Blueprint, Material）。
- [ ] **2.6.2 动态工具屏蔽 (Tool Masking)**
    - **操作**：在 MCP `list_tools` 返回时进行条件过滤。
    - **细节设置**：若用户正在“材质编辑器”中，优先展示 `set_texture_parameter`，暂时隐藏 `spawn_actor`，减少 AI 选错工具的概率。
- [ ] **2.6.3 智能提示词注入 (Contextual Prompting)**
    - **操作**：将当前编辑器状态（例如：“用户正在编辑 BP_Hero 蓝图”）作为隐藏的 Resource 传给 OpenClaw。
    - **细节设置**：AI 能主动说：“我看到你正在优化角色蓝图，需要我检查变量命名规范吗？”。

## 核心实现代码 (context_aware.py)

```python
import unreal

def get_editor_context():
    """获取当前编辑器的上下文环境"""
    asset_subsystem = unreal.get_editor_subsystem(unreal.AssetEditorSubsystem)
    opened_assets = asset_subsystem.get_all_edited_assets()
    
    context = {
        "active_assets": [a.get_name() for a in opened_assets],
        "mode": "LevelEditor" # 默认为关卡编辑
    }
    
    if opened_assets:
        last_asset = opened_assets[-1]
        # 判断最后操作的资产类型
        if isinstance(last_asset, unreal.Material):
            context["mode"] = "MaterialEditor"
        elif isinstance(last_asset, unreal.Blueprint):
            context["mode"] = "BlueprintEditor"
            
    return context

# 在 MCP 协议层：每次用户发起对话时，自动带上此 Context 
```

2. **防呆机制**：AI 不会在你编辑蓝图时，错误地尝试在蓝图视口里生成物理 Actor。
3. **沉浸感**：AI 的回答会根据上下文变化（如：“在当前的材质节点图中，我建议...”）。
