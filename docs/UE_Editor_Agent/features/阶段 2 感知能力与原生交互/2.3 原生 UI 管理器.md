在 阶段 2.3，我们要解决 AI 操作的“最后一步确认”。
当 AI 计划执行批量删除、大规模移动或修改资产设置时，我们不希望它直接执行，而是通过 UE 原生 UI 弹出一个确认框，让开发者进行物理审核。这能够极大增强生产环境的安全性。


# 阶段 2.3：原生 UI 管理器 (Native UI Manager)
**目标**：构建 C++ 与 Python 的异步桥梁，允许 AI 脚本唤起标准的 Slate/UMG 对话框，并挂起逻辑等待用户点击结果。

- [ ] **2.3.1 定义通知与确认接口 (C++ Interface)**
    - **操作**：在 C++ 模块中创建一个单例 `UUEAgentUIManager`。
    - **细节设置**：提供 `ShowConfirmationDialog(FText Message)` 静态函数，使用 `FMessageDialog` 或自定义 `SWindow` 实现。
- [ ] **2.3.2 Python 异步挂起机制 (Awaitable UI)**
    - **操作**：利用 `asyncio.Future` 封装 C++ 的 UI 回调。
    - **细节设置**：当 Python 脚本调用 `ui.confirm()` 时，协程进入 `await` 状态，释放主线程，直到用户点击按钮。
- [ ] **2.3.3 多态 UI 模板 (UI Templates)**
    - **操作**：支持多种预设 UI 模式：
        - `SimpleConfirm`: 只有“是/否”。
        - `ParameterAdjust`: 带有滑动条或输入框的微调面板。
        - `AssetPicker`: 强制用户在 AI 建议的基础上手动选择一个资产。
- [ ] **2.3.4 交互反馈环 (Feedback Loop)**
    - **操作**：将用户的“取消”行为转化为特定的 MCP 异常。
    - **细节设置**：告知 AI：“用户拒绝了该操作，请尝试其他方案”，而不是简单地报错。

## 核心实现代码 (C++ 与 Python 协作)

### C++ 端 (实现原生弹窗)
```cpp
// 在 UUEAgentUIManager.cpp 中
bool UUEAgentUIManager::ShowModalConfirm(FString Message)
{
    EAppReturnType::Type Result = FMessageDialog::Open(
        EAppMsgType::YesNo, 
        FText::FromString(Message), 
        FText::FromString("AI Agent 确认")
    );
    return Result == EAppReturnType::Yes;
}
```

### Python 端 (异步封装)
```python
import unreal
import asyncio

async def request_user_confirmation(message: str) -> bool:
    """
    唤起 UE 原生对话框并等待用户确认。
    这是一个异步操作，不会卡死编辑器渲染。
    """
    # 切换到主线程执行 UI 弹出
    future = asyncio.get_event_loop().create_future()
    
    def _popup():
        # 调用 C++ 暴露的函数
        result = unreal.UEAgentUIManager.show_modal_confirm(message)
        # 将结果传回异步 Future
        unreal.get_interpreter().execute_on_main_thread(lambda: future.set_result(result))

    unreal.get_interpreter().execute_on_main_thread(_popup)
    return await future

# 在 Skill 逻辑中使用：
@mcp_tool
async def bulk_delete_actors(pattern: str):
    actors = find_actors(pattern)
    confirmed = await request_user_confirmation(f"AI 计划删除 {len(actors)} 个物体，确认吗？")
    if confirmed:
        # 执行删除逻辑...
        return "Deleted."
    return "User cancelled."
```

## 关键技术细节
1. **模态与非模态**：确认框必须是**模态 (Modal)** 的，防止用户在弹窗出现时继续操作场景导致状态冲突。
2. **Slate 句柄管理**：如果用户直接关闭了对话框（点击 X），需要捕获该事件并默认返回 `False` 以保证安全。
3. **样式统一**：使用 `FEditorStyle` 获取虚幻引擎标准的图标和字体，确保 AI 弹窗看起来像引擎原生功能，而非第三方外挂。

**🚩 里程碑**：
1. AI 尝试执行危险操作时，UE 视口中央能立即跳出原生的“确认/取消”蓝色对话框。
2. 在对话框未关闭前，Python 脚本保持挂起，不消耗额外 CPU 资源。
3. 点击“否”后，AI 能够在对话框中同步显示：“由于您的拒绝，我已取消该批处理任务”。
