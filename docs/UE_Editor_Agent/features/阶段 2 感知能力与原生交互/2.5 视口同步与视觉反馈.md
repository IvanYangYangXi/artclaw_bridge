在完成“感知”与“交互”的基础建设后，第二阶段（感知与交互） 还有一个极其关键的“舒适度”补丁：2.5 视口同步与视觉反馈 (Viewport Sync & Visual Feedback)。
如果 AI 操作了物体，但用户的视口（Viewport）没有跟随，或者没有任何视觉提示（如描边、高亮），用户会感到迷茫。这一步将 AI 的逻辑操作转化为可见的编辑器反馈。

# 阶段 2.5：视口同步与视觉反馈 (Viewport Feedback)
**目标**：在 AI 执行操作时，提供即时的视口对齐、物体高亮及闪烁提示，确保用户能瞬间捕捉到 AI 的改动。

- [ ] **2.5.1 自动聚焦与对齐 (Auto-Focus)**
    - **操作**：实现 `focus_on_actor(actor_name)` 工具。
    - **细节设置**：当 AI 生成或移动物体后，自动调用 `GEditor->SelectActor` 并触发 `Viewport->FocusSelected`，让视口平滑飞向目标。
- [ ] **2.5.2 临时高亮描边 (Selection Highlight)**
    - **操作**：利用 `unreal.EditorLevelLibrary.set_selected_level_actors`。
    - **细节设置**：AI 修改某个属性时，将该物体设为选中状态，利用 UE 原生的黄色描边提供反馈。
- [ ] **2.5.3 场景文字提示 (Debug Canvas/Text)**
    - **操作**：使用 `DrawDebugString` 或 `TextRenderActor`。
    - **细节设置**：在 AI 操作的位置上方显示临时的悬浮文字（如：“AI: 材质已更新”），并在 3 秒后自动消失。
- [ ] **2.5.4 操作进度条 (Progress Notification)**
    - **操作**：调用 UE 原生的 `FScopedSlowTask`。
    - **细节设置**：当 AI 执行复杂任务（如：生成 100 棵树）时，在编辑器右下角显示百分比进度条。

## 核心实现代码 (visual_feedback.py)

```python
import unreal

@mcp_tool
@run_on_main_thread
def focus_and_highlight(actor_name: str, flash: bool = True):
    """
    将视口对准指定 Actor 并进行高亮。
    """
    actor = find_actor(actor_name)
    if not actor: return "Actor not found"

    # 1. 选中物体 (触发原生描边)
    unreal.EditorLevelLibrary.set_selected_level_actors([actor])
    
    # 2. 视口对齐 (平滑移动相机)
    # 调用 C++ 暴露的全局编辑器聚焦指令
    unreal.EditorLevelLibrary.editor_set_game_view(False)
    unreal.SystemLibrary.execute_console_command(unreal.EditorLevelLibrary.get_editor_world(), "CAMERA ALIGN ACTIVEViewport TOP")
    
    # 3. 绘制调试球体 (视觉闪烁提示)
    if flash:
        loc = actor.get_actor_location()
        unreal.SystemLibrary.draw_debug_sphere(
            actor.get_world(), loc, 100.0, 12, unreal.LinearColor(0, 1, 0, 1), 2.0, 2.0
        )
    
    return f"Focused on {actor_name}"
```

## 关键技术细节
1. **防止干扰用户**：提供一个开关（如 `silent_mode`），如果用户正在手动操作，AI 的视口跟随应自动禁用，避免“抢夺相机”。
2. **多物体处理**：如果 AI 修改了一组物体，视口应自动计算包围盒（Bounding Box）并聚焦于整体中心。
3. **撤销关联**：高亮和聚焦不应计入 Undo 栈，它们仅作为临时的视觉引导。

**🚩 里程碑**：
1. 用户说“把那个桌子放大两倍”，AI 执行后，视口自动拉近到桌子前，桌子呈现选中高亮状态。
2. 执行批量任务时，编辑器右下角弹出标准的 UE 进度条，完成后消失。
3. 在 AI 调整灯光时，视口实时更新阴影和光效，用户能直观看到变化。
