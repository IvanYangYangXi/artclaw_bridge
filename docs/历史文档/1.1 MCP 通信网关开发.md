# 阶段 1.1：MCP 通信网关开发 (MCP Transport Gateway)
在 阶段 1.1，我们将构建插件的“神经连接”。由于 UE 编辑器是一个持久运行的进程，使用 WebSocket 作为传输层是最稳定的选择。OpenClaw 作为 Client 连接到 UE 内部的 Server，通过 JSON-RPC 2.0 协议进行对话。

**目标**：在 UE 进程内建立高性能、异步的 WebSocket 服务器，实现与 OpenClaw 的双向指令握手。

- [ ] **1.1.1 引入网络依赖 (Module Setup)**
    - **操作**：在 `UEClawBridge.Build.cs` 中添加 `WebSockets`, `Networking`, `Sockets` 模块。
    - **细节设置**：确保插件具备底层 TCP/UDP 访问权限，以支持 WebSocket 握手。
- [ ] **1.1.2 C++ WebSocket Server 实现**
    - **操作**：由于 UE 官方 `IWebSocket` 主要是 Client 端，建议集成轻量级 C++ 库（如 `libwebsockets` 或使用 Python 层的 `websockets` 库作为网关）。
    - **推荐路径**：为了快速迭代，优先在 **Python 层** 启动服务器，并由 C++ 模块进行生命周期管理。
- [ ] **1.1.3 JSON-RPC 2.0 协议处理器**
    - **操作**：编写消息分发器（Dispatcher），解析符合 MCP 规范的请求。
    - **细节设置**：必须处理 `initialize`, `list_tools`, `call_tool` 三类核心消息。
- [ ] **1.1.4 异步心跳与断线重连**
    - **操作**：实现 Ping-Pong 机制。
    - **细节设置**：当 OpenClaw 客户端关闭后，UE 端需自动重置监听状态，避免端口占用。

## 核心实现逻辑 (Python Server 端原型)
将此逻辑集成到 `init_unreal.py` 调用的子模块中：

```python
import asyncio
import json
import websockets
import unreal

class UEMCPServer:
    def __init__(self, host="127.0.0.1", port=8080):
        self.host = host
        self.port = port
        self.server = None

    async def handle_message(self, websocket, path):
        unreal.log(f"LogUEAgent: OpenClaw 已连接")
        try:
            async for message in websocket:
                data = json.loads(message)
                # 处理 MCP 请求
                response = await self.process_mcp_request(data)
                await websocket.send(json.dumps(response))
        except websockets.exceptions.ConnectionClosed:
            unreal.log("LogUEAgent: OpenClaw 连接已断开")

    async def process_mcp_request(self, request):
        method = request.get("method")
        # 基础握手响应示例
        if method == "initialize":
            return {
                "jsonrpc": "2.0",
                "id": request.get("id"),
                "result": {"protocolVersion": "2024-11-05", "capabilities": {}}
            }
        return {"jsonrpc": "2.0", "id": request.get("id"), "error": {"code": -32601, "message": "Method not found"}}

    def start(self):
        # 在 UE 的主线程之外运行事件循环
        loop = asyncio.new_event_loop()
        asyncio.set_event_loop(loop)
        start_server = websockets.serve(self.handle_message, self.host, self.port)
        loop.run_until_complete(start_server)
        loop.run_forever()

# 注意：在 UE 中需要通过线程拉起，避免阻塞主线程（GameThread）
import threading
mcp_server = UEMCPServer()
thread = threading.Thread(target=mcp_server.start, daemon=True)
thread.start()
```

## 关键技术细节
1. **主线程安全 (Thread Safety)**：
   - **坑点**：WebSocket 运行在后台线程，但 **`unreal` API 只能在 GameThread（主线程）执行**。
   - **对策**：在 `process_mcp_request` 中涉及引擎操作时，必须使用 `unreal.EditorLevelLibrary` 等接口前，通过 `unreal.get_interpreter().execute_on_main_thread(callback)` 切换回主线程。
2. **端口冲突检查**：
   - 启动前检查 `8080` 端口是否被占用，若被占用则自动递增或在日志中报错提示。

**🚩 里程碑**：
1. 启动 UE 后，命令行执行 `telnet 127.0.0.1 8080` 或使用 WebSocket 调试工具能成功建立连接。
2. OpenClaw 配置文件指向该地址后，能正确接收到来自 UE 的 `initialize` 响应。
3. UE Output Log 实时显示“OpenClaw 已连接”的确认信息。
