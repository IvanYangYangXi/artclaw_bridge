在 WebSocket 通讯中，心跳（Heartbeat）与断线重连（Reconnection）机制是保证 AI 指令不丢失的关键。如果 OpenClaw 崩溃或网络波动，UE 插件必须能够自动释放资源并重新进入“监听（Listening）”状态，而不是锁死端口。

# 阶段 1.1.4：心跳监控与自动重置机制
**目标**：确保 UE 插件与 OpenClaw 之间的连接具有“自愈”能力，防止因异常断开导致的端口残留或指令阻塞。

- [ ] **1. 异步 Ping-Pong 机制**
    - **操作**：在 WebSocket 握手后启动一个后台协程，每隔 30 秒发送一个 Ping 包。
    - **细节设置**：若 OpenClaw 在规定时间内未回传 Pong，则主动断开连接。
- [ ] **2. 资源自动释放 (Cleanup)**
    - **操作**：使用 `try...finally` 或 `with` 语句确保连接关闭时触发清理函数。
    - **细节设置**：清除所有挂起的 MCP 请求 ID，重置 C++ 层的“连接状态”标志位。
- [ ] **3. 自动重新监听 (Auto-Relisten)**
    - **操作**：当连接断开时，服务器不退出，而是循环回到 `accept` 状态。
    - **细节设置**：在 UE 日志中打印 `Waiting for OpenClaw to reconnect...`。

## 核心实现逻辑 (Python 强化版)

```python
import asyncio
import unreal
import websockets

class UEMCPServer:
    def __init__(self, host="127.0.0.1", port=8080):
        self.host = host
        self.port = port
        self.is_active = True

    async def heartbeat(self, websocket):
        """定期发送 Ping 以维持连接"""
        try:
            while True:
                # 发送 WebSocket 原生 Ping
                pong_waiter = await websocket.ping()
                # 等待 10 秒超时
                await asyncio.wait_for(pong_waiter, timeout=10)
                await asyncio.sleep(30) 
        except (asyncio.TimeoutError, websockets.exceptions.ConnectionClosed):
            unreal.log_warning("LogUEAgent: OpenClaw 心跳超时，关闭连接。")
            await websocket.close()

    async def handle_connection(self, websocket, path):
        """处理新连接及其生命周期"""
        unreal.log("LogUEAgent: >>> OpenClaw 已接入")
        
        # 并发运行消息处理和心跳监控
        message_task = asyncio.create_task(self.process_messages(websocket))
        heartbeat_task = asyncio.create_task(self.heartbeat(websocket))

        done, pending = await asyncio.wait(
            [message_task, heartbeat_task],
            return_when=asyncio.FIRST_COMPLETED,
        )

        # 任何一个任务结束（断开或超时），取消另一个
        for task in pending:
            task.cancel()
        
        unreal.log("LogUEAgent: <<< OpenClaw 已断开，等待下次连接...")

    async def start_server(self):
        """启动持久化服务器"""
        while self.is_active:
            try:
                async with websockets.serve(self.handle_connection, self.host, self.port):
                    # 保持协程运行，直到 self.is_active 被设为 False
                    while self.is_active:
                        await asyncio.sleep(1)
            except Exception as e:
                unreal.log_error(f"LogUEAgent: 服务器异常: {e}，5秒后尝试重启...")
                await asyncio.sleep(5)

# 在 init_unreal.py 中启动
def run_mcp_server():
    server = UEMCPServer(port=8080)
    asyncio.run(server.start_server())
```

## 关键机制说明
1. **防止端口僵死**：通过 `async with websockets.serve`，确保即使 Python 发生未捕获异常，底层 Socket 也会被正确关闭。
2. **状态反馈给 C++**：建议在 `handle_connection` 的开始和结束处，调用一个 C++ 暴露的函数（如 `SetAgentConnectionStatus(bool)`），从而实时更新阶段 0.3 中定义的 **Toolbar 图标颜色**（绿/红）。
3. **主线程协作**：心跳是在后台线程跑的，不会卡住虚幻引擎的渲染主线程。

**🚩 里程碑**：
1. 强制关闭 OpenClaw 客户端，UE `Output Log` 立即显示“连接已断开”。
2. 无需重启虚幻引擎，再次打开 OpenClaw，两者能瞬间**自动重新配对**。
3. 长时间挂机（如 1 小时）不操作，连接依然维持稳定（证明心跳有效）。
