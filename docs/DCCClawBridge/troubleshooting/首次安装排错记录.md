## 1. install.bat LF 换行导致 cmd 解析失败

**现象**: 双击 install.bat 大量乱码报错：`'鈺愨晲鈺' 不是内部或外部命令`

**根因**: bat 文件使用 Unix LF 换行，Windows cmd.exe 要求 CRLF。LF-only 导致解析错位，UTF-8 中文字节被截断后当命令执行。

**修复**: 
- LF → CRLF 转换
- 新增 `.gitattributes`: `*.bat *.cmd text eol=crlf`

**教训**: Windows bat 文件必须用 CRLF，git 可能在 checkout 时转换为 LF。

---

## 2. echo 行重定向符号

**现象**: `'3.' is not recognized`，项目根目录出现垃圾文件 `UE`, `ArtClaw`, `Chat`, `连`

**根因**: `echo ... -> UE Claw Bridge` 中 `->` 的 `>` 被 cmd 当成文件重定向。

**修复**: `→` 和 `->` 全部改为 `--`

**教训**: bat echo 行中不能有 `>`、`→`、`->` 等含 `>` 的字符。

---

## 3. Maya 中文版 locale 目录

**现象**: install.bat 安装成功但 Maya 启动无 ArtClaw 菜单

**根因**: Maya 中文版 scripts 目录是 `Documents/maya/2023/zh_CN/scripts/`，install.bat 只写 `Documents/maya/2023/scripts/`。

**修复**:
- install.bat 自动扫描 locale 子目录
- 同时安装到 scripts/ 和 locale/scripts/
- userSetup.py 加防重复初始化保护

**教训**: DCC 中文版的用户目录结构可能与英文版不同。

---

## 4. chat_panel.py 语法错误

**现象**: `invalid syntax (chat_panel.py, line 29)`

**根因**: `QFlowLayout if False else QWidget` 放在 import 语句的括号里，不是合法 Python 语法。

**修复**: 删除该行，QFlowLayout 没有实际引用。

---

## 5. bridge_core 裸导入失败

**现象**: `No module named 'bridge_core'`

**根因**: `from bridge_core import ...` 是裸导入，需要 bridge_core.py 所在目录在 sys.path 上。DCCClawBridge/ 在 sys.path 里但 core/ 不在。

**修复**: bridge_dcc.py 导入前把 `os.path.dirname(__file__)` (即 core/) 加到 sys.path。

---

## 6. websockets 依赖

**现象**: 点连接无反应，chat_panel.py 初始化 bridge 时静默失败

**根因**: Maya Python 环境是独立的，没有安装 websockets 库。

**修复**: `mayapy.exe -m pip install websockets`

**建议**: install.bat 应自动安装 Maya Python 依赖（类似 UE 部分的处理）。

---

## 7. Agent 不识别 DCC 环境

**现象**: 在 Maya 里对话，agent 以为在 UE 里

**根因**: 
1. session key 硬编码 `agent/ue-editor`
2. 消息不含 DCC 上下文信息

**修复**:
- bridge_core.py 新增 `client_id` 参数，session key 变为 `agent/{dcc}-editor`
- 各 adapter 启动时调用 `DCCBridgeManager.set_dcc_name("maya"/"max")`
- `_enrich_with_briefing()` 首条消息注入 `[DCC Context]`

---

## 8. 快捷按钮 lambda 报错

**现象**: `TypeError: <lambda>() missing 1 required positional argument: 'checked'`

**根因**: Qt `clicked` signal 在部分 PySide2 版本不传 checked 参数。

**修复**: `lambda _checked=False, c=content: ...`

---

## 9. 流式更新冲掉历史消息

**现象**: AI 回复时之前的对话记录消失

**根因**: `_update_last_message` 用 block navigation 定位最后一条消息，但 HTML 渲染后 block 数量不可控。

**修复**: 记录 AI 消息的 cursor position 锚点，流式更新时从锚点到文档末尾精确替换。

---

## 10. Max MCP Server 未配置

**现象**: Max 工具不可用

**根因**: openclaw.json 的 mcp-bridge servers 里没有 max-primary 配置。

**修复**: 添加 `max-primary: ws://127.0.0.1:8082`，工具权限加到所有 agent。

---

## 11. setup_openclaw_config.py 不自动注入工具权限

**现象**: 运行配置脚本后 agent 仍然没有 DCC MCP 工具

**根因**: 脚本只添加 server 配置，不修改 agent 的 tools.allow 列表。

**修复**: 自动根据 server name 注入对应的 MCP 工具到所有 agent 的 allow 列表。
