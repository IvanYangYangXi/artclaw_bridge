# DCCClawBridge 开发路线图

> **落地策略**：以 Maya 2023 为首发验证平台，完成通用框架后移植 3ds Max。
> **复用策略**：MCP 通信、Skill 体系、知识库等核心模块从 UEClawBridge 直接移植，不重复造轮子。

---

## 阶段 0：环境搭建与框架骨架 (Bootstrap)

**目标**：在 Maya 2023 中跑通最小可用原型——插件加载、Qt 窗口弹出、依赖自动安装。

- [ ] **0.1 项目骨架与入口**
    - 创建 `DCCClawBridge/` 目录结构
    - 编写 `userSetup.py` 自动加载入口（Maya 启动时自动执行）
    - 实现 Maya 菜单注册（ArtClaw → Open Chat Panel）
    - **验证**：Maya 启动后菜单栏出现 ArtClaw 入口

- [ ] **0.2 Maya Adapter 基础实现**
    - 实现 `BaseDCCAdapter` 抽象接口
    - 实现 `MayaAdapter`：软件信息、主窗口获取、菜单注册
    - 主线程调度：`executeDeferred` / `executeInMainThreadWithResult`
    - **验证**：`adapter.get_software_version()` 返回 "2023"

- [ ] **0.3 依赖管理**
    - 移植 UEClawBridge 的 `dependency_manager.py` 逻辑
    - 首次启动检测 `websockets`，缺失时自动安装到 `Lib/`
    - Maya 2023 Python 3.9 环境适配（pip target 路径）
    - **验证**：干净环境首次启动，自动安装依赖无报错

- [ ] **0.4 Qt 窗口骨架**
    - 最小 Chat Panel：消息列表 + 输入框 + 发送按钮
    - 正确设置 `parent=maya_main_window` 使窗口跟随 Maya
    - 窗口可关闭/重新打开，不重复创建
    - **验证**：菜单点击弹出 Chat Panel，窗口可正常交互

**🚩 里程碑**：Maya 2023 启动后自动加载插件，菜单栏可打开 Chat Panel 窗口，依赖自动安装。

---

## 阶段 1：MCP 通信与万能执行器 (Core Communication)

**目标**：打通 Maya ↔ OpenClaw 的完整 MCP 通信链路，AI 可以在 Maya 中执行代码。

- [ ] **1.1 MCP WebSocket 服务器**
    - 移植 `mcp_server.py` 到 DCCClawBridge
    - 适配 Maya 环境（独立线程运行 asyncio）
    - 端口自动探测（8081+，与 UE 的 8080 错开）
    - **验证**：Maya 启动后 MCP 服务监听成功

- [ ] **1.2 OpenClaw Bridge 通信**
    - 移植 `openclaw_bridge.py` 到 `core/bridge.py`
    - 适配 Qt 信号机制（替代 UE 的文件轮询）
    - 流式响应通过 Qt signal/slot 回传到 Chat Panel
    - **验证**：Chat Panel 发消息 → OpenClaw 回复显示

- [ ] **1.3 万能执行器 `run_python`**
    - 在 Maya 主线程安全执行用户/AI 代码
    - 安全扫描（ast 模块拦截危险操作）
    - 上下文注入：`S`=选中对象, `W`=当前场景文件, `L`=`maya.cmds`
    - Undo 包装：`cmds.undoInfo(openChunk=True/closeChunk=True)`
    - **验证**：AI 发送 `cmds.polySphere()` → Maya 中创建球体 → Ctrl+Z 撤销

- [ ] **1.4 命令队列（主线程安全）**
    - 移植 `command_queue.py`
    - MCP 线程 → 队列 → QTimer 主线程消费
    - 支持同步等待结果（Future 模式）
    - **验证**：MCP 工具调用不会导致 Maya 崩溃

- [ ] **1.5 基础查询工具**
    - `get_editor_context` — 软件/版本/场景/模式
    - `get_selected_objects` — 选中对象列表
    - `get_scene_info` — 场景概览
    - **验证**：AI 能"看到"当前场景状态

**🚩 里程碑**：在 Maya Chat Panel 中与 AI 对话，AI 可通过 `run_python` 在 Maya 中执行操作，选中对象等上下文信息可被 AI 读取。

## 阶段 2：界面完善与感知增强 (UI & Perception)

**目标**：Chat Panel 达到 UEClawBridge 同等体验，具备流式显示、/ 命令、Skill 管理等完整功能。

- [ ] **2.1 Chat Panel 完善**
    - 流式消息逐字显示
    - Markdown 渲染（加粗、列表、代码块）
    - 代码块语法高亮 + 一键复制
    - / 命令补全下拉
    - 消息历史滚动
    - **验证**：AI 长回复流式显示流畅，代码块可复制

- [ ] **2.2 本地命令实现**
    - `/connect` `/disconnect` — 连接管理
    - `/status` — 连接状态 + 服务器信息
    - `/diagnose` — 健康检查（移植 health_check.py）
    - `/clear` `/cancel` `/help` — 基础命令
    - **验证**：所有本地命令功能正常

- [ ] **2.3 主题与配色**
    - Maya 主题适配（深灰 + Maya 蓝）
    - 字体大小可配置
    - 高 DPI 支持
    - **验证**：面板视觉融入 Maya 界面

- [ ] **2.4 Dockable 面板（Maya 特有）**
    - 使用 `cmds.workspaceControl` 实现可停靠面板
    - 支持拖拽到 Maya 布局任意位置
    - 面板状态持久化（重启 Maya 后恢复位置）
    - **验证**：面板可停靠在 Maya 侧边栏

- [ ] **2.5 Maya 上下文感知**
    - 当前编辑模式检测（建模/动画/渲染/节点编辑器）
    - 选中对象变化实时通知（`scriptJob`）
    - 场景切换/新建通知
    - **验证**：AI 知道用户当前在做什么

**🚩 里程碑**：Chat Panel 体验与 UE 版本一致，面板可 Dock 到 Maya 布局，AI 能感知用户操作上下文。

## 阶段 3：Skill 体系与知识库 (Skills & Knowledge)

**目标**：移植 Skill 管理系统和知识库，Maya 具备独立的技能集合。

- [ ] **3.1 Skill Hub 移植**
    - 移植 `skill_hub.py` / `skill_loader.py` / `skill_manifest.py`
    - 适配 Maya 的 Skill 扫描路径
    - 热重载支持（文件变化自动重新加载 Skill）
    - **验证**：Maya 启动后自动发现并注册 Skill

- [ ] **3.2 Maya P0 核心 Skill 开发**
    - **场景操作**：创建/删除/复制对象、组织大纲
    - **材质管理**：创建/分配材质、查看着色网络
    - **动画基础**：设置关键帧、查看动画曲线
    - **文件操作**：导入/导出 FBX/OBJ/ABC
    - **渲染设置**：相机管理、渲染层、渲染设置查询
    - **验证**：AI 可通过 Skill 完成常见 Maya 操作

- [ ] **3.3 知识库移植**
    - 移植 `knowledge_base.py`
    - 索引 Maya Python API 文档（cmds / pymel / OpenMaya）
    - 项目命名规范和代码范例
    - **验证**：`knowledge_search("polygon smooth")` 返回相关 API

- [ ] **3.4 记忆存储**
    - 移植 `memory_store.py`
    - 存储路径：`Documents/maya/2023/ArtClaw/memory/`
    - 项目级事实 + 用户偏好
    - **验证**：AI 记住用户偏好，重启后仍然有效

- [ ] **3.5 Skill 管理面板**
    - Skill 浏览列表（按分类/来源分组）
    - Skill 启用/禁用开关
    - 自然语言创建 Skill（复用 `skill_generate` 流程）
    - **验证**：面板中可浏览和管理所有 Skill

**🚩 里程碑**：Maya 具备完整的 Skill 体系和知识库，AI 能执行 Maya 特有操作且准确率高。

## 阶段 4：Max 移植与生态 (Max Port & Ecosystem)

**目标**：验证通用架构的可移植性，在 3ds Max 上快速复制 Maya 已有能力。

- [ ] **4.1 Max Adapter 实现**
    - 实现 `MaxAdapter`：pymxs 主线程调度、菜单注册
    - Max 主窗口获取（`GetQmaxMainWindow`）
    - Max 特有的命令队列调度
    - **验证**：Max 启动后加载插件，菜单可用

- [ ] **4.2 Max 环境适配**
    - Python 版本适配（Max 2024+ 使用 Python 3.9+）
    - 依赖安装路径适配
    - Max 的 `startup` 脚本加载机制
    - **验证**：干净环境可自动部署

- [ ] **4.3 Max P0 核心 Skill**
    - 场景操作（MaxScript/pymxs 接口）
    - 材质管理
    - 修改器操作
    - **验证**：AI 可在 Max 中完成基础操作

- [ ] **4.4 团队分发**
    - Skill 包标准封装（复用 UEClawBridge 的 manifest 规范）
    - Git 同步工作流
    - **验证**：团队成员可共享 Skill

**🚩 里程碑**：3ds Max 具备与 Maya 同等的 AI Agent 能力，验证通用架构的可移植性。
