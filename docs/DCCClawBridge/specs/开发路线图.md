> **落地策略**：以 Maya 2023 为首发验证平台，完成通用框架后移植 3ds Max。

## 阶段 1：MCP 通信与万能执行器 (Core Communication)

**目标**：打通 Maya ↔ OpenClaw 的完整 MCP 通信链路，AI 可以在 Maya 中执行代码。

- [ ] **1.1 MCP WebSocket 服务器**
    - 移植 `mcp_server.py` 骨架到 `core/mcp_server.py`
    - 注册 Maya 特有的 MCP 工具（`run_python`、`get_selected_objects` 等）
    - 适配 Maya 环境（独立线程运行 asyncio，QTimer tick 驱动主线程操作）
    - 端口自动探测（8081+，与 UE 的 8080 错开）
    - ⚠️ **UE 教训 #1**: mcp-bridge 延迟发现已在 G1.1 修复，但 DCC 侧端口探测要稳定
    - ⚠️ **UE 教训 #6**: 热重载时端口未释放 → 实现 `shutdown()` 先关闭旧 server 再启新的
    - **验证**：Maya 启动后 MCP 服务监听成功，mcp-bridge 可连接并发现工具

- [ ] **1.2 OpenClaw Bridge 通信** ⚡ 大幅简化
    - `core/bridge_dcc.py` 已在 G3.1 完成，直接调用 `bridge_core.py`
    - 剩余工作：将 Qt signal 连接到 Chat Panel UI
    - ⚠️ **UE 教训 #2~5,7**: Gateway 协议踩坑已全部在 bridge_core.py 修复，DCC 侧零成本复用
    - 📝 **UE 教训 #8**: /cancel 双端联动 — UI 清"思考中"状态 + bridge.cancel() 释放等待
    - 📝 **UE 教训 #9**: 新请求前先 cancel 旧请求，bridge_dcc.py 的 send_message 已内置
    - **验证**：Chat Panel 发消息 → OpenClaw 回复 → 流式显示

- [ ] **1.3 万能执行器 `run_python`**
    - MCP 工具注册到 `core/mcp_server.py`
    - 在 Maya 主线程安全执行（通过命令队列）
    - 🔁 **UE 移植 #10**: Static Guard — AST 安全扫描拦截危险调用
      - 移植 `static_guard.py`：黑名单模块 (os.system, subprocess, shutil.rmtree)
      - 白名单路径（只允许访问项目目录和 Maya 脚本目录）
    - 🔁 **UE 移植 #11**: Undo Guard — `cmds.undoInfo(openChunk/closeChunk)` 包装
    - 🔁 **UE 移植 #12**: 上下文变量注入
      - `S` = `cmds.ls(selection=True, long=True)` 选中对象
      - `W` = `cmds.file(query=True, sceneName=True)` 场景文件
      - `L` = `maya.cmds` 模块（方便 AI 直接用 `L.polySphere()`）
    - 🔁 **UE 移植 #13**: stdout 捕获 — `io.StringIO` 重定向 print 输出回传给 AI
    - 📝 **UE 教训 #15**: 执行超时保护 — 用 threading.Timer 或 signal.alarm 防止死循环卡死
    - **验证**：AI 发送 `cmds.polySphere()` → Maya 创建球体 → Ctrl+Z 撤销

- [ ] **1.4 命令队列（主线程安全）**
    - 实现 `core/command_queue.py`
    - MCP 线程 → queue.Queue → QTimer.singleShot(0) 或 executeDeferred 主线程消费
    - 支持同步等待结果（concurrent.futures.Future 模式）
    - 📝 **UE 教训 #14**: UE 用 FTSTicker 轮询，Maya 用 QTimer 更自然
    - **验证**：MCP 工具调用不会导致 Maya 崩溃

- [ ] **1.5 基础查询工具**
    - `get_editor_context` — 软件名/版本/场景文件/当前模式/Python版本
    - `get_selected_objects` — 选中对象列表（name/long_name/type）
    - `get_scene_info` — 场景概览（对象数/Mesh数/帧范围/FPS/上轴）
    - 📝 **UE 教训 #24**: MCP 工具精简原则 — 查询保留独立工具，写操作全走 run_python
    - **验证**：AI 能"看到"当前场景状态

**🚩 里程碑**：在 Maya Chat Panel 中与 AI 对话，AI 可通过 `run_python` 在 Maya 中执行操作。
