# DCCClawBridge 子项目概述

> **定位**：Maya / 3ds Max 共享的 AI Agent 桥接插件，通过统一的 Qt 界面和薄适配层接入 ArtClaw MCP 体系。

---

## 一、项目背景

UEClawBridge 已在 Unreal Engine 上完成从 MCP 通信到 Skill 管理的完整链路验证。Maya 和 3ds Max 作为影视/游戏行业的核心 DCC 工具，是 ArtClaw 覆盖的第二优先级目标。

**关键洞察**：Maya 和 Max 的 AI Agent 需求（聊天面板、Skill 管理、连接状态、上下文采集）高度相似，且两者均支持 PySide2/Qt 作为 UI 框架，具备做统一界面的技术基础。

## 二、设计目标

| 目标 | 说明 |
|------|------|
| **一套 UI** | 统一的 Qt Chat Panel + Skill 管理面板，Maya/Max 共享 |
| **薄适配层** | 每个 DCC 只需 ~200 行 adapter 代码处理启动/调度/上下文 |
| **复用通信框架** | Gateway 连接（bridge_core.py）、MCP 协议处理（mcp_server.py 骨架）从 UE 直接复用 |
| **独立 MCP 工具** | 各 DCC 有自己的 MCP Server 实例和工具集（Maya 操作 cmds/pymel，Max 操作 pymxs） |
| **一键部署** | 复制到 DCC 脚本目录即可工作，自动处理依赖 |

## 三、与 UEClawBridge 的关系

```
ArtClaw Bridge
├── openclaw-mcp-bridge/              ← 通信框架（所有 DCC 共享代码）
│   ├── bridge_core.py                #   Gateway WebSocket 核心（平台无关）
│   ├── bridge_config.py              #   配置加载
│   ├── bridge_diagnostics.py         #   连接诊断
│   └── mcp-bridge/                   #   OpenClaw Gateway 侧插件
│
├── subprojects/UEDAgentProj/         ← UE 专用（Slate UI + C++ 生命周期）
│   └── Plugins/UEClawBridge/
│       └── Content/Python/
│           └── openclaw_bridge.py    #   UE 适配层（文件轮询 IPC）
│
└── subprojects/DCCClawBridge/        ← Maya/Max 共享（Qt UI + Python 生命周期）
    ├── artclaw_ui/                   #   通用 Qt 界面
    ├── adapters/                     #   DCC 薄适配层
    ├── core/                         #   共享核心逻辑
    │   └── bridge_dcc.py             #   DCC 适配层（Qt signal/slot IPC）
    └── skills/                       #   DCC 特有 Skill
```

**共享的通信框架**（代码共享，各 DCC 进程独立运行）：
- `bridge_core.py` — OpenClaw Gateway WebSocket RPC 客户端（平台无关）
- `bridge_config.py` — OpenClaw 配置加载
- `mcp_server.py` 骨架 — MCP WebSocket 服务器框架（JSON-RPC 协议处理、工具注册机制）

**每个 DCC 独立的**（代码和运行时都独立）：
- MCP Server 实例 — 各 DCC 进程内独立运行，端口不同
- MCP 工具集 — UE 暴露 `run_ue_python`/`get_selected_actors`，Maya 暴露 `run_python`/`get_selected_objects`
- 适配层 — UE 用 `openclaw_bridge.py`（文件轮询），Maya/Max 用 `bridge_dcc.py`（Qt signal/slot）
- Skill 集合 — 各 DCC 操作各自原生 API

**多 DCC 并存时的架构**：
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   UE Editor │     │   Maya 2023 │     │  3ds Max    │
│             │     │             │     │             │
│ MCP Server  │     │ MCP Server  │     │ MCP Server  │
│ :8080       │     │ :8081       │     │ :8082       │
│ 22 tools    │     │ 12 tools    │     │ 10 tools    │
└──────┬──────┘     └──────┬──────┘     └──────┬──────┘
       │                   │                   │
       └─────────┬─────────┴─────────┬─────────┘
                 │ MCP JSON-RPC       │
          ┌──────▼──────────────────▼──────┐
          │    OpenClaw Gateway             │
          │    mcp-bridge 插件              │

```
每个 DCC 是独立的 MCP Server，AI 通过工具名前缀区分操作目标。

| DCC | 最低版本 | 首发测试版本 | Python | Qt |
|-----|---------|------------|--------|-----|

## 五、核心技术约束

1. **主线程安全**：Maya/Max 的场景 API 只能在主线程调用
   - Maya：`maya.utils.executeDeferred()` / `maya.utils.executeInMainThreadWithResult()`
   - Max：`pymxs.runtime.execute()` 或 Qt `QTimer.singleShot(0, fn)`
2. **Python 环境隔离**：使用 DCC 内置 Python，依赖包安装到插件私有 `Lib/` 目录
3. **UI 线程**：Qt 事件循环与 DCC 主线程共享，WebSocket 通信必须在独立线程
4. **无 C++ 依赖**：纯 Python 实现，降低部署复杂度
