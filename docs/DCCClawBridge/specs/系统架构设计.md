## 三、核心模块设计

### 3.1 DCC Adapter 抽象接口

```python
# adapters/base_adapter.py

from abc import ABC, abstractmethod
from typing import Any, Dict, List, Optional, Callable

class BaseDCCAdapter(ABC):
    """DCC 适配层抽象接口，每个 DCC 实现一个子类"""

    # ── 基础信息 ──

    @abstractmethod
    def get_software_name(self) -> str:
        """返回软件名称，如 'maya', 'max'"""

    @abstractmethod
    def get_software_version(self) -> str:
        """返回软件版本，如 '2023', '2024'"""

    @abstractmethod
    def get_python_version(self) -> str:
        """返回内置 Python 版本"""

    # ── 生命周期 ──

    @abstractmethod
    def on_startup(self) -> None:
        """DCC 启动时调用：注册菜单、shelf、启动面板"""

    @abstractmethod
    def on_shutdown(self) -> None:
        """DCC 关闭时调用：清理资源、断开连接"""

    # ── 主线程调度 ──

    @abstractmethod
    def execute_on_main_thread(self, fn: Callable, *args) -> Any:
        """在 DCC 主线程执行函数（场景 API 安全）"""

    @abstractmethod
    def execute_deferred(self, fn: Callable, *args) -> None:
        """延迟到主线程空闲时执行（非阻塞）"""

    # ── 上下文采集 ──

    @abstractmethod
    def get_selected_objects(self) -> List[Dict]:
        """获取当前选中对象列表"""

    @abstractmethod
    def get_scene_info(self) -> Dict:
        """获取当前场景基本信息（场景名、对象数、帧范围等）"""

    @abstractmethod
    def get_current_file(self) -> Optional[str]:
        """获取当前文件路径"""

    # ── UI 集成 ──

    @abstractmethod
    def get_main_window(self) -> Any:
        """获取 DCC 主窗口句柄（作为 Qt 窗口 parent）"""

    @abstractmethod
    def register_menu(self, menu_name: str, callback: Callable) -> None:
        """在 DCC 菜单栏注册入口"""

    # ── 代码执行 ──

    @abstractmethod
    def execute_code(self, code: str, context: Dict = None) -> Dict:
        """在 DCC 环境中执行 Python 代码（万能执行器）"""
```

### 3.2 Maya Adapter 实现要点

```python
# adapters/maya_adapter.py （核心骨架）

import maya.cmds as cmds
import maya.utils
import maya.OpenMayaUI as omui
from shiboken2 import wrapInstance
from PySide2 import QtWidgets

class MayaAdapter(BaseDCCAdapter):

    def get_software_name(self) -> str:
        return "maya"

    def get_software_version(self) -> str:
        return cmds.about(version=True)  # "2023"

    def execute_on_main_thread(self, fn, *args):
        return maya.utils.executeInMainThreadWithResult(fn, *args)

    def execute_deferred(self, fn, *args):
        maya.utils.executeDeferred(fn, *args)

    def get_selected_objects(self):
        sel = cmds.ls(selection=True, long=True)
        result = []
        for obj in (sel or []):
            result.append({
                "name": obj.split("|")[-1],
                "long_name": obj,
                "type": cmds.objectType(obj),
            })
        return result

    def get_main_window(self):
        ptr = omui.MQtUtil.mainWindow()
        return wrapInstance(int(ptr), QtWidgets.QMainWindow)

    def register_menu(self, menu_name, callback):
        if cmds.menu(menu_name, exists=True):
            cmds.deleteUI(menu_name)
        cmds.menu(menu_name, label="ArtClaw", parent="MayaWindow")
        cmds.menuItem(label="Open Chat Panel", command=lambda _: callback())

    def execute_code(self, code, context=None):
        # 在主线程安全执行，注入上下文变量
```

### 3.3 主线程命令队列

与 UEClawBridge 相同的模式：MCP/WebSocket 线程不直接调用 DCC API，而是通过命令队列转发到主线程。

```
WebSocket 线程                    主线程（Qt 事件循环）
     │                                │
     │  command_queue.put(cmd)         │
     │ ──────────────────────────────► │
     │                                │  QTimer(0) 触发消费
     │                                │  adapter.execute_on_main_thread(cmd)
     │  result_future.set_result()    │
     │ ◄────────────────────────────── │
```

**Maya 特有**：Maya 的 Qt 事件循环已经是主线程，可直接用 `QTimer.singleShot(0, fn)` 或 `maya.utils.executeDeferred()` 实现。

沿用 UEClawBridge 已验证的工具设计原则：

| `skill_list` | 列出已注册 Skill | 查询 |

## 五、数据流

### 5.1 用户发送消息

```
用户在 Chat Panel 输入
    │
    ▼
chat_panel.py 判断是否 / 命令
    │
    ├── 本地命令 → 直接执行（/clear, /status 等）
    │
    └── AI 消息 → core/bridge.py
                    │
                    ▼
              openclaw_bridge.py → WebSocket → OpenClaw Gateway
                                                    │
                                                    ▼
                                              AI Agent 处理
                                                    │
                                                    ▼
                                              MCP Tool Call
                                                    │
                    ┌───────────────────────────────┘
                    ▼
              mcp_server.py 接收 Tool Call
                    │
                    ▼
              command_queue.put(tool_call)
                    │
                    ▼ （主线程消费）
              adapter.execute_on_main_thread()
                    │
                    ▼
              DCC 原生 API 执行
                    │
                    ▼
              结果回传 → AI → 流式显示到 Chat Panel
```

### 5.2 AI 流式响应

```
OpenClaw Gateway
    │ WebSocket delta 事件
    ▼
bridge.py (独立线程)
    │ signal/callback
    ▼
chat_panel.py (Qt 主线程)
    │ QMetaObject.invokeMethod / signal-slot
    ▼
消息气泡逐字追加显示
```

## 六、依赖管理

### 6.1 必需依赖

| 包 | 用途 | 安装方式 |
|----|------|---------|
| `websockets` | MCP WebSocket 通信 | pip install --target |
| PySide2 | Qt 界面 | DCC 内置（无需安装） |

### 6.2 安装策略

```
插件首次启动
    │
    ▼
检测 Lib/ 目录下是否有 websockets
    │
    ├── 有 → 正常启动
    │
    └── 无 → 后台线程 pip install --target Lib/
              │
              ▼
         安装完成 → 加载模块 → 正常启动
```

与 UEClawBridge 策略一致：安装到插件私有 `Lib/` 目录，不污染 DCC 环境。

## 七、与 UEClawBridge 的差异对照

| 方面 | UEClawBridge | DCCClawBridge |
|------|-------------|---------------|
| 语言 | C++ + Python | 纯 Python |
| UI 框架 | Slate (UE 原生) | PySide2/Qt |
| 主线程调度 | `FTSTicker` + `ExecuteOnMainThread` | `QTimer` + `executeDeferred` |
| 生命周期 | `UEditorSubsystem` | `userSetup.py` / 启动脚本 |
| 部署方式 | UE 插件 (.uplugin) | Python 包（复制到 scripts/） |
| 万能执行器 | `run_ue_python` (操作 `unreal` 模块) | `run_python` (操作 `maya.cmds` / `pymxs`) |
| 上下文变量 | S=选中Actor, W=World, L=unreal | S=选中对象, W=当前场景, L=cmds/pymxs |
